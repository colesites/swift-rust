# SR0130 — Promise returned from config

The Swift-Rust config loader treats most config hooks as synchronous. A config file that returns a `Promise` from a sync hook fails the load with this error. The same loader is used for `swift-rust.config.json`, `.toml`, `.yaml`, and the equivalent `.js`/`.mjs` files.

## Why this error occurred

The config is read at process start, before any async runtime is initialized. If a hook returns a Promise, there is no scheduler to await it on. Swift-Rust has to fail.

Two patterns cause this:

- You wrote `module.exports = async () => ({...})` and the framework expects a sync value.
- A third-party plugin (often a webpack/turbo/swc extension) returns a Promise from a hook that the framework invokes synchronously.

The async form is supported on a small set of hooks (documented in the config reference). Anything else must be sync.

## Reproducing the error

```js
// swift-rust.config.mjs
export default async function config() {
  const pkg = await import("./package.json", { with: { type: "json" } });
  return { name: pkg.default.name };
}
```

## Error output

```
[ERROR] [SR0130] promise returned from config
  --> swift-rust.config.mjs:1:16
    |
  1 | export default async function config() {
    |                ^^^^^^^^^^^^^^^^^^^^^^^^^
    | this hook must be synchronous; Swift-Rust does not await the returned Promise.
    |
    help: convert the function to a sync one. for hooks documented as async, use the async form.
```

## How to fix it

1. Convert the function to a sync one and resolve all data eagerly:

```js
// swift-rust.config.mjs
import { readFileSync } from "node:fs";
import { fileURLToPath } from "node:url";

const pkg = JSON.parse(
  readFileSync(new URL("./package.json", import.meta.url), "utf8"),
);

export default {
  name: pkg.name,
};
```

2. If the hook is documented as async, keep the async form:

```js
// swift-rust.config.mjs
export default async function config() {
  const r = await fetch("https://api.example.com/flags");
  const flags = await r.json();
  return { featureFlags: flags };
}
```

3. Re-run the command. The config should load cleanly.

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=config=trace bun swift-rust dev
```

`config=trace` prints the resolved config and the file the loader is currently inspecting.

## Common pitfalls

- JSON, TOML, and YAML configs are always sync. Only the `.js`/`.mjs` variants support hooks.
- A `Promise` returned from a plugin (for example, a webpack plugin) is a different code path — see SR0071.
- `await import("...")` at the top level of a config file is allowed only inside an async hook. Move the import to a sync `import` statement.
- Some IDEs auto-format async functions into a Promise-returning shape. Disable autoformatting for the config file.
