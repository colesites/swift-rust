# SR0075 — Invalid React version

Swift-Rust requires React 18 or newer. The framework depends on streaming SSR, the `use` hook, server components, and a handful of other React 18 features that are not backported. When the installed `react`/`react-dom` is below the minimum, the framework refuses to start with this error.

## Why this error occurred

The version check runs at boot. The framework reads the resolved version of `react` from `node_modules/react/package.json` and compares it against the `peerDependencies` declared in `swift-rust`. If the resolved version is below `18.0.0`, the error is raised.

Common causes:

- The project's `package.json` pins `react: 17.x` because the team has not finished a React 18 upgrade.
- A library in the dependency tree depends on `react@17` and hoists a copy into `node_modules/react`, masking the project's intended version.
- A workspace package (in a monorepo) installs React 17 locally and is accidentally imported by a top-level route.
- The lockfile was generated against React 18 and the user is restoring it on a system where `bun install` resolved a different version.

The error fires before any code is executed, so the dev server fails immediately.

## Reproducing the error

```json
{
  "dependencies": {
    "react": "17.0.2",
    "react-dom": "17.0.2"
  }
}
```

```bash
bun swift-rust dev
```

## Error output

```
error: [SR0075] invalid React version
  --> node_modules/react/package.json
   |
   = detected react@17.0.2. Swift-Rust requires react@^18.0.0.
   = upgrade react and react-dom to a supported version, then run
     `bun install` and try again.
```

## How to fix it

1. Upgrade `react` and `react-dom` to a supported version.

   ```bash
   bun add react@latest react-dom@latest
   ```

2. In a monorepo, ensure every workspace package resolves to the same React copy. Use a single root `node_modules` or a tool like `bun install --linker=hoisted` to dedupe.

   ```bash
   bun install --linker=hoisted
   ```

3. If a transitive dependency pins React 17, file an issue upstream or use a peer-dep override.

   ```json
   {
     "overrides": {
       "react": "^18.3.0",
       "react-dom": "^18.3.0"
     }
   }
   ```

4. After upgrading, restart the dev server. The version check is cached for the lifetime of the process.

## Diagnosis

```bash
bun swift-rust doctor
cat node_modules/react/package.json | grep version
bun pm ls react react-dom
RUST_LOG=swift_rust=react=trace bun swift-rust dev
```

## Common pitfalls

- Multiple React copies is the most common cause of "this looks right but still fails". `bun pm ls react` shows every copy in the tree.
- `react-dom/server` is a separate subpath that may not match the top-level `react-dom` version. The framework checks both.
- React 19 is supported as of Swift-Rust 0.2; older 0.1.x releases cap at 18.x.
- This is the Swift-Rust analogue of Next.js's `invalid-react-version`. The fix is identical.
