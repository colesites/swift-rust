# SR0024 — Circular structure in serialized props

A server component passed a value to a client component, and that value contained a cycle (object A references object B, which references A). The RSC payload uses JSON, so circular structures cannot be encoded.

## Why this error occurred

Server components stream their output to the client as a structured payload. The serializer walks the prop graph and emits each value once. As soon as it sees an object it has already visited, it cannot decide whether to inline it again (which would multiply the payload) or emit a reference (which JSON does not support). Either choice would change semantics, so the build refuses to serialize the prop and raises this error.

The most common accidental cycle is a Node-style `req` instance, where `req.socket` points back into the request object itself.

## Reproducing the error

```tsx
import { headers } from "swift-rust/headers";

export default async function Page() {
  const h = await headers();
  return <Child value={h} />;
}
```

`headers()` returns a request-bound object whose internal references cycle. Passing it raw is the typical mistake.

## Error output

```
[ERROR] [SR0024] circular structure in serialized props
   --> app/page.tsx:5:18
   |
 5 |   return <Child value={h} />;
   |                  ^^^^^^^^^ cycle detected: value -> headers -> request -> value
   |
  help: pass plain fields (e.g. userAgent) instead of the request object.
```

## How to fix it

1. Pick the specific fields the client needs and pass them as primitives or plain objects.
2. If two objects really do reference each other, give one side an `id` and look the other up on the client.
3. For stateful handles (DB connections, sockets), keep them inside a server-only module and call server functions instead of passing the handle.

```tsx
import { headers } from "swift-rust/headers";

export default async function Page() {
  const h = await headers();
  return <Child userAgent={h.get("user-agent") ?? "unknown"} />;
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log prints the path through the prop graph that contained the cycle, so you can find the exact field that needs to be replaced.

## Common pitfalls

- `Date`, `Map`, `Set`, and `BigInt` are not cycles, but they also fail to round-trip through JSON. Convert them to plain values before passing.
- A `class` instance that references `this` somewhere in its prototype chain will look like a cycle to the serializer.
- A `WeakMap`/`WeakRef` is unserializable for the same reason; pass the underlying key/value instead.
- This error only fires for props that actually cross the server/client boundary. Two server components passing the same object is fine.
