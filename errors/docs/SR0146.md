# SR0146 — SWC minifier enabled

The build pipeline is using the SWC minifier instead of Terser. The minifier is 7x faster than Terser with comparable output, and the framework prints this informational message the first time the minifier runs in a session.

## Why this error occurred

The minifier is a separate stage from the compiler. SWC's minifier is a Rust-based replacement for Terser, and the framework defaults to it because the build is meaningfully faster with it. The informational message exists so the user knows the choice is being made and can opt out if they have a reason to.

The minifier is fully compatible with Terser's output for the common case. The few differences (aggressive string deduplication, comment stripping) do not affect runtime behaviour.

## Reproducing the error

```json
{
  "compiler": {
    "swcMinify": true
  }
}
```

## Error output

```
[INFO] [SR0146] SWC minifier enabled
  --> swift-rust.config.json:2:5
    |
  2 |     "swcMinify": true,
    |     ^^^^^^^^^^^^^^^
    | minification is performed by SWC (~7x faster than Terser, comparable output).
    |
    help: this is informational; no action required. set `swcMinify: false` to use Terser.
```

## How to fix it

1. No action is required. The build proceeds and the minifier runs.

2. If you need to opt out (for example, to debug a minification regression), set the flag to `false`:

```json
{
  "compiler": {
    "swcMinify": false
  }
}
```

3. Re-run the build.

## Diagnosis

```bash
bun swift-rust doctor
RUST_LOG=swift_rust=compiler=debug bun swift-rust build
```

`compiler=debug` prints which minifier is active and how long it ran. The doctor output lists the resolved compiler flags.

## Common pitfalls

- The flag controls the *minifier* (the post-compile step that shrinks the output), not the *compiler* (the step that transforms JSX/TypeScript). They are different stages with different configs.
- SWC's minifier is not 100% drop-in compatible with Terser. If you depend on a specific Terser option (for example, `keep_classnames`), the framework may not have a 1:1 equivalent.
- The informational message is printed once per process. Repeated builds in the same terminal will not reprint it.
- The flag is independent of `forceSwcTransforms`. You can use SWC for the compiler and Terser for the minifier, or vice versa.
