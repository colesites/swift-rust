# SR0032 — CSS imported by a dependency

A package in `node_modules` imports a raw CSS file from its own source. The Swift-Rust bundler cannot know how that CSS should be treated (global vs module, what order to inject it in, what the emitted class names should be), so the build warns and skips the file.

## Why this error occurred

Unlike a CSS Module — which has class names that need to be hashed, but at least has a well-defined contract — a raw CSS import raises several questions at once: should the file be loaded as global CSS or treated as a module? What order does it need to be injected in? Should class names be preserved as-is, or mangled? The answers are not in the file itself, so the bundler has to ask the package author, not guess.

A package that needs its styles to be loaded by the consumer should either (a) ship a compiled build that has the CSS inlined and the import rewritten, or (b) document a specific import path the consumer should add to their root layout.

## Reproducing the error

```bash
node_modules/some-pkg/src/styles.css
```

```ts
import "./styles.css";
```

## Error output

```
[WARN] [SR0032] CSS imported by a dependency: some-pkg
   file: node_modules/some-pkg/src/styles.css
   |
   help: the package was installed from source. Use the published build, or add it to transpilePackages.
```

## How to fix it

1. Replace the source install with the published npm tarball.
2. If the package is a first-party monorepo package, add it to `transpilePackages` so Swift-Rust compiles it through the standard pipeline.
3. If neither option is viable, ask the maintainer to publish a build that either inlines the CSS or ships a `style.css` import that the consumer should add to their root layout.

```json
{
  "name": "consumer",
  "build": {
    "transpilePackages": ["some-pkg"]
  }
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log lists every CSS file pulled in from a dependency, so you can find and fix each one.

## Common pitfalls

- A `transpilePackages` entry only works for in-tree packages. Random npm packages cannot be transpiled this way.
- Some packages ship both `style.css` (consumer import) and `style.module.css` (internal). Make sure the consumer is importing the right one.
- The warning is non-fatal. A consumer app that never renders the affected component will not notice anything is wrong, which makes the warning easy to miss.
- This warning is a sibling of `SR0031`, which covers CSS Modules specifically.
