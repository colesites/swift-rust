# SR0099 — Unconfigured image localPatterns

The `<Image>` component received a `src` that points to a local path, and the path is not allowed by `image.localPatterns` in `swift-rust.config.json`. The build refuses to optimize the image.

## Why this error occurred

Local images (paths that start with `/` and resolve to a file in `public/` or a colocated source) go through the same optimizer as remote images. To keep the optimizer's behavior predictable and to prevent it from being used as a file-existence oracle for arbitrary paths, Swift-Rust requires an explicit allow-list of path globs.

The match is on the URL pathname only; protocol and hostname are not relevant for local images. A pattern is an object with:

- `pathname` — a glob. `*` matches a single segment, `**` matches any number of segments.
- `search` — optional. If set, the URL's query string must match exactly, including the leading `?`. If `search` is `''`, only URLs without a query string are allowed. If `search` is omitted, any query string is allowed.

A pattern that is too narrow, a `pathname` that is misspelled, or a `search` that does not match the actual URL all produce this error.

## Reproducing the error

```tsx filename="app/page.tsx"
import Image from "swift-rust/image"

export default function Page() {
  return (
    <Image
      src="/assets/hero.jpg?v=2"
      alt="Hero"
      width={1200}
      height={600}
    />
  )
}
```

`swift-rust.config.json`:

```json
{
  "image": {
    "localPatterns": [
      { "pathname": "/assets/", "search": "" }
    ]
  }
}
```

The pattern has two problems: the pathname is missing the `**` glob, and `search: ""` blocks the `?v=2` query string.

## How to fix it

1. Widen the `pathname` glob to cover the actual paths.
2. Decide whether query strings should be allowed (omit `search`) or pinned (set it explicitly).

```json filename="swift-rust.config.json"
{
  "image": {
    "localPatterns": [
      {
        "pathname": "/assets/**"
      }
    ]
  }
}
```

```tsx filename="app/page.tsx"
import Image from "swift-rust/image"

export default function Page() {
  return (
    <Image
      src="/assets/hero.jpg?v=2"
      alt="Hero"
      width={1200}
      height={600}
    />
  )
}
```

## Common pitfalls

- `pathname` patterns are case-sensitive. `/Assets/**` does not match `/assets/...`.
- Setting `search: ''` is a way to forbid all query strings, but it is rarely what you want. Omit `search` to allow any.
- A pattern that does not end in `**` matches only one segment under the prefix. `/assets/*` matches `/assets/hero.jpg` but not `/assets/sub/hero.jpg`.
- A trailing slash in `pathname` matters: `/assets` is different from `/assets/`.
