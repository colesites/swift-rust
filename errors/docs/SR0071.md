# SR0071 — Invalid `<Link>` with `<a>` child

Older versions of link components required an explicit `<a>` element as a child, since the link wrapper only handled the click handler and the router integration. The current `<Link>` component from `swift-rust/link` renders as an `<a>` itself: it accepts the standard anchor props (`href`, `className`, `style`, `id`, `target`, `rel`, …) and emits a single tag in the DOM. Wrapping it in another `<a>` is invalid HTML and is rejected at build time.

## Why this error occurred

The component surface changed once `<Link>` started owning its own `<a>` rendering. The old pattern was redundant and produced nested anchors (`<a><a>…</a></a>`), which browsers parse but accessibility tools flag. The framework raises this error to catch the leftover code rather than ship a broken DOM.

Common sources:

- A project migrated from a pre-`13` framework version without running the codemod.
- A developer added an `<a>` "just to be safe" without checking the docs.
- A snippet was copied from a third-party tutorial that targets the old API.

The error fires at the next dev rebuild, or at the start of `bun swift-rust build`.

## Reproducing the error

```tsx
import Link from "swift-rust/link";

export function Nav() {
  return (
    <Link href="/about">
      <a id="about-link" className="nav-link">About</a>
    </Link>
  );
}
```

## Error output

```
error: [SR0071] invalid <Link> with <a> child
  --> app/components/Nav.tsx:5:3
   |
 5 |       <a id="about-link" className="nav-link">About</a>
   |       ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
   |
   = `<Link>` from `swift-rust/link` renders its own anchor. nesting
     an `<a>` inside it produces an invalid DOM. move the attributes
     onto `<Link>` itself.
```

## How to fix it

1. Move the inner anchor's attributes onto `<Link>` and drop the inner tag.

   ```tsx
   import Link from "swift-rust/link";

   export function Nav() {
     return (
       <Link href="/about" id="about-link" className="nav-link">
         About
       </Link>
     );
   }
   ```

2. To upgrade an entire codebase in one pass, run the codemod.

   ```bash
   bunx @swift-rust/codemod new-link .
   ```

   This rewrites `<Link><a id="x">Foo</a></Link>` into `<Link id="x">Foo</Link>` across the project.

## Diagnosis

```bash
bun swift-rust doctor
rg -l "<Link" app/ | xargs rg "<a "
RUST_LOG=swift_rust=link=trace bun swift-rust build
```

## Common pitfalls

- The codemod only handles the static pattern. Dynamic or conditional `<a>` children have to be migrated by hand.
- If you actually need a custom anchor element (e.g. a React component that renders a different tag), do not use `<Link>` — use a plain `<a>` and call `router.push` on click.
- The error fires at build time. The dev server will surface it as a fast-refresh warning, but a production build will fail.
- The original Next.js version of this error suggested running `npx @next/codemod new-link .`. The Swift-Rust equivalent is `@swift-rust/codemod`.
