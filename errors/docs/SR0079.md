# SR0079 — Invalid script

In Swift-Rust, scripts are loaded using a standard `<script>` tag inside the `app/` tree. The framework expects the tag to contain a `src` URL (for an external script) or inline script content (either as a string child or via `dangerouslySetInnerHTML`). A script that has neither is rejected because it would render an empty `<script>` element in the DOM.

## Why this error occurred

The script component is responsible for hoisting its content to the right position in the document, deduplicating identical scripts, and applying the framework's CSP defaults. An empty script element is always a bug — the browser will execute it as a no-op, the bundler will skip it, and the developer will be confused about why their analytics tag is missing. The framework raises this error to make the bug visible.

Common causes:

- A developer wrote `<Script src={maybe}>` and `maybe` is `undefined`.
- The script content is a JSX element (`<Script><span /></Script>`) that was meant to be wrapped in a string template.
- The `dangerouslySetInnerHTML` field is `undefined` because of a typo (`dangerouslySetInnerHtml` — capitalized `H` in `Html` is the React convention, but Swift-Rust accepts either).
- A linter or formatter stripped the script body during a refactor.

The error fires at the next dev rebuild, or at the start of `bun swift-rust build`.

## Reproducing the error

```tsx
import Script from "swift-rust/script";

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        {children}
        <Script />
      </body>
    </html>
  );
}
```

## Error output

```
error: [SR0079] invalid script
  --> app/layout.tsx:7:3
   |
 7 |         <Script />
   |         ^^^^^^^^^^
   |
   = a <Script> must have either a `src` URL or inline content
     (string children or `dangerouslySetInnerHTML`). got neither.
```

## How to fix it

1. Provide a `src` URL for an external script.

   ```tsx
   <Script src="https://example.com/analytics.js" />
   ```

2. Provide inline content as a string child. Curly braces with a template literal are the usual form.

   ```tsx
   <Script id="show-banner">
     {`document.getElementById('banner').classList.remove('hidden')`}
   </Script>
   ```

3. Provide inline content via `dangerouslySetInnerHTML`.

   ```tsx
   <Script
     id="show-banner"
     dangerouslySetInnerHTML={{
       __html: `document.getElementById('banner').classList.remove('hidden')`,
     }}
   />
   ```

## Diagnosis

```bash
bun swift-rust doctor
rg -n "<Script" app/
RUST_LOG=swift_rust=script=trace bun swift-rust build
```

## Common pitfalls

- A `Script` that is conditionally rendered (`{flag && <Script … />}`) is fine, but a `Script` that always renders with no content is the typical cause.
- Inline content must be a string, not a JSX element. The framework reads it as text; an element is serialized to `[object Object]`.
- The `id` prop is required when the script is inline, because the framework uses it to deduplicate identical inline scripts.
- This is the Swift-Rust analogue of Next.js's `invalid-script`. The fix is identical.
