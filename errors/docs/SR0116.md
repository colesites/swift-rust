# SR0116 — Unwanted Polyfill.io polyfill detected

The build pipeline detected a polyfill coming from `polyfill.io` (or a custom domain serving the same bundle) that duplicates functionality already shipped in Swift-Rust's baseline. The duplicate bytes inflate the JS payload sent to every visitor and slow down the first paint.

## Why this error occurred

Swift-Rust targets a modern browser baseline and inlines only the polyfills actually needed for that baseline. The framework scans every script that gets loaded and rejects any entry that overlaps with the built-in set, because shipping the same code twice is wasted bandwidth.

The check is purely advisory — it does not fail the build. The build still completes and the page still works, but you have measurable dead weight in the bundle.

## Reproducing the error

```tsx
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        {children}
        <script src="https://polyfill.io/v3/polyfill.min.js" />
      </body>
    </html>
  );
}
```

## Error output

```
[INFO] [SR0116] unwanted polyfill.io polyfill detected
  --> app/layout.tsx:9:14
    |
  9 |         <script src="https://polyfill.io/v3/polyfill.min.js" />
    |                 ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
    |                  polyfill: Array.prototype.flat
    |                  already provided by Swift-Rust baseline (see: docs/browsers)
    |
    help: remove the polyfill or drop features that Swift-Rust already covers.
```

## How to fix it

1. Remove the `<script src="https://polyfill.io/...">` tag.
2. If a polyfill is genuinely needed, host it yourself or pick a target feature that is not already in the baseline.
3. Re-run `bun swift-rust build` to confirm the warning is gone.

```tsx
import { Metadata } from "swift-rust/server";

export const metadata: Metadata = {
  title: "My app",
};

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust dev
```

The `trace` log prints the full list of polyfills considered so you can confirm which ones duplicate the baseline.

## Common pitfalls

- The detector flags any source from the `polyfill.io` domain or any custom domain that serves the canonical polyfill bundle. If you must self-host, use a custom URL that does not match the polyfill bundle signature.
- The check is `info` severity, so it does not block deploys on CI. Wire it into your review process instead.
- Some `polyfill.io` features are not in the Swift-Rust baseline (for example, very recent Web APIs). The detector only flags the overlap, not every polyfill.
