# SR0097 — Config file loading error

The framework's config evaluator threw an error while reading or evaluating `swift-rust.config.json` (or `.toml`/`.yaml`). The build cannot continue without a valid config, so the error is fatal.

## Why this error occurred

Swift-Rust's config is read by a small, restricted evaluator — not by a full Node.js process. This keeps the config fast to load, deterministic, and safe to evaluate in build workers. The trade-off is that you cannot freely use `require`, `import`, or arbitrary JS in the config.

Common causes of a load failure:

- A syntax error in the file (a missing comma in JSON, a misaligned `[[array]]` in TOML, a tab-vs-spaces problem in YAML).
- A `require()` or `import` of a module that is not available in the eval environment.
- Use of a feature outside the config subset (top-level `await`, dynamic import, `process.exit`, etc.).
- A function in the config (for example a `redirects` resolver) that threw at evaluation time.

## Reproducing the error

```json filename="swift-rust.config.json"
{
  "image": {
    "remotePatterns": [
      { "protocol": "https", "hostname": "cdn.example.com" }
    ]
  ,
  "experimental": { "foo": true }
}
```

Missing `}` after the `remotePatterns` array. The JSON parser fails.

## Error output

```
error: [SR0097] config file loading error: unexpected token at line 7
  --> swift-rust.config.json:5:3
   |
 5 |   ,
   |   ^
   |  parser expected `}` or `,` at this position
```

## How to fix it

1. Validate the file's syntax with a linter (e.g. `bunx jsonlint swift-rust.config.json`).
2. Resolve any module imports in the config before the build runs.
3. If the config is dynamic (e.g. routes resolved from a database), generate the static file at build time rather than evaluating it at load.

```json filename="swift-rust.config.json"
{
  "image": {
    "remotePatterns": [
      { "protocol": "https", "hostname": "cdn.example.com", "pathname": "/**" }
    ]
  },
  "experimental": {
    "largePageDataBytes": 200000
  }
}
```

## Diagnosis

```bash
bun swift-rust info
RUST_LOG=swift_rust=config=trace bun swift-rust build
```

`bun swift-rust info` prints the resolved config, with the location of the file and the error.

## Common pitfalls

- The config evaluator supports JSON, TOML, and YAML, but only one format per file. Pick the extension deliberately.
- Top-level functions in the config (e.g. `redirects: async () => [...]`) are supported, but they must be async. Sync functions returning a value are also supported.
- A failed config load surfaces the line and column from the parser. Cross-check the editor's squiggles against the reported location.
- Older blog posts reference `next.config.js`. The Swift-Rust equivalent is `swift-rust.config.json` (or `.toml`/`.yaml`); a stray `next.config.js` is ignored.
