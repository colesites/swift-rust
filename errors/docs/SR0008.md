# SR0008 — PDF error

The PDF component tree is compiled into a `printpdf` document at request time (when the route is hit) or at build time (for `pre-render: true` routes). Failures in the tree shape, page dimensions, or font references surface here.

## Why this error occurred

A PDF document has strict structural rules: it must have at least one page, every page must declare a size, and every text run must reference a font that is registered with the document. The component tree is validated before the bytes are written, so an invalid tree fails the render with a clear message rather than producing a corrupt PDF.

The PDF component is the only major component that is purely additive — it does not replace any HTML render — so most teams only hit it when they integrate invoice or report generation into an existing app.

## Reproducing the error

```tsx
import { Document, Text, View } from "swift-rust/pdf";

export default function Invoice() {
  return (
    <Document>
      <Text>Invoice</Text>
    </Document>
  );
}
```

The tree has a `<Text>` but no `<Page>`. The renderer refuses.

```tsx
<Page size="Letter">
  <View>
    <Text style={{ fontFamily: "Inter" }}>Hello</Text>
  </View>
</Page>
```

`Inter` is referenced as the font but the font system has not been asked to embed it.

```

  --> app/invoice.tsx:7:14
   |
 7 |     <Text style={{ fontFamily: "Inter" }}>Hello</Text>
   |              ^^^^^^^ referenced but not embedded via `swift-rust/font`
```

## How to fix it

1. Wrap content in a `<Page>` with a valid size:

```tsx
import { Document, Page, Text, View } from "swift-rust/pdf";

export default function Invoice() {
  return (
    <Document>
      <Page size="A4" orientation="portrait">
        <View>
          <Text>Invoice</Text>
        </View>
      </Page>
    </Document>
  );
}
```

2. Embed the font via the font system before referencing it:

```tsx
import { Inter } from "swift-rust/font/google";
const inter = Inter({ subsets: ["latin"] });

<Text style={{ fontFamily: inter.fontFamily }}>Hello</Text>
```

3. Re-run the route (or `bun swift-rust dev`) to regenerate the PDF.

## Diagnosis

```bash
bun swift-rust doctor
RUST_LOG=swift_rust::pdf=trace bun swift-rust dev
```

The trace log prints the full component tree and the validation step that failed. Use `--output` on the dev server to write the PDF to disk and inspect it with `pdftotext`.

## Common pitfalls

- Nested `<Page>` components are not supported. A page must be a direct child of `<Document>`.
- Page sizes are case-sensitive. `a4` is rejected; `A4` is accepted.
- The PDF component does not load remote fonts at render time. The font must be embedded at build time via `swift-rust/font`.
- A `<View>` outside a `<Page>` is silently dropped. Move it inside.
- The PDF renderer measures text in points (1pt = 1/72in). A 12pt font is the same as a 12px font in HTML.
