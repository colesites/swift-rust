# SR0114 — Synchronous script tag

A `<script>` tag was rendered without `async` or `defer`. Synchronous scripts block the parser: the browser stops building the DOM until the script is downloaded, parsed, and executed. The build flags the tag so the rendering cost is visible during code review.

## Why this error occurred

The HTML parser is single-threaded. A `<script src="...">` (no `async`, no `defer`) halts parsing while the script is fetched and run. For a third-party script that lives on a different origin, that is a guaranteed round-trip on the critical path. The user sees a blank page until the script finishes.

`async` and `defer` both fix this:

- `async` downloads the script in parallel with parsing and runs it as soon as it is ready. Order is not preserved.
- `defer` downloads the script in parallel and runs it after the document is parsed, in source order.

The framework's `<Script>` component picks the right one for you based on the `strategy` prop (`afterInteractive` defaults to `async`, `lazyOnload` to `defer`, etc.) and inlines the right tag. A literal `<script>` in JSX is a regression that the build catches.

## Reproducing the error

```tsx filename="app/page.tsx"
export default function Page() {
  return (
    <main>
      <h1>Welcome</h1>
      <script src="https://third-party.example.com/widget.js"></script>
    </main>
  )
}
```

The `<script>` has neither `async` nor `defer`.

## Error output

```
error: [SR0114] synchronous <script>: add `async` or `defer`
  --> app/page.tsx:4:7
   |
 4 |       <script src="https://third-party.example.com/widget.js"></script>
   |       ^^^^^^^ this script blocks the parser; use the <Script> component or add `async`/`defer`
```

## How to fix it

1. **Recommended.** Use the framework's `<Script>` component.

```tsx filename="app/page.tsx"
import Script from "swift-rust/script"

export default function Page() {
  return (
    <main>
      <h1>Welcome</h1>
      <Script src="https://third-party.example.com/widget.js" />
    </main>
  )
}
```

2. **If you must use a literal `<script>`.** Add `async` (independent scripts) or `defer` (scripts that depend on each other or that should run after parsing).

```tsx filename="app/page.tsx"
export default function Page() {
  return (
    <main>
      <h1>Welcome</h1>
      <script src="https://third-party.example.com/widget.js" async />
    </main>
  )
}
```

## Diagnosis

```bash
bun swift-rust doctor --check scripts
bun swift-rust info --third-party
```

`--third-party` lists every external script in your project and the strategy the framework has assigned to it.

## Common pitfalls

- Inline scripts (no `src`) are also synchronous, but they do not block on a network round-trip. The framework's lint does not flag them because the cost is small. If your inline script is large, move it to an external file and use `<Script>`.
- `type="module"` scripts are deferred by definition. The lint recognizes `type="module"` and does not flag the tag.
- A `<script>` inside a `<noscript>` is not a problem (it only runs when JS is disabled). The check is on the active document path.
- The framework's `<Script>` is a thin wrapper around a `<script>` tag with the right attributes; it does not add a runtime cost. The output bytes are essentially identical.
