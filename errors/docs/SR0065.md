# SR0065 — Invalid usage of `suspense` option in a dynamic import

The `suspense: true` option for dynamic imports tells the framework to use `React.lazy` plus the closest `<Suspense>` boundary's `fallback` for the loading state. This only works under a strict set of conditions; otherwise the framework rejects the call.

## Why this error occurred

There are three distinct invalid combinations, each with its own cause:

1. **`suspense: true` with React < 18.** Suspense-for-data was unstable before React 18. If the project is on React 17, the framework cannot wire the dynamic import into a Suspense boundary safely.
2. **`suspense: true` with `ssr: false`.** Suspense boundaries are always resolved on the server when React 18's streaming renderer is used. Disabling SSR for an import that is wrapped in Suspense is contradictory: there is nothing to stream. The framework raises the error rather than silently ignoring the `ssr: false`.
3. **`suspense: true` with `loading`.** When the import is wrapped in Suspense, the boundary's `fallback` is the loading state. A second `loading` would be a duplicate and is rejected.

In all three cases, the fix is to align the options with what Suspense actually means in React 18.

## Reproducing the error

```tsx
import { dynamic } from "swift-rust/dynamic";

const HeavyChart = dynamic(() => import("./HeavyChart"), {
  ssr: false,
  suspense: true,
  loading: () => <p>Loading…</p>,
});
```

## Error output

```
error: [SR0065] invalid usage of `suspense` in a dynamic import
  --> app/dashboard/page.tsx:4:1
   |
   = `suspense: true` cannot be combined with `ssr: false`.
   = `suspense: true` cannot be combined with a custom `loading` component.
   = hint: drop `ssr: false` and `loading`, and wrap the dynamic
           component in a `<Suspense fallback={…}>` boundary.
```

## How to fix it

1. Drop `ssr: false` and the `loading` prop, and wrap the import in a `<Suspense>` boundary.

   ```tsx
   import { Suspense } from "react";
   import { dynamic } from "swift-rust/dynamic";

   const HeavyChart = dynamic(() => import("./HeavyChart"), {
     suspense: true,
   });

   export default function Dashboard() {
     return (
       <Suspense fallback={<p>Loading…</p>}>
         <HeavyChart />
       </Suspense>
     );
   }
   ```

2. If you must keep `ssr: false`, use the WASM-mode import (formerly `next/dynamic({ ssr: false })`) — but be aware it ships as a client-only chunk and cannot participate in streaming.

   ```tsx
   const HeavyChart = dynamic(() => import("./HeavyChart"), {
     ssr: false,
   });
   ```

3. If the project is on React 17, upgrade to React 18 or remove `suspense: true`.

## Common pitfalls

- React 18 streams Suspense boundaries by default. If your hosting target does not support streaming (e.g. a legacy Node server without HTTP/2), the fallback will only appear after the full payload is received.
- `ssr: false` + `suspense: true` was a common copy-paste from older Next.js code. The combination was always ignored in Next.js as well; Swift-Rust now raises a hard error.
- The error fires at build time, not at runtime. Static analysis of the call site is what catches the bad combo.
