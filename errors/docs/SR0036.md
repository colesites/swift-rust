# SR0036 — target config is deprecated

The `target` field in `swift-rust.config.json` was removed in 0.1. The field used to switch the build between a Node server, a serverless bundle, and an experimental edge target. Swift-Rust now picks the right output automatically based on the `output` field.

## Why this error occurred

When Swift-Rust was still an alpha, the runtime had to make a build-time decision about what artifact to emit. The `target` field was the explicit knob. In 0.1 the framework started doing the right thing automatically based on the chosen `output` mode, so the `target` field is no longer consulted. Parsing it would mislead a reader into thinking it does something, so the field is rejected with a deprecation.

If you depended on the serverless target, the modern equivalent is to set `output: 'standalone'` (which produces a single binary) or `output: 'export'` (which produces a static site). Both work on any platform, including serverless hosts.

## Reproducing the error

```json
{
  "name": "site",
  "target": "serverless"
}
```

## Error output

```
[WARN] [SR0036] target config is deprecated: the `target` field has no effect
   --> swift-rust.config.json:2:2
   |
 2 |   "target": "serverless"
   |   ^^^^^^^^^^^^^^^^^^^^^^ remove this field. Use `output: "standalone"` or `output: "export"` instead.
   |
  help: see https://swift-rust.dev/docs/deploy for the modern deployment options.
```

## How to fix it

1. Remove the `target` field from `swift-rust.config.json`.
2. Decide between a long-running server (`output: "standalone"`) and a static export (`output: "export"`).
3. If you need serverless-style deployment, enable `outputFileTracing` to keep the bundle small.

```json
{
  "name": "site",
  "output": "standalone",
  "outputFileTracing": true
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log records every read of the deprecated field. In most cases the build just ignores it, but the warning makes sure the situation is visible.

## Common pitfalls

- Some templates set `target` together with `experimental-server-actions`. Both must be removed.
- A custom CI script may still pass `--target` on the command line. Remove that flag too.
- The `experimental-edge` target is no longer relevant — Swift-Rust does not ship an edge runtime.
- This deprecation will become a hard error in Swift-Rust 0.4.0.
