# SR0085 — No children passed to <Link>

A `<Link>` element must contain at least one child to render the visible label of the anchor. An empty link is rejected at build time.

## Why this error occurred

`<Link>` renders an `<a>` element with an `href` and a click handler. Without a child, the anchor has no accessible name, no visible text, and no click target. Screen readers announce it as "link" with no label, and keyboard users cannot focus an anchor that is empty.

The compiler enforces this so the issue is caught in CI rather than in front of users with assistive technology.

## Reproducing the error

```tsx filename="app/page.tsx"
import Link from "swift-rust/link"

export default function Home() {
  return (
    <>
      <Link href="/about" />
    </>
  )
}
```

## Error output

```
error: [SR0085] no children passed to <Link>: <Link href="/about" /> is empty
  --> app/page.tsx:5:6
   |
 5 |       <Link href="/about" />
   |       ^^^^^
```

## How to fix it

1. Always provide a child element or text node.
2. If you want a "skip to content" link, use a meaningful label and visually hide it with CSS.

```tsx filename="app/page.tsx"
import Link from "swift-rust/link"

export default function Home() {
  return (
    <Link href="/about">To About</Link>
  )
}
```

```tsx filename="app/skip-link.tsx"
import Link from "swift-rust/link"

export function SkipLink() {
  return (
    <Link href="#main" className="visually-hidden focus:not-sr-only">
      Skip to main content
    </Link>
  )
}
```

## Diagnosis

```bash
bun swift-rust doctor --check routing
bun swift-rust dev --strict
```

`--strict` runs accessibility checks on every page in development.

## Common pitfalls

- Icon-only links must still have an `aria-label` and visible text or a screen-reader-only label.
- Conditional children: `<Link>{show && "Back"}</Link>` will fail when `show` is `false`; default to a non-empty fallback.
- The compiler does not warn about whitespace-only children — `<Link>{" "}</Link>` is technically valid but useless.
