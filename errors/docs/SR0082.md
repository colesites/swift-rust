# SR0082 — Invalid "use server" value

Swift-Rust enforces the `"use server"` contract: every named export from a file with the directive must be an async function. Any other shape (a constant, a class, a non-async function) is rejected at compile time so it never reaches the client boundary.

## Why this error occurred

Server actions are RPC endpoints. The client receives a reference to the function and the framework must guarantee the wire format is a function call. Constants and objects cannot be invoked. A non-async function cannot be awaited, and a sync function would block the server worker. Because the client router has no way to represent "I called something that wasn't a function", the framework rejects these shapes during the build's module analysis pass.

This is checked statically against the source, not at runtime, so the error fires before the first request.

## Reproducing the error

```tsx filename="app/actions/post.ts"
"use server"

export const MAX_TITLE_LEN = 120

export function publishPost(title: string, body: string) {
  return db.posts.create({ title, body })
}
```

`MAX_TITLE_LEN` is a constant, and `publishPost` is not `async`. The build will refuse to compile this file as a server module.

## How to fix it

1. Make every exported function in the module `async`.
2. Move non-function exports (constants, types) to a separate file without the directive.
3. If you must share a module, prefer named re-exports of async functions over `export *`.

```tsx filename="app/actions/post.ts"
"use server"

import { db } from "@/lib/db"

export async function publishPost(title: string, body: string) {
  return db.posts.create({ title, body })
}
```

```ts filename="app/actions/constants.ts"
export const MAX_TITLE_LEN = 120
```

## Diagnosis

```bash
bun swift-rust info
bun swift-rust doctor --check server-actions
```

`--check server-actions` walks `app/` and validates every `"use server"` file in isolation.

## Common pitfalls

- Constants and enums sneaking in via `export *`.
- Anonymous arrow functions assigned to `const` — they are functions, but a missing `async` still trips the rule.
- Type-only exports (`export type { Foo }`) are stripped and are fine; value exports are not.
