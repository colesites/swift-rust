# SR0134 — React version below the minimum

Swift-Rust requires `react@^19.0.0` and `react-dom@^19.0.0` at the version resolver level. An older React can compile and even render, but the framework disables several features and prints this error so you know the version is unsupported.

## Why this error occurred

The framework uses a number of React 19 features that do not exist in older releases: the `use()` hook for unwrapping Promises in client components, the new form actions, the new error boundary semantics, and the new compiler hints. Backporting those features to React 18 is not done; instead, the framework simply refuses to run with an older React and prints this message.

The error fires the first time the build or dev server tries to import React. The build fails immediately, before any work is done, so the message is short and points to `package.json`.

## Reproducing the error

```json
{
  "dependencies": {
    "react": "^17.0.2",
    "react-dom": "^17.0.2"
  }
}
```

## Error output

```
[ERROR] [SR0134] react version below the minimum
  --> package.json:3:5
    |
  3 |     "react": "^17.0.2",
    |               ^^^^^^^^^
    | Swift-Rust requires `react@^19.0.0` and `react-dom@^19.0.0`.
    |
    help: run `bun add react@latest react-dom@latest` to upgrade.
```

## How to fix it

1. Upgrade both packages:

```bash
bun add react@latest react-dom@latest
```

2. Verify the resolved version:

```bash
bun pm ls react react-dom
```

3. Remove any `overrides` or `resolutions` block that pins React to an older version.

4. Re-run the build.

## Diagnosis

```bash
bun swift-rust doctor
bun pm ls react react-dom
RUST_LOG=swift_rust=react=trace bun swift-rust dev
```

`bun pm ls` prints the resolved version of every React-related package in the dep tree, which is the fastest way to find a stale override.

## Common pitfalls

- A single workspace in a monorepo can pin an old React. Make sure all workspaces share the same major version.
- `bun install` does not automatically migrate `^17` to `^19` because the major changed. Explicit upgrade required.
- Some component libraries (e.g. older versions of `react-hook-form`) declare `peerDependencies` that conflict. Resolve the peer conflict first.
- A lockfile pinned to React 17 will keep installing the old version even after `bun add react@latest`. Delete the lockfile entry and reinstall.
