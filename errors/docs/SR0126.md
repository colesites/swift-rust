# SR0126 — Invalid PostCSS configuration shape

The PostCSS config file is in a shape that Swift-Rust cannot parse. The framework expects a flat array of plugin entries, where each entry is either a string (plugin name) or a 2-tuple `[pluginName, options]`. Anything else — `require('plugin')(...)`, a single-element array, a boolean — is rejected.

## Why this error occurred

The Swift-Rust bundler evaluates `postcss.config.js` in a worker thread with a stripped-down `require`. The expected shape is intentionally limited so the config can be statically analyzed for caching. Any plugin that tries to evaluate side effects at config time (for example, calling a factory function returned by `require`) cannot be supported.

The framework prints a warning and falls back to the built-in default plugin set. The build still succeeds, but your custom plugins are not applied.

## Reproducing the error

```js
// postcss.config.js
const pluginA = require("postcss-plugin-a");
module.exports = {
  plugins: [require("postcss-plugin")({ optionA: "..." }), pluginA],
};
```

```js
// postcss.config.js
module.exports = {
  plugins: [
    ["postcss-plugin-1", { optionA: "..." }],
    ["postcss-plugin-2"],
  ],
};
```

## Error output

```
[WARN] [SR0126] invalid postcss configuration shape
  --> postcss.config.js:3:14
    |
  3 |   plugins: [require("postcss-plugin")({ optionA: "..." }), pluginA],
    |              ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
    | plugin entries must be strings or [name, options] tuples; got a function value.
    |
    help: replace the call with the tuple form `["postcss-plugin", { optionA: "..." }]`.
```

## How to fix it

1. Convert every plugin entry to the tuple form. The first element is the plugin name; the second is the options object. The second element can be omitted if the plugin takes no options.

```js
// postcss.config.js
module.exports = {
  plugins: [
    "postcss-plugin",
    [
      "postcss-plugin-with-config",
      {
        optionA: "...",
      },
    ],
    [
      "postcss-plugin-toggled",
      process.env.NODE_ENV === "production" ? { optionA: "..." } : false,
    ],
    ["postcss-plugin-boolean", true],
  ],
};
```

2. Drop the `require()` of the plugin module from the config — the loader resolves the plugin by name from `package.json` and your local `node_modules`.

3. Verify by running `bun swift-rust info` and inspecting the postcss section.

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=css=debug bun swift-rust dev
```

`css=debug` prints the parsed plugin list with the source of each entry.

## Common pitfalls

- A single-element array `["plugin-name"]` is treated as a misconfigured tuple. Use the string form `"plugin-name"` instead.
- A `false` entry disables the plugin; a `true` entry enables it with no options. Both are valid; anything else (an empty object, a number) is not.
- `require.resolve` of the plugin package from the config file is not supported. The loader resolves plugins itself.
- The expected shape mirrors what `postcss-cli` accepts, so most starter configs work without changes.
