# SR0041 — env loading disabled

Swift-Rust did not load the project's `.env` files into `process.env`. The build continues, but a warning is printed because this is usually a sign of a stale `dotenv` install or a misconfigured project root.

## Why this error occurred

Swift-Rust loads `.env`, `.env.local`, and friends automatically using the same precedence rules as Next.js. To avoid clobbering a user-installed `dotenv` (which can be configured with arbitrary expansion, custom paths, etc.), the framework opts out of the built-in loader when it sees a `dotenv` package in the dependency tree. The opt-out is the safe default because two loaders reading the same files in different orders is a common source of bugs.

A second trigger is a missing `package.json` in the project root. Without one, the framework cannot be sure the project is in a state where env loading makes sense, so it disables the loader as a precaution.

The warning is non-fatal: env loading is disabled but the rest of the build works.

## Reproducing the error

```json
{
  "name": "site",
  "dependencies": {
    "dotenv": "^16.0.0"
  }
}
```

## Error output

```
[WARN] [SR0041] env loading disabled: a `dotenv` dependency was detected
   --> package.json:3:5
   |
 3 |     "dotenv": "^16.0.0"
   |     ^^^^^^^^^^^^^^^^^^^^^ remove this package to let Swift-Rust load .env files automatically
   |
  help: Swift-Rust's built-in loader handles .env, .env.local, etc. with proper precedence.
```

## How to fix it

1. Remove `dotenv` (and `dotenv-expand`, if present) from `package.json`.
2. Reinstall to refresh the lockfile.
3. Confirm a `package.json` exists at the project root (the framework refuses to load env if it cannot find one).

```bash
bun remove dotenv
bun install
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust dev
```

The trace log lists the `.env` files it considered, the order it would load them in, and the reason loading was skipped (a `dotenv` dep, a missing `package.json`, or an explicit `env.disableLoading: true`).

## Common pitfalls

- A transitive dependency that pulls in `dotenv` will also trigger this. Use `bun why dotenv` to find the chain.
- A custom `dotenv` config (e.g. `dotenv.config({ path: ".env.production" })`) is no longer applied. If you relied on a non-default path, set `env.files` in `swift-rust.config.json`.
- CI environments typically do not have a `.env` file at all, so this warning is harmless in CI; it is more meaningful in local dev.
- The warning can be promoted to a hard error by setting `env.failOnDisable: true` in the config.
