# SR0091 — Missing root layout tags

The root layout in `app/layout.tsx` did not return the required document shell. Every route in the app needs `<html>` and `<body>` from the root layout, and the framework refuses to build a route tree without them.

## Why this error occurred

In Swift-Rust (and Next.js App Router before it), the root layout is responsible for the entire HTML document, not just the visible page. The bundler streams the document directly to the browser, and there must be exactly one `<html>` and one `<body>` for that document to be valid. If the root layout returns a fragment or a bare `{children}`, the framework cannot produce a well-formed document and aborts the build.

This check is enforced at compile time. The bundler inspects `app/layout.tsx` and verifies that the JSX it returns contains an `<html>` element at the top level and a `<body>` element inside it.

## Reproducing the error

```tsx filename="app/layout.tsx"
export default function Layout({ children }: { children: React.ReactNode }) {
  return children
}
```

## Error output

```
error: [SR0091] missing root layout tags: <html> and <body> must be present in app/layout.tsx
  --> app/layout.tsx:2:9
   |
 2 |   return children
   |          ^^^^^^^^ expected <html><body>{children}</body></html>
```

## How to fix it

1. Wrap `{children}` in `<html>` and `<body>` at the top level of the returned JSX.
2. Add language, theme, and any required attributes to `<html>`.
3. Put providers (theme, query client, context) inside `<body>`, not above `<html>`.

```tsx filename="app/layout.tsx"
import type { ReactNode } from "react"

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="en">
      <body>
        {children}
      </body>
    </html>
  )
}
```

```tsx filename="app/layout.tsx"
import type { ReactNode } from "react"
import { ThemeProvider } from "@/components/theme-provider"

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="en" data-theme="light">
      <body>
        <ThemeProvider>{children}</ThemeProvider>
      </body>
    </html>
  )
}
```

## Diagnosis

```bash
bun swift-rust doctor --check root-layout
```

`--check root-layout` validates that `app/layout.tsx` exports a default function that returns the required document shell.

## Common pitfalls

- Conditional rendering around `<html>` (`{cond && <html>...}`) will not satisfy the check unless `cond` is statically `true`.
- The `metadata` export and the `viewport` export must be declared in the same file as the root layout. If you split them out, the metadata is dropped.
- A layout deeper in the tree (e.g. `app/dashboard/layout.tsx`) is allowed to omit `<html>` and `<body>` — only the root is checked.
