# SR0128 — Prerender error

The build pipeline tried to prerender a route to static HTML and the render threw. The build fails with this error and the offending route is named in the message. The prerender runs in the same Rust server runtime that powers production, so the failure usually points at a real bug that would also break at request time.

## Why this error occurred

During `bun swift-rust build`, the framework walks the route table, evaluates the server component tree for each route, and writes the resulting HTML to the build output. Any throw inside the tree — a missing prop, an unhandled rejection, a `window is not defined` — aborts the prerender.

Common root causes:

- A component reads `localStorage` or `window` directly. These exist only in the browser. Either move the call into a `useEffect` or mark the component as a client component.
- A server component fetches data that the prerender does not provide (for example, headers that only exist at request time). Add a guard.
- A page colocates non-route files next to a route file, and the router tries to interpret one of them as a route. The error names the offending path.
- A data dependency throws, propagating up through the server component.

## Reproducing the error

```tsx
"use client";

import { useEffect, useState } from "react";

export default function ClientOnly() {
  const [v, setV] = useState<string | null>(null);
  useEffect(() => {
    setV(window.localStorage.getItem("k") ?? "");
  }, []);
  return <p>{v}</p>;
}
```

```tsx
import { ClientOnly } from "./client-only";

export default function Page() {
  return (
    <main>
      <h1>Dashboard</h1>
      <ClientOnly />
    </main>
  );
}
```

When the route is prerendered, the framework evaluates the server tree, encounters `<ClientOnly>` and tries to render it. Without a `typeof window` guard, the prerender throws.

## Error output

```
[ERROR] [SR0128] prerender error
  --> app/dashboard/page.tsx:1:1
    |
    | route: /dashboard
    | cause: ReferenceError: window is not defined
    |   at ClientOnly (app/dashboard/client-only.tsx:6:10)
    |
    help: guard browser-only APIs with a `useEffect`, or mark the file as a client component with `'use client'`.
```

## How to fix it

1. Mark the file as a client component so it is not prerendered:

```tsx
"use client";

import { useEffect, useState } from "react";

export default function ClientOnly() {
  const [v, setV] = useState<string | null>(null);
  useEffect(() => {
    setV(window.localStorage.getItem("k") ?? "");
  }, []);
  return <p>{v}</p>;
}
```

2. Or guard the browser API:

```tsx
import { useEffect, useState } from "react";

export default function Safe() {
  const [v, setV] = useState<string>("");
  useEffect(() => {
    setV(window.localStorage.getItem("k") ?? "");
  }, []);
  return <p>{v}</p>;
}
```

3. Re-run the build. If the error is still opaque, run with `--debug-prerender` for an unminified stack trace.

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust build --debug-prerender
RUST_LOG=swift_rust=render=trace bun swift-rust build
```

`--debug-prerender` keeps the source map attached to the prerender error so the line number points to the original file rather than the minified output.

## Common pitfalls

- A "use client" file that does not export a default React component still throws during prerender if a server component imports it directly. Always export a component.
- `Date()` and `Math.random()` produce different output on the server and the client and can cause hydration errors (SR0133) even when prerender succeeds.
- A `Promise` that never resolves blocks the prerender. Add a timeout.
- The error names the *first* route that throws. The real bug may be in a shared component imported by many routes.
