# SR0050 â€” `generateBuildId` did not return a string

`generateBuildId` is an opt-in hook in the project config that lets you stamp each build with a stable identifier â€” typically a git SHA, a CI run number, or a content hash. Swift-Rust reads the value synchronously, writes it into a `BUILD_ID` file, and ships it to the browser so that the client can detect when a new build is live and prompt the user to reload.

## Why this error occurred

The contract is simple: the function (sync or async) must return a string. Any other return type â€” `undefined` from a missing `return`, a numeric CI build number coerced into the wrong type, a `Promise<undefined>` â€” fails this contract. Swift-Rust calls the function once per build and validates the result with a strict type check.

The error most often appears when:

- A user copies a Next.js snippet that returns `process.env.BUILD_ID` but the variable is unset, so the value is the string `"undefined"`.
- The function awaits a remote service that occasionally times out, resolving to `undefined`.
- A misconfigured `return` statement falls through because it sits after a conditional branch.

## Reproducing the error

```ts
// swift-rust.config.ts
import type { SwiftRustConfig } from "swift-rust";

const config: SwiftRustConfig = {
  generateBuildId: async () => {
    const sha = process.env.GIT_SHA;
    if (!sha) return;
    return sha;
  },
};

export default config;
```

## How to fix it

1. Ensure the function always returns a string. Provide a sensible fallback.

   ```ts
   import type { SwiftRustConfig } from "swift-rust";

   const config: SwiftRustConfig = {
     generateBuildId: async () => {
       const sha = process.env.GIT_SHA;
       return sha ?? `dev-${Date.now()}`;
     },
   };

   export default config;
   ```

2. If you use TypeScript, annotate the return type so the compiler catches the mistake.

   ```ts
   const config: SwiftRustConfig = {
     generateBuildId: async (): Promise<string> => {
       return process.env.GIT_SHA ?? "local";
     },
   };
   ```

## Common pitfalls

- `process.env.GIT_SHA` is `undefined` in local dev unless your shell exports it; an unconditional `return process.env.GIT_SHA` will pass `undefined` along.
- The function is awaited even if it is declared `async`, so a `Promise<undefined>` is treated the same as `undefined`. Always return a value.
- Swift-Rust does not coerce numbers to strings. Cast explicitly with `String(n)` or `` `${n}` ``.
- The build ID is exposed at `/_swift-rust/static/BUILD_ID` at runtime â€” curl that to verify what was actually stamped.
