# SR0086 — passHref on <Link> is deprecated

The `passHref` prop on `<Link>` is no longer required and emits a deprecation warning. It will be removed in a future major version of Swift-Rust.

## Why this error occurred

`passHref` existed in Next.js 12 because `<Link>` did not always render an `<a>`. Wrapping a custom component required telling `<Link>` to forward the `href` prop manually. In Swift-Rust (and in Next.js 13+), `<Link>` always renders the `<a>`, so the prop is redundant. Passing it now is harmless but noisy — the deprecation surfaces the migration so the prop can be removed cleanly.

## Reproducing the error

```tsx filename="app/nav-link.tsx"
import Link from "swift-rust/link"
import styled from "styled-components"

const StyledLink = styled.a`
  color: red;
`

export function NavLink({ href, name }: { href: string; name: string }) {
  return (
    <Link href={href} passHref>
      <StyledLink>{name}</StyledLink>
    </Link>
  )
}
```

## Error output

```
warn: [SR0086] passHref on <Link> is deprecated and has no effect
  --> app/nav-link.tsx:9:21
   |
 9 |     <Link href={href} passHref>
   |                     ^^^^^^^^
```

## How to fix it

1. Remove `passHref` from `<Link>`.
2. Make sure the styled component (or custom wrapper) accepts `href` as a prop.

```tsx filename="app/nav-link.tsx"
import Link from "swift-rust/link"
import styled from "styled-components"

const StyledLink = styled.a`
  color: red;
`

export function NavLink({ href, name }: { href: string; name: string }) {
  return (
    <Link href={href}>
      <StyledLink>{name}</StyledLink>
    </Link>
  )
}
```

## Diagnosis

```bash
bun swift-rust doctor --check deprecations
```

`--check deprecations` scans the project for known deprecated APIs and prints a migration table.

## Common pitfalls

- A codemod from a Next.js 12 codebase may have left `passHref` behind. Run the Swift-Rust upgrade codemod to clean up.
- The `legacyBehavior` prop is also deprecated and is reported as a separate warning.
- Some older third-party UI kits still pass `passHref`. Disable the warning for a specific line with `// swift-rust-disable-next-line SR0086`.
