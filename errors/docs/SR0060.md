# SR0060 — Install `sass` to use built-in Sass support

Swift-Rust has first-class support for Sass out of the box — the bundler recognizes `.scss` and `.sass` files and compiles them through a Dart-driven Sass runtime. Unlike Babel, the framework does not bundle a Sass compiler to keep the install size small. You bring your own.

## Why this error occurred

The moment the bundler sees an `import './styles.scss'` or a CSS Modules import of a `.scss` file, it looks for the `sass` package in `node_modules`. If it is missing, the dev server falls back to a stub that emits this `info` message and treats the file as plain CSS (so the variables and mixins are passed through to the browser as-is, where they fail to parse).

The most common reasons the package is missing:

- A team member added `.scss` files to a new project that was scaffolded without the Sass dep.
- The dep was installed with `bun install --production` in CI, which strips devDependencies.
- The dep was uninstalled as part of a "trimming" pass that did not realize the framework was relying on it.
- The project was migrated from `pages/` to `app/` and the dependency was lost in the move.

This is `info`-level because the build does succeed — the CSS is just not compiled.

## Reproducing the error

```bash
bun add -D sass
```

```bash
bun swift-rust dev
```

```tsx
import styles from "./Button.module.scss";

export function Button({ children }: { children: React.ReactNode }) {
  return <button className={styles.root}>{children}</button>;
}
```

## Error output

```
info: [SR0060] install `sass` to use built-in Sass support
  --> app/components/Button.module.scss
   |
   = the bundler could not find the `sass` package. .scss files will
     be passed through to the browser as-is. run `bun add -D sass` to
     enable compilation.
```

## How to fix it

1. Install `sass` as a dev dependency.

   ```bash
   bun add -D sass
   ```

2. If you previously used `node-sass` (the deprecated C++ port), uninstall it and install the Dart version. Swift-Rust does not support `node-sass`.

   ```bash
   bun remove node-sass
   bun add -D sass
   ```

3. If you want to keep the dep optional, gate the imports behind a CSS Modules toggle in the project config.

   ```json
   {
     "style": {
       "sass": { "enabled": false }
     }
   }
   ```

## Diagnosis

```bash
bun swift-rust doctor
ls node_modules/sass/package.json 2>/dev/null || echo "sass is not installed"
RUST_LOG=swift_rust=style=trace bun swift-rust dev
```

## Common pitfalls

- The Dart-based `sass` package is the only supported compiler. The legacy `node-sass` package is not wired up.
- `sass-embedded` (the new IPC-based runtime) is supported starting with Swift-Rust 0.2. Use it if you need faster recompiles on large stylesheets.

  ```bash
  bun add -D sass-embedded
  ```

- The `info` severity means the build does not fail. The CSS is just broken. If you are seeing visual regressions, check the network tab for unprocessed `.scss` responses.
- In CI, `bun install --production` strips devDependencies. Make sure your build job uses the full install.
