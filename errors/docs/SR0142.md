# SR0142 — Sharp version does not support AVIF

The installed `sharp` is older than 0.27.0 (December 2020) and the image pipeline was asked to encode AVIF. Sharp added AVIF support in 0.27.0; older versions do not export the encoder and the pipeline errors out.

## Why this error occurred

Swift-Rust's default image config lists AVIF as one of the supported output formats. When `<Image>` is asked to produce an AVIF variant (because the request's `Accept` header advertises it, or because the config hard-codes it), the pipeline calls into `sharp`. If `sharp` does not have an AVIF encoder, the call throws.

The error fires at request time, not at startup, because the pipeline only tries to encode AVIF when a request actually asks for it. A site that does not serve AVIF will not see the error.

## Reproducing the error

Request any optimized image with `Accept: image/avif` in the headers (or visit the site in a browser that advertises AVIF, like a recent Chrome / Firefox).

```bash
curl -H "Accept: image/avif" http://localhost:3000/_swift-rust/image?url=%2Fhero.jpg&w=1080
```

```log
[ERROR] [SR0142] sharp version does not support AVIF
  --> .
    |
    | installed sharp: 0.26.0
    | required: >= 0.27.0
    |
    help: run `bun add sharp@latest`, or disable AVIF in `swift-rust.config.json`.
```

## How to fix it

1. Upgrade `sharp`:

```bash
bun add sharp@latest
```

2. If `SHARP_PATH` points to a vendored copy, upgrade that copy:

```bash
cd "$SHARP_PATH/../" && bun add sharp@latest
```

3. If you cannot upgrade `sharp`, disable AVIF in the image config:

```json
{
  "image": {
    "formats": ["image/webp"]
  }
}
```

4. Re-run the build and restart the server.

## Diagnosis

```bash
bun swift-rust doctor
bun pm ls sharp
```

`bun pm ls sharp` shows the resolved version. Doctor also reports the supported formats.

## Common pitfalls

- The `sharp` package has separate platform-specific binaries. Upgrading the JS wrapper does not always upgrade the binary. Force a reinstall with `bun pm trust sharp` or by removing the platform package from `node_modules`.
- A lockfile pinned to `0.26.x` will keep the old version even after `bun add sharp@latest`. Remove the lockfile entry and reinstall.
- Disabling AVIF is a legitimate choice for performance — many browsers do not request it because the decode cost outweighs the size win.
- The error fires on the first AVIF request. The first webp request still succeeds. The two are not coupled.
