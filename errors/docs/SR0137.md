# SR0137 â€” Reserved port in use

The server was started on a port that browsers refuse to connect to. Swift-Rust detects the port against the WHATWG fetch "port blocking" list and aborts startup to prevent a confusing experience where the server runs fine but no client can reach it.

## Why this error occurred

The WHATWG fetch specification maintains a list of ports that browsers treat as "unsafe" for non-secure contexts. If a dev server listens on one of those ports, the browser will silently refuse to make a request to it. The server itself is not at fault, but the user experience is broken.

The reserved ports include `1`, `7`, `9`, `11`, `13`, `15`, `17`, `19`, `20`, `21`, `22`, `23`, `25`, `37`, `42`, `43`, `53`, `69`, `77`, `79`, `87`, `95`, `101`, `102`, `103`, `104`, `109`, `110`, `111`, `113`, `115`, `117`, `119`, `123`, `135`, `137`, `139`, `143`, `161`, `179`, `389`, `427`, `465`, `512`, `513`, `514`, `515`, `526`, `530`, `531`, `532`, `540`, `548`, `554`, `556`, `563`, `587`, `601`, `636`, `989`, `990`, `993`, `995`, `1719`, `1720`, `1723`, `2049`, `3659`, `4045`, `4190`, `5060`, `5061`, `6000`, `6566`, `6665`, `6666`, `6667`, `6668`, `6669`, `6679`, `6697`, `10080`.

## Reproducing the error

```bash
bun swift-rust dev --port 4045
```

## Error output

```
[ERROR] [SR0137] reserved port: 4045
  --> .
    |
    | port 4045 is on the WHATWG fetch port-blocking list.
    | browsers will refuse to connect to http://localhost:4045 from a non-secure context.
    |
    help: pick a different port (e.g. 3000, 5173, 8080) or run behind a reverse proxy.
```

## How to fix it

1. Pick a non-reserved port:

```bash
bun swift-rust dev --port 3000
```

2. If `process.env.PORT` is set by your CI, override it:

```bash
PORT=3000 bun swift-rust start
```

3. If you must serve on a reserved port, terminate TLS and run behind a reverse proxy on a non-reserved port. The browser will then connect to the proxy on the safe port and the proxy will forward to the internal reserved port.

## Diagnosis

```bash
bun swift-rust doctor
```

The doctor output lists the port Swift-Rust is configured to use and flags reserved values.

## Common pitfalls

- The list is the same one Chromium, Firefox, and Safari use; there is no override.
- A reserved port works in `curl` and from server-to-server requests, which is why the check is a Swift-Rust-side guardrail rather than a runtime detection.
- Some PaaS platforms inject `PORT=80` or `PORT=443`. These are not in the reserved list, so the check does not fire.
- A custom `PORT` value of `0` asks the OS for an ephemeral port; the check accepts it.
