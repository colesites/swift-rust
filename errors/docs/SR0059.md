# SR0059 — Incompatible `href` and `as` values

The `as` prop on `<Link>` (and the second argument to `router.push` / `router.replace`) lets you override the path that the browser actually navigates to, while keeping the `href` as the route identifier used to look up the page in the route table. The two must agree: `as` is the materialized form of `href`, with one concrete value per dynamic segment.

## Why this error occurred

The runtime reconstructs the URL by substituting the values in `as` back into the `href` pattern. If the substitution does not produce a valid instance of the pattern — because `as` has the wrong number of segments, or the values are the wrong shape — the navigation is rejected.

The most common mistakes:

- Using `href="/[post]"` with `as="/post-1/comments"`. The pattern only expects one segment, but `as` provides two.
- Using `href="/blog"` with `as="https://example.com/blog"`. The pattern has no placeholders, so the external URL is not a valid materialization.
- Mixing strings: `href="/users/[id]"` with `as="/users/abc/?tab=profile"`. The trailing `?tab=profile` is parsed as a query string, not a path segment, and leaves `[id]` without a value.

The error fires on click, not on render, mirroring the behaviour of `href-interpolation-failed` (SR0055).

## Reproducing the error

```tsx
import Link from "swift-rust/link";

export default function Page() {
  return (
    <Link href="/[post]" as="/post-1/comments">
      Read more
    </Link>
  );
}
```

## Error output

```
[ERROR] [SR0059] incompatible href and as values
  --> app/page.tsx:4:3
   |
   = href "/[post]" expects one segment. as "/post-1/comments" has two.
   = the navigation cannot be reconstructed. drop `as` or align the
     number of segments.
```

## How to fix it

1. Align `as` with the segments expected by `href`.

   ```tsx
   <Link href="/[post]" as="/post-1">
     Read more
   </Link>
   ```

2. For external URLs, drop `as` and pass the absolute URL as `href`.

   ```tsx
   <Link href="https://example.com/blog">Read more</Link>
   ```

3. If you need to navigate to a deep URL that has its own dynamic segments, build the `href` object instead of using `as`.

   ```tsx
   <Link
     href={{
       pathname: "/blog/[post]/[comment]",
       query: { post: "post-1", comment: "comment-1" },
     }}
   >
     Open
   </Link>
   ```

## Common pitfalls

- `as` is sometimes used as a way to inject a query string. The correct way is to pass `query` alongside `pathname` in the `href` object, or to add a `?…` to the `as` value (which Swift-Rust parses as a search string).
- The error fires on click, not on render. If you have a list of links generated in a loop, the warning may only surface when the first bad link is clicked.
- `as` is rarely needed in `app/`. The framework can almost always reconstruct the correct URL from the `href` alone.
