# SR0040 â€” env key not allowed

An entry in the `env` block of `swift-rust.config.json` collides with an internal variable the framework controls. Swift-Rust refuses to let user-defined env entries shadow or override the internals.

## Why this error occurred

Swift-Rust's compiler and runtime rely on a small set of well-known environment variables (`SWIFT_RUST_RUNTIME`, `__SWIFT_RUST_*`, anything starting with `NODE_`, etc.) to coordinate build and runtime behavior. Letting user config override those values would mean the framework cannot trust its own environment, which would be a security and correctness issue.

The validation is strict: any key starting with `__` or `NODE_` is rejected, and a small set of reserved names are also blocked.

## Reproducing the error

```json
{
  "name": "site",
  "env": {
    "__SECRET": "value",
    "NODE_ENV": "production"
  }
}
```

## Error output

```
[ERROR] [SR0040] env key not allowed: "__SECRET"
   --> swift-rust.config.json:3:5
   |
 3 |     "__SECRET": "value"
   |     ^^^^^^^^^^ keys starting with `__` are reserved
   |
  help: rename to e.g. `APP_SECRET`. Use `process.env` in server code for per-deploy overrides.
```

## How to fix it

1. Rename the key so it does not start with `__` or `NODE_`.
2. If you need a per-deploy override, set it in the deployment environment (e.g. via your platform's env var UI) and read it with `process.env.MY_KEY` from a server module.
3. Re-run the build and confirm the error is gone.

```json
{
  "name": "site",
  "env": {
    "APP_SECRET": "value"
  }
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log lists every env key it saw and which ones were rejected, so you can find each collision without re-reading the file by hand.

## Common pitfalls

- A `NODE_ENV` override in `env` is silently ignored by Node, and the framework now refuses it explicitly. Set the real `NODE_ENV` in your shell or platform config.
- `SWIFT_RUST_RUNTIME` is the runtime identifier (e.g. `node`, `bun`, `workerd`). It is set by Swift-Rust itself and cannot be overridden from config.
- The list of reserved prefixes is documented in the config reference. If you find a name that should be reserved but is not, file an issue.
- Use `process.env` for any secret that should not be inlined into the build output.
