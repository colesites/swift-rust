# SR0125 — PostCSS plugin was ignored

Your `postcss.config.js` includes a plugin that Swift-Rust already applies automatically. The framework cannot run the same plugin twice, so the duplicate entry is silently dropped. The warning is raised so you can clean up the config.

## Why this error occurred

Swift-Rust's CSS pipeline bakes in a number of plugins:

- `postcss-modules-values`
- `postcss-modules-scope`
- `postcss-modules-extract-imports`
- `postcss-modules-local-by-default`
- `postcss-modules`

These are part of the built-in CSS Modules support. If your config also lists them, the framework dedupes and the user-authored entry is the one that gets dropped. The visible behaviour is usually correct (CSS Modules work), but the config is misleading.

The same logic applies to a few other plugins that are always part of the Swift-Rust pipeline (for example, the autoprefixer-equivalent in `lightningcss`).

## Reproducing the error

```js
// postcss.config.js
module.exports = {
  plugins: ["postcss-modules-values", "tailwindcss", "autoprefixer"],
};
```

## Error output

```
[WARN] [SR0125] postcss plugin ignored
  --> postcss.config.js:2:13
    |
  2 |   plugins: ["postcss-modules-values", "tailwindcss", "autoprefixer"],
    |             ^^^^^^^^^^^^^^^^^^^^^^^^
    | `postcss-modules-values` is applied automatically by Swift-Rust and cannot be added twice.
    |
    help: remove the entry, or configure CSS Modules through `swift-rust.config.json` -> `css.modules`.
```

## How to fix it

1. Remove the redundant entry from `postcss.config.js`:

```js
// postcss.config.js
module.exports = {
  plugins: ["tailwindcss"],
};
```

2. If you need to customize CSS Modules behaviour, use the framework config:

```json
{
  "css": {
    "modules": {
      "localsConvention": "camelCaseOnly",
      "generateScopedName": "[name]__[local]___[hash:base64:5]"
    }
  }
}
```

3. Restart the dev server and verify the CSS is still being scoped correctly.

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=css=debug bun swift-rust dev
```

`css=debug` prints the final list of active plugins and which ones came from the framework, the user config, or both.

## Common pitfalls

- The list of auto-applied plugins is internal. When in doubt, the warning text names the exact plugin and the file it was found in.
- A `tailwindcss` entry is fine — Tailwind is not built in.
- Some legacy templates bundle the entire `postcss-modules` set. You can drop the whole `postcss.config.js` if CSS Modules is the only thing it configured.
- Removing the entry does not change runtime behaviour because the framework still applies the plugin; it only removes the noise from the warning.
