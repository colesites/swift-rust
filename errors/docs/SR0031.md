# SR0031 — CSS Modules imported by a dependency

A package in `node_modules` imports a CSS Module from its own source. This usually means the consumer is pulling in the package's un-built source files (e.g. from a monorepo or a git install) instead of the published distribution. The build warns because the resulting behavior depends on the consumer's bundler.

## Why this error occurred

A well-behaved dependency ships a `dist/` with the CSS already extracted, or it expects the consumer to import a specific CSS file from a documented path. When the consumer instead pulls the source `*.module.css` directly, the bundler has to make decisions on the package's behalf: which file is the entry, how to hash class names, where to inject the styles. None of those decisions are the consumer's to make, so the build warns.

In a properly published package, the CSS Modules are inlined into the package's own compiled JS (with class names already mangled) and the consumer never sees the source.

## Reproducing the error

```bash
node_modules/some-pkg/src/button.module.css
```

```ts
import "./button.module.css";
```

## Error output

```
[WARN] [SR0031] CSS Modules imported by a dependency: some-pkg
   file: node_modules/some-pkg/src/button.module.css
   |
   help: this usually means the package was installed from source. Install the published build instead.
```

## How to fix it

1. Install the package from npm rather than from a git URL or local path.
2. If the package is a sibling in a monorepo, make sure the workspace runs the dependency's build before the consumer resolves it.
3. If the warning is from a first-party package in your own monorepo, include that package in the Swift-Rust compilation pipeline so its CSS Modules are inlined.

```json
{
  "name": "consumer",
  "build": {
    "transpilePackages": ["some-pkg"]
  }
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log names the package and the file that was the source of the import, so you can find the bad install.

## Common pitfalls

- The warning is informational, but the resulting class-name output will not match the upstream docs. Custom styles will silently not apply.
- `transpilePackages` only works for in-tree monorepo packages. Random npm packages cannot be transpiled this way.
- A package that publishes both source and a compiled build will work if the consumer's package manager picked the compiled build. Verify the resolved path.
- This warning is a sibling of `SR0032`, which covers non-module CSS.
