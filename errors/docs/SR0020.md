# SR0020 — Built-in swift-rust/font is now the default

The project still imports from `@next/font` (the standalone font package that predates the framework integration). Since Swift-Rust 0.1, `swift-rust/font` ships the same loader and is the supported path. The deprecation is logged so the migration is unmissable.

## Why this error occurred

In the 13.x line of Next.js, `@next/font` was the published package. The Swift-Rust port folded that loader into the core framework so the font bundle is part of the same compile pass as the rest of the app. The old package still resolves, but the import paths are deprecated and the package will be removed from the registry in a future major.

The deprecation is non-fatal so a single stale import in a deep dependency does not break the build, but every match is logged.

## Reproducing the error

```tsx
import { Inter } from "@next/font/google";

const inter = Inter({ subsets: ["latin"] });

export default function Page() {
  return <p className={inter.className}>hello</p>;
}
```

## Error output

```
[WARN] [SR0020] @next/font is deprecated; use swift-rust/font instead
   --> app/page.tsx:1:1
   |
 1 | import { Inter } from "@next/font/google";
   | ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^ import from "swift-rust/font" instead
   |
  help: run `bunx @swift-rust/codemod built-in-font .` to rewrite imports automatically.
```

## How to fix it

1. Uninstall `@next/font` from `package.json`.
2. Replace every `@next/font/...` import with `swift-rust/font`.
3. Re-run the build and confirm the warning is gone.

```tsx
import { Inter } from "swift-rust/font";

const inter = Inter({ subsets: ["latin"] });

export default function Page() {
  return <p className={inter.className}>hello</p>;
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log lists every file that still imports `@next/font` so you can confirm there are no hidden references inside a node_module.

## Common pitfalls

- The codemod only rewrites imports inside your project. If a dependency imports `@next/font` internally, you have to wait for that package to update.
- A bare `import "next/font"` is also flagged — both the `next/...` and `@next/...` paths are deprecated.
- The codemod is best-effort. It will not rewrite non-default exports or re-exports; check the diff carefully.
- This deprecation will become a hard error in Swift-Rust 0.4.0.
