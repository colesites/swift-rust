## Why this error occurred

The <Video> component embeds YouTube and Vimeo via their `/embed/` endpoints, which require a video ID. The component extracts the ID from one of three sources, in order:

1. `youtubeId` / `vimeoId` prop (if provided directly).
2. The first element of a `VideoSource[]` (if `src` is an array).
3. The `src` string itself (if it is a YouTube/Vimeo URL).

If none of these yield a valid ID, the component cannot build the embed URL and reports the failure. The previous behavior was to render a generic "Invalid URL" div with no error code; the new behavior surfaces the error through `onError` and a stable `code: "SR0155"` for log aggregation.

## Reproducing the error

```tsx
<Video src="https://www.youtube.com/@somechannel" />
```

The viewer renders the error UI. The console logs:

```
VideoError [SR0155]: Invalid youtube URL or ID: https://www.youtube.com/@somechannel
    at <VideoSurface> (packages/video/src/index.ts:362:7)
```

## Error output

```
VideoError: invalid-id
  code: "SR0155"
  kind: "invalid-id"
  url: "https://www.youtube.com/@somechannel"
```

The `url` field is the input that failed to parse. For `youtubeId` / `vimeoId` props it is the literal value; for `src` it is the first source's `src`.

## How to fix it

1. **Use a full watch URL.**

   ```tsx
   <Video src="https://www.youtube.com/watch?v=dQw4w9WgXcQ" />
   <Video src="https://www.youtube.com/shorts/dQw4w9WgXcQ" />
   <Video src="https://www.youtube.com/embed/dQw4w9WgXcQ" />
   <Video src="https://youtu.be/dQw4w9WgXcQ" />
   ```

   All four are valid. Shorts and `/embed/` URLs are recognized as of 0.2.0.

2. **Pass the ID directly.**

   ```tsx
   <Video provider="youtube" youtubeId="dQw4w9WgXcQ" />
   ```

   A valid YouTube ID is exactly 11 characters of `[A-Za-z0-9_-]`. A valid Vimeo ID is 6+ digits.

3. **Validate before render.** Use the exported helpers:

   ```ts
   import { isYouTubeUrl, getYouTubeId } from "@swift-rust/video";

   if (isYouTubeUrl(url) && getYouTubeId(url)) {
     return <Video src={url} />;
   } else {
     return <a href={url}>Watch on YouTube</a>;
   }
   ```

4. **Handle playlists and channels explicitly.** The viewer only embeds single videos. If the user pastes a channel URL, treat it as a link, not an embed.

## Diagnosis

| Input | Result |
|---|---|
| `https://www.youtube.com/watch?v=dQw4w9WgXcQ` | ID = `dQw4w9WgXcQ` |
| `https://youtu.be/dQw4w9WgXcQ` | ID = `dQw4w9WgXcQ` |
| `https://www.youtube.com/shorts/dQw4w9WgXcQ` | ID = `dQw4w9WgXcQ` |
| `https://www.youtube.com/embed/dQw4w9WgXcQ` | ID = `dQw4w9WgXcQ` |
| `dQw4w9WgXcQ` (11 chars) | ID = `dQw4w9WgXcQ` |
| `https://www.youtube.com/@channelname` | **invalid** — no video ID |
| `https://www.youtube.com/playlist?list=PL...` | **invalid** — playlist, not a video |
| `https://vimeo.com/channels/staffpicks/123456789` | ID = `123456789` |
| `123456789` (Vimeo, 9 digits) | ID = `123456789` |
| `abc` (3 chars) | **invalid** — too short |
| `""` (empty) | **invalid** |

## Common pitfalls

- **Shorts URLs that point to a `/watch?v=...` link.** Some mobile YouTube URLs are `/shorts/<id>`. Both forms are recognized; if you have a custom domain that proxies YouTube, the parser will not recognize it and you must use `youtubeId` directly.
- **A Vimeo URL with a hash fragment.** `#t=1m30s` is fine; the ID is still extracted from the path.
- **Forgetting to update when the video moves.** If the source migrates the video, the old ID will fail with SR0155. Re-fetch the URL from the source.
- **Embedding a private or unlisted video.** Private videos return a 404 from the embed endpoint, which is reported as `SR0152`-adjacent but surfaces as a network error. The embed will load and show a "Video unavailable" message from YouTube; this is *not* an SR0155 because the ID is valid.
