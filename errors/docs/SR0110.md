# SR0110 — location.assign to a relative URL

Code used `location.assign()` or set `location.href` to a relative URL, which triggers a full page reload and bypasses the Swift-Rust client router. The build flags this so the navigation can be replaced with a router-aware call that preserves prefetched data and view transitions.

## Why this error occurred

The Swift-Rust client router maintains an in-memory cache of route payloads, prefetches links the user is likely to click, and animates view transitions when navigating. A `location.assign('/foo')` or `location.href = '/foo'` reloads the entire document, drops the cache, and re-runs the full SSR pipeline. The user sees a flash of nothing while the next page streams in.

For external URLs (e.g. `https://example.com`), a full page load is the right call and is allowed. For internal routes, the router-aware equivalents are strictly better.

The check is static for direct uses (`location.href = "/foo"`) and runtime for indirect ones (a variable assigned to a relative URL). Direct uses are caught at build time; runtime uses produce a 500 with the same code.

## Reproducing the error

```tsx filename="app/components/my-button.tsx"
"use client"

export function MyButton() {
  return (
    <button onClick={() => { window.location.href = "/dashboard" }}>
      Go to Dashboard
    </button>
  )
}
```

## How to fix it

1. **In a server component or a render-phase hook.** Use `redirect()` from `swift-rust/navigation`.

```tsx filename="app/page.tsx"
import { redirect } from "swift-rust/navigation"

export default function Page() {
  redirect("/dashboard")
}
```

2. **In a client component (event handler).** Use `useRouter().push()`.

```tsx filename="app/components/my-button.tsx"
"use client"

import { useRouter } from "swift-rust/navigation"

export function MyButton() {
  const router = useRouter()
  return (
    <button onClick={() => router.push("/dashboard")}>
      Go to Dashboard
    </button>
  )
}
```

3. **External URLs.** If the target is on a different origin, the rule does not apply.

```ts
window.location.href = "https://example.com"
window.location.assign("https://other.org/path")
```

## Diagnosis

Not applicable: this error is detected at runtime/build only, no CLI command.

## Common pitfalls

- A relative URL is anything that does not start with a protocol. `/foo`, `foo/bar`, `./next`, `../up` are all relative.
- A `window.location.href = "/foo"` inside a `useEffect` is a common migration footgun from class components. Replace with `router.push("/foo")`.
- Hash-only navigation (`window.location.hash = "#section"`) is allowed. The check is for path changes.
- A `meta http-equiv="refresh"` tag is a different mechanism and is not covered by this rule.
