# SR0053 â€” Google font is missing a preload subset

`swift-rust/font/google` builds a static font payload at build time and emits a `<link rel="preload">` for the first subset in the `subsets` array. If you do not provide a `subsets` array, the wrapper would have to download every subset (sometimes 30+ `.woff2` files per weight) just to decide which one to preload, which defeats the purpose. The build aborts with this error to prevent the production output from accidentally shipping the entire Google Fonts catalog.

## Why this error occurred

`preload` is on by default because the most common case â€” Latin body text â€” benefits enormously from a preloaded `.woff2`. But preload is a hint that costs a real connection, so the wrapper requires you to be explicit about which subset is being shipped.

Common causes:

- A team member copies an example from the docs that omits `subsets`.
- A font loader was migrated from a CSS `@import` to the framework wrapper without updating the call signature.
- The previous Next.js project relied on `experimental.fontLoaders` to set a default subset globally. That knob is gone; subsets are now per-font.

In Swift-Rust the error code is the same as the Next.js one, but the build refuses to emit the preload link rather than silently dropping it.

## Reproducing the error

```tsx
import { Inter } from "swift-rust/font/google";

const inter = Inter({ weight: ["400", "700"] });
```

## How to fix it

1. Specify the subset(s) you actually need.

   ```tsx
   import { Inter } from "swift-rust/font/google";

   const inter = Inter({
     subsets: ["latin"],
     weight: ["400", "700"],
   });
   ```

2. For CJK or other large alphabets where preloading is impractical, opt out.

   ```tsx
   import { Noto_Sans_JP } from "swift-rust/font/google";

   const noto = Noto_Sans_JP({
     weight: "400",
     preload: false,
   });
   ```

## Common pitfalls

- A subset name must match Google's naming exactly. Common values: `latin`, `latin-ext`, `cyrillic`, `greek`, `vietnamese`, `arabic`, `hebrew`, `japanese`, `chinese-simplified`, `chinese-traditional`, `korean`.
- For Japanese, prefer `preload: false` â€” the `.woff2` for `japanese` can exceed 4 MB.
- If you are migrating from the legacy `experimental.fontLoaders` config, you have to set the subset on each font call. There is no longer a project-wide default.
- Multiple `subsets` in the array means multiple preload links, which is usually a mistake. The first entry is the one that gets the `<link rel="preload">`; the rest are fetched normally on first paint.
