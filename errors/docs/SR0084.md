# SR0084 — Multiple children passed to <Link>

A `<Link>` component must contain exactly one child element. When more than one element is detected, the compiler rejects the JSX at build time.

## Why this error occurred

The Swift-Rust `<Link>` is implemented as a low-level `<a>` element with a click handler that intercepts navigation. Prefetching, scroll restoration, and view transitions all assume the link resolves to a single navigable target. Two children would make it ambiguous which element receives focus, which click navigates, and which child owns the prefetch.

Historically Next.js 12 required an inner `<a>` and allowed arbitrary siblings. Swift-Rust does not — `<Link>` renders the `<a>` itself, and the children go inside it as the only anchor content.

## Reproducing the error

```tsx filename="app/page.tsx"
import Link from "swift-rust/link"

export default function Home() {
  return (
    <Link href="/about">
      <span>To About</span>
      <span>Second To About</span>
    </Link>
  )
}
```

Two sibling `<span>` elements are passed to `<Link>`.

## Error output

```
error: [SR0084] multiple children passed to <Link>: expected exactly 1 child, got 2
  --> app/page.tsx:5:4
   |
 5 |     <Link href="/about">
 6 |       <span>To About</span>
 7 |       <span>Second To About</span>
 8 |     </Link>
   |     ^^^^^
```

## How to fix it

1. Wrap the children in a single element.
2. If you genuinely need two links, render two `<Link>` instances.

```tsx filename="app/page.tsx"
import Link from "swift-rust/link"

export default function Home() {
  return (
    <Link href="/about">
      <span>
        To About <small>(legacy URL)</small>
      </span>
    </Link>
  )
}
```

```tsx filename="app/page.tsx"
import Link from "swift-rust/link"

export default function Home() {
  return (
    <nav>
      <Link href="/about"><span>About</span></Link>
      <Link href="/contact"><span>Contact</span></Link>
    </nav>
  )
}
```

## Diagnosis

```bash
bun swift-rust doctor --check routing
RUST_LOG=swift_rust=link=trace bun swift-rust dev
```

`--check routing` runs a static analysis pass over `app/` and reports JSX patterns that violate the `<Link>` contract.

## Common pitfalls

- JSX fragments (`<>...</>`) count as multiple children.
- Conditional rendering with `{cond && <A/>}` plus a sibling element fails the check when `cond` is `true`.
- Strings and numbers count as a single child, so this rule does not fire for `<Link>{"text"}</Link>`.
