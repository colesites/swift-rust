# SR0100 — Unconfigured image quality

The `<Image>` component received a `quality` value that is not in the `image.qualities` allow-list defined in `swift-rust.config.json`. The build refuses to optimize the image at the requested quality because an unconstrained quality would inflate egress and CDN cost.

## Why this error occurred

The image optimizer produces a separate variant for every quality value it sees. A page that asks for `quality={37}` produces a unique cached variant that is not shared with `quality={75}`. In a large app, allowing arbitrary qualities produces a combinatorial explosion of variants.

To prevent this, Swift-Rust ships with a default allow-list of `[25, 50, 75]` and requires you to declare any additional values you want to use. The allow-list is also a hint to image providers like Cloudinary and imgix, which charge per variant.

The check is performed at build time on every `<Image>` use in the codebase. A literal value of `42` is caught statically; a runtime value (e.g. `quality={dynamic}`) is caught at request time and reported as a 500 with the same code.

## Reproducing the error

```tsx filename="app/page.tsx"
import Image from "swift-rust/image"

export default function Page() {
  return (
    <Image
      src="/assets/hero.jpg"
      alt="Hero"
      width={1200}
      height={600}
      quality={42}
    />
  )
}
```

`42` is not in the default allow-list.

## Error output

```
error: [SR0100] unconfigured image quality: 42
  --> app/page.tsx:6:4
   |
 6 |   quality={42}
   |            ^^ 42 is not in image.qualities; allowed: [25, 50, 75]
```

## How to fix it

1. Add the value to the `image.qualities` array in `swift-rust.config.json`.
2. Or use one of the default values.

```json filename="swift-rust.config.json"
{
  "image": {
    "qualities": [25, 50, 75, 42, 90]
  }
}
```

```tsx filename="app/page.tsx"
import Image from "swift-rust/image"

export default function Page() {
  return (
    <Image
      src="/assets/hero.jpg"
      alt="Hero"
      width={1200}
      height={600}
      quality={42}
    />
  )
}
```

## Diagnosis

```bash
bun swift-rust doctor --check image
bun swift-rust info --image
```

`--image` prints the merged qualities list and the variants already in the cache.

## Common pitfalls

- The default `qualities` are merged with your config, not replaced. Adding `[42]` to the config does not remove the defaults.
- A literal `quality` of `100` is not in the default allow-list. If you want maximum quality, add `100` explicitly.
- A dynamic value (e.g. `quality={userPref}`) is checked at request time. The 500 response includes the same code, but the build will not catch it.
- The first-party Swift-Rust platform prices per quality variant, so extending the allow-list increases your bill.
