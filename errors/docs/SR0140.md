# SR0140 — Custom routes must be an array

A function in `swift-rust.config.json` (`headers`, `rewrites`, or `redirects`) returned an object instead of an array of route objects. The loader rejects the value because the array shape is what makes it possible to have zero, one, or many rules.

## Why this error occurred

The route configuration is a list of patterns, not a single pattern. The loader iterates the return value; an object cannot be iterated for routes. The framework used to accept both shapes in older versions (treating an object as a one-element array) but the support was removed in favor of an unambiguous array shape.

The error fires at config-load time, so the rest of the config is not parsed. Fix the offending function and the rest will load.

## Reproducing the error

```json
{
  "rewrites": [
    {
      "source": "/feedback",
      "destination": "/feedback/general"
    }
  ]
}
```

```ts
// swift-rust.config.mjs
export default {
  async rewrites() {
    return {
      source: "/feedback",
      destination: "/feedback/general",
    };
  },
};
```

The config file overrides the JSON shape with the JS form, and the JS form returns an object — which is rejected.

## Error output

```
[ERROR] [SR0140] custom routes must be an array
  --> swift-rust.config.mjs:3:5
    |
  3 |   async rewrites() {
  4 |     return {
  5 |       source: '/feedback',
    |       ^^^^^^
    | expected an array of route objects; got an object.
    |
    help: wrap the return value in an array: `return [{ source: ..., destination: ... }]`.
```

## How to fix it

1. Wrap the return value in an array:

```ts
// swift-rust.config.mjs
export default {
  async rewrites() {
    return [
      {
        source: "/feedback",
        destination: "/feedback/general",
      },
    ];
  },
};
```

2. Verify the same shape for `headers()` and `redirects()`:

```ts
export default {
  async headers() {
    return [
      {
        source: "/(.*)",
        headers: [{ key: "x-frame-options", value: "DENY" }],
      },
    ];
  },
  async redirects() {
    return [
      { source: "/old", destination: "/new", permanent: true },
    ];
  },
};
```

3. Re-run the command; the config should load cleanly.

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=config=trace bun swift-rust dev
```

`config=trace` prints the parsed value of every route function, including the resolved array length, so you can confirm the fix.

## Common pitfalls

- An empty config (returning `null` or `undefined`) is allowed. The check only fires for non-array, non-null values.
- A function that conditionally returns either an object or an array is rejected. Always return an array.
- A JSON config cannot return objects-vs-arrays dynamically; the shape is fixed by the JSON itself. This error is more common in `.mjs` configs.
- The check applies to all three functions: `headers`, `rewrites`, `redirects`. The error names the offending function.
