# SR0035 — analyticsId config is deprecated

The `analytics.analyticsId` field in `swift-rust.config.json` is deprecated. The field was the original way to opt the app into Vercel's Web Vitals service. The replacement is the `useReportWebVitals` hook, which lets the app send metrics to any backend.

## Why this error occurred

When the analytics field was added, it was tightly coupled to a single platform. Swift-Rust is platform-agnostic, so the framework now exposes a hook and lets the app route the metrics anywhere. The legacy field is still parsed so old projects keep working, but the deprecation warning is printed so the migration path is unmissable.

The deprecation is non-fatal. The build succeeds, but every config read prints a warning so the field cannot be left in by accident.

## Reproducing the error

```json
{
  "name": "site",
  "analytics": {
    "analyticsId": "abc123"
  }
}
```

## Error output

```
[WARN] [SR0035] analyticsId is deprecated: use the useReportWebVitals hook instead
   --> swift-rust.config.json:3:5
   |
 3 |     "analyticsId": "abc123"
   |     ^^^^^^^^^^^^^^^^^^^^^^^^ remove this field and call useReportWebVitals() in your root layout
   |
  help: see https://swift-rust.dev/docs/analytics for the migration guide.
```

## How to fix it

1. Remove the `analytics.analyticsId` field from `swift-rust.config.json`.
2. In `app/layout.tsx`, call `useReportWebVitals` and forward the metrics to your analytics backend of choice.
3. Re-run the build and confirm the warning is gone.

```json
{
  "name": "site"
}
```

```tsx
"use client";
import { useReportWebVitals } from "swift-rust/analytics";

export function WebVitals() {
  useReportWebVitals((metric) => {
    fetch("/api/metrics", {
      method: "POST",
      body: JSON.stringify(metric),
    });
  });
  return null;
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log records every read of the deprecated field, so you can confirm no third-party tooling is silently depending on it.

## Common pitfalls

- A nested `analytics` object can have other valid keys (e.g. `analytics.debug`). Remove only `analyticsId`.
- Some hosted platforms inject the field automatically during deployment. Check your platform's config too.
- The hook only runs in client components. Add it from a small `"use client"` component inside the root layout.
- This deprecation will become a hard error in Swift-Rust 0.4.0.
