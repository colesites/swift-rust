# SR0066 — Invalid external rewrite

Rewrites in `swift-rust.config.json` map an incoming URL to a different one served by the same app. The `basePath: false` flag is a hint to the router that this particular rewrite targets an external system — typically used during a partial migration from an old site to a new Swift-Rust app, where some URLs need to be proxied to a legacy backend.

If `basePath: false` is set but the `destination` is an internal path (e.g. `'/legacy/home'`), the router cannot decide whether the rewrite should be served locally or proxied. The framework raises this error to disambiguate.

## Why this error occurred

The `basePath: false` flag has a specific meaning: "do not prefix the destination with the project's `basePath`". When the destination is a relative path, that flag is irrelevant and the rewrite is just an internal redirect. When the destination is an absolute URL, the flag tells the runtime to dispatch the request to that URL as a proxy.

Mixing the two is almost always a typo:

- The author wanted an internal rewrite and forgot to remove `basePath: false`.
- The author wanted a proxy and forgot to make the destination absolute.
- The config was copy-pasted from a different rewrite that did target an external system.

The fix is to align the flag with the destination shape.

## Reproducing the error

```json
{
  "rewrites": [
    {
      "source": "/legacy/:path*",
      "destination": "/v1/:path*",
      "basePath": false
    }
  ]
}
```

## Error output

```
error: [SR0066] invalid external rewrite
  --> swift-rust.config.json:3:5
   |
 3 |     {
 4 |       "source": "/legacy/:path*",
 5 |       "destination": "/v1/:path*",
 6 |       "basePath": false
   |       ^^^^^^^^^^^^^^
   |
   = `basePath: false` requires the destination to be an external URL
     (starting with http:// or https://). `/v1/:path*` is an internal
     path. either drop `basePath: false` or make the destination a URL.
```

## How to fix it

1. For an internal rewrite, drop the flag.

   ```json
   {
     "rewrites": [
       {
         "source": "/legacy/:path*",
         "destination": "/v1/:path*"
       }
     ]
   }
   ```

2. For a proxy, make the destination absolute.

   ```json
   {
     "rewrites": [
       {
         "source": "/legacy/:path*",
         "destination": "https://legacy-backend.example.com/:path*",
         "basePath": false
       }
     ]
   }
   ```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust route:list --rewrites
RUST_LOG=swift_rust=routing=trace bun swift-rust dev
```

## Common pitfalls

- The `basePath: false` flag is independent of the global `basePath` config. Setting it on a rewrite that does not need it is a no-op when the destination is internal, and a hard error when paired with an external URL.
- The destination URL must include the scheme (`https://`). A bare `cdn.example.com/foo` is treated as a path and triggers this error.
- This is the Swift-Rust analogue of Next.js's `invalid-external-rewrite`. The fix is identical.
