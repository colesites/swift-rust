# SR0107 — Manual stylesheet <link> tag

A `<link rel="stylesheet" href="...">` was rendered in JSX. Swift-Rust's CSS pipeline does not pick up stylesheets loaded this way, so the styles never reach the optimized CSS payload and may not apply.

## Why this error occurred

Swift-Rust's CSS handling is bundler-driven. The bundler scans every file in your `app/` directory for `import './foo.css'` statements, follows the chain into CSS Modules and Sass, deduplicates, and produces a single optimized CSS file that the framework injects into the document. Anything loaded through a manual `<link>` tag bypasses this pipeline: the bundler does not see it, the framework does not know to wait for it before declaring the document ready, and the order of rules is no longer guaranteed.

This is a hard error at build time because silent breakage here is common — a `<link>` to a missing file produces a 404 in the network tab and a flash of unstyled content, with no build-time signal that anything is wrong.

## Reproducing the error

```tsx filename="app/page.tsx"
export default function Page() {
  return (
    <main>
      <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter" />
      <h1>Welcome</h1>
    </main>
  )
}
```

## Error output

```
error: [SR0107] manual <link rel="stylesheet"> is not allowed
  --> app/page.tsx:4:7
   |
 4 |       <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter" />
   |       ^^^^^ import the stylesheet from a component or use the font API instead
```

## How to fix it

1. **Local stylesheet.** Import it from any module the bundler traverses (a server component, a client component, the root layout, or another CSS file).

```tsx filename="app/page.tsx"
import "./page.css"

export default function Page() {
  return (
    <main>
      <h1>Welcome</h1>
    </main>
  )
}
```

2. **Third-party stylesheet.** Use CSS `@import` in your root stylesheet, or use the appropriate Swift-Rust API (for example, `swift-rust/font` for Google Fonts).

```css filename="app/globals.css"
@import "https://fonts.googleapis.com/css2?family=Inter&display=swap";

body {
  font-family: "Inter", sans-serif;
}
```

```tsx filename="app/page.tsx"
import "./globals.css"

export default function Page() {
  return (
    <main>
      <h1>Welcome</h1>
    </main>
  )
}
```

3. **CSS Modules.** Import the module and use the exported class names.

```tsx filename="app/button.tsx"
import styles from "./button.module.css"

export function Button({ children }: { children: React.ReactNode }) {
  return <button className={styles.primary}>{children}</button>
}
```

## Diagnosis

```bash
bun swift-rust doctor --check styles
bun swift-rust info --css
```

`--css` prints the merged stylesheet graph and the entry points the bundler discovered.

## Common pitfalls

- A `<link>` tag in the root layout (`app/layout.tsx`) is also rejected, even though it is the closest analog to the old `_document.tsx` pattern.
- Server-rendered HTML in route handlers (`new Response(html)`) is a different code path and is not subject to this rule — but it is also not subject to any of the optimizations, so prefer the framework's CSS pipeline.
- A `<link>` with `rel="preload"` for a non-CSS resource is fine. The check is on `rel="stylesheet"` specifically.
- Dynamic stylesheet insertion (`document.head.appendChild(link)`) at runtime is not caught by the build, but is not recommended for the same reasons.
