# SR0039 — Empty configuration file

A `swift-rust.config.json` (or `.toml`/`.yaml` equivalent) exists in the project root but does not export a configuration object. The build proceeds with defaults, but the warning is printed so the situation is not invisible.

## Why this error occurred

Most scaffolded projects include a placeholder config file that the developer is expected to fill in. If the placeholder is left untouched, the build still works because every field has a default. But the empty file is a smell: it usually means the developer was unaware the file existed, or a migration script deleted its contents. Printing a warning makes sure the project never silently ships an unintentionally empty config.

The warning is non-fatal. The framework reads the defaults and continues.

## Reproducing the error

```json
{}
```

## Error output

```
[WARN] [SR0039] empty configuration: swift-rust.config.json exports no fields
   --> swift-rust.config.json:1:2
   |
 1 | {}
   | ^^ no configuration provided; using defaults
   |
  help: either fill in the file or delete it. An empty file is a footgun, not a default.
```

## How to fix it

1. If you do not need to customize anything, delete the file. The defaults will be used.
2. If you intend to set fields, fill in at least `name` (and any other required field for your deployment).
3. Re-run the build and confirm the warning is gone.

```json
{
  "name": "my-site"
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust dev
```

The trace log records the resolved config (after defaults are merged) and the source file that produced it, so you can confirm whether the warning is real or a false positive.

## Common pitfalls

- A JSON file with only comments is **invalid** and will produce a parse error, not this warning. Use TOML or YAML if you need comments.
- A YAML file with a single bare word (`name`) parses as a string, not a mapping. Wrap it in `{ name: "my-site" }` or use a block mapping.
- The same check applies to `.toml` and `.yaml` files. The framework auto-detects the format from the extension.
- A config with only deprecated fields (`target`, `analyticsId`) is also flagged as "empty" for the purposes of this warning.
