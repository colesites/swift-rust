# SR0001 â€” Configuration error

The framework reads `swift-rust.config.json` (or `.toml`/`.yaml`) from the project root. If the file is missing, malformed, or contains unknown fields, this error is raised at startup, before any source file is touched.

## Why this error occurred

The configuration is the source of truth for the rendering mode, asset domains, font subsets, and PDF defaults. Every other subsystem reads from this file, so an unparseable value would silently corrupt the rest of the build. The framework fails fast instead.

The config is resolved once at process start and re-validated every time a watcher event fires on the file, so a typo introduced during a refactor is caught the next time the dev server reloads.

## Reproducing the error

```json filename="swift-rust.config.json"
{
  "name": "my-app",
  "rendering": "ssr-wasm",
  "imag": { "domains": ["cdn.example.com"] }
}
```

## Error output

```
error: [SR0001] configuration error: unknown field `imag`, expected one of `rendering`, `image`, `font`, `pdf`
  --> swift-rust.config.json:4:2
   |
 4 |   "imag": { "domains": ["cdn.example.com"] }
   |   ^^^^^^
   |
   help: did you mean `image`?
```

## How to fix it

1. Inspect the field name and casing. Swift-Rust expects camelCase in JSON, kebab-case in TOML, and snake_case in YAML.
2. Cross-reference the field against `errors/schema.json` or `docs/configuration.md`.
3. Re-run `bun swift-rust dev` to confirm the config parses.

```json filename="swift-rust.config.json"
{
  "name": "my-app",
  "rendering": "ssr-wasm",
  "image": { "domains": ["cdn.example.com"] }
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust::config=trace bun swift-rust dev
```

`doctor` walks the file tree and prints which fields it could and could not resolve. `info` dumps the fully merged config. The trace log includes the line/column of any failed parse.

## Common pitfalls

- YAML interprets `no` as the boolean `false` and `yes` as `true`. Quote strings that should be text: `name: "my-app"`.
- A trailing comma in JSON is valid in modern runtimes but the framework's parser rejects it. Run `bunx jsonlint swift-rust.config.json` to catch this.
- The `rendering` enum is `ssr`, `ssr-wasm`, `ssr-htmx`, or `wasm`. Anything else triggers this error, not a runtime fallback.
- Environment variables in the config (`"${PORT}"`) are expanded by the shell before the framework reads the file, so missing vars surface here, not at the listening port.
