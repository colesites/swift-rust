# SR0042 — experimental.appDir config is deprecated

The `experimental.appDir` field in `swift-rust.config.json` is no longer needed. The `app/` directory has been the only supported router in Swift-Rust since 0.1, so the flag is dead weight.

## Why this error occurred

Before Swift-Rust 0.1, the `app/` directory was opt-in via the `experimental.appDir` flag. The flag existed to let projects run with the legacy pages-router while incrementally migrating. Once `app/` became the default, the flag stuck around in some configs as a no-op, which is confusing for readers. The deprecation logs every read so a stale config cannot survive a CI migration unnoticed.

The deprecation is non-fatal. The build continues; the field is simply ignored.

## Reproducing the error

```json
{
  "name": "site",
  "experimental": {
    "appDir": true
  }
}
```

## Error output

```
[WARN] [SR0042] experimental.appDir is deprecated: `app/` is the only supported router
   --> swift-rust.config.json:3:5
   |
 3 |     "appDir": true
   |     ^^^^^^^^^^^^^^ remove this field; `app/` is the default and only router
   |
  help: see https://swift-rust.dev/docs/app for the app router guide.
```

## How to fix it

1. Remove the `experimental.appDir` block from `swift-rust.config.json`.
2. Verify the `app/` directory exists at the project root and contains your routes.
3. Re-run the build and confirm the warning is gone.

```json
{
  "name": "site"
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log records every read of the deprecated field, which is useful when a CI script regenerates the config from a template.

## Common pitfalls

- A nested `experimental` block can have other valid keys (e.g. `experimental.swcPlugins`). Remove only `appDir`.
- Some hosts inject the flag automatically during deployment. Check the platform's project settings as well.
- Renaming the directory from `app/` to something else is **not** supported — the router name is hard-coded.
- This deprecation will become a hard error in Swift-Rust 0.4.0.
