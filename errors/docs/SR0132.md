# SR0132 — React client hook used in a server component

A server component is calling a React hook that only exists in the client runtime (`useState`, `useEffect`, `useRef`, `useMemo`, `useCallback`, `useContext`, custom hooks that wrap any of these). The server runtime does not provide these hooks, so the call throws at render time.

## Why this error occurred

Swift-Rust's renderer splits the tree into two environments. The server runtime is a stripped-down React renderer that supports async components, server-only data fetching, and JSX → HTML output, but it does not implement state, effects, refs, or context. Those are client-only.

A component is "client" when its source file starts with the `'use client'` directive. Anything that does not have the directive is treated as a server component, and the client hooks are unavailable.

The directive is detected by string-matching the very first non-whitespace token of the file. If the directive is missing — because the file does not have it, or because a server file re-exports a client component without preserving the directive — the hook call fails.

## Reproducing the error

```tsx
import { useEffect, useState } from "react";

export default function Banner() {
  const [dismissed, setDismissed] = useState(false);
  useEffect(() => {
    console.log("mounted");
  }, []);
  return (
    <aside>
      <button onClick={() => setDismissed(true)}>x</button>
    </aside>
  );
}
```

## Error output

```
[ERROR] [SR0132] react client hook used in a server component
  --> app/banner.tsx:3:1
    |
  3 | export default function Banner() {
  4 |   const [dismissed, setDismissed] = useState(false);
    |                                ^^^^^^^
    | `useState` is a client-only hook; the file is missing the `'use client'` directive.
    |
    help: add `'use client';` at the top of the file, or move the hook call into a child client component.
```

## How to fix it

1. Add the directive at the top of the file:

```tsx
"use client";

import { useEffect, useState } from "react";

export default function Banner() {
  const [dismissed, setDismissed] = useState(false);
  useEffect(() => {
    console.log("mounted");
  }, []);
  return (
    <aside>
      <button onClick={() => setDismissed(true)}>x</button>
    </aside>
  );
}
```

2. If the hook belongs in a child of a server component, keep the server component and create a new client file for the child:

```tsx
// app/banner.tsx (server)
import { BannerClient } from "./banner-client";

export default function Banner() {
  return <BannerClient />;
}
```

```tsx
// app/banner-client.tsx (client)
"use client";

import { useState } from "react";

export default function BannerClient() {
  const [dismissed, setDismissed] = useState(false);
  return <button onClick={() => setDismissed(true)}>x</button>;
}
```

3. Re-run the dev server and confirm the error is gone.

## Diagnosis

```bash
bun swift-rust doctor
RUST_LOG=swift_rust=render=trace bun swift-rust dev
```

`render=trace` prints the resolved mode (server / client) of every component in the tree, which makes it easy to see which file is being treated as a server component by mistake.

## Common pitfalls

- Re-exporting a client component from a barrel file (`index.ts`) preserves the directive only if the barrel re-exports the directive itself. A `export { Banner } from "./banner"` is fine; an `export { default as Banner } from "./banner"` is also fine; a manual re-export that re-implements the JSX is not.
- The directive must be on the very first line. A leading comment or `/* eslint-disable */` line above it is allowed; any executable code above it is not.
- A custom hook that wraps `useState` is also client-only. The directive applies to the file that defines the hook.
- A `use server` file is the opposite of a `use client` file: it runs in the server runtime. Mixing the two directives in the same file is an error.
