# SR0115 â€” Typo in configuration key

A configuration value or option name is misspelled and was silently ignored. The build did not fail â€” typos in the config are not fatal â€” but the option you intended is not in effect, which can mask other problems (an unconfigured image host, a redirect that never fires, a server option that never gets passed through).

## Why this error occurred

`swift-rust.config.json` (and its `.toml` and `.yaml` siblings) is parsed against a strict schema. Unknown keys are reported as a warning rather than rejected, because:

- A typo is almost always a mistake, but the framework cannot tell a typo from a comment-like extra field.
- A schema rejection would also fail the build for legitimate "future" fields that the running version does not know about yet.
- Some users intentionally add metadata fields (`_comment`, `x-internal-id`) for documentation purposes.

So the framework prints a warning with the unknown key, the closest valid key, and a pointer to the schema. The message is `info` rather than `warning` because the build still succeeds, but it is the kind of message you want to fix in code review.

## Reproducing the error

```json filename="swift-rust.config.json"
{
  "imag": {
    "remotePatterns": [
      { "protocol": "https", "hostname": "cdn.example.com" }
    ]
  }
}
```

The key is `imag`, not `image`.

## Error output

```
info: [SR0115] typo in configuration key: `imag` is not a known option
       did you mean: `image`?
       the field was ignored; the framework is using its default (no image hosts allowed)
  --> swift-rust.config.json:2:3
   |
 2 |   "imag": {
   |   ^^^^^^
```

## How to fix it

1. Compare the unknown key with the suggested alternative.
2. Update the config to use the correct spelling.
3. Re-run `bun swift-rust info` to confirm the resolved config now contains the field you intended.

```json filename="swift-rust.config.json"
{
  "image": {
    "remotePatterns": [
      { "protocol": "https", "hostname": "cdn.example.com", "pathname": "/**" }
    ]
  }
}
```

## Diagnosis

```bash
bun swift-rust info
RUST_LOG=swift_rust=config=trace bun swift-rust build
```

`bun swift-rust info` prints the merged config and flags every unknown key with the suggestion.

## Common pitfalls

- The closest-key suggestion is computed by edit distance, not by meaning. A typo that is equidistant from two valid keys (e.g. `headers` vs `header`) will pick one arbitrarily â€” read the suggestion, don't trust it blindly.
- Casing matters. `RemotePatterns` is not the same as `remotePatterns` in JSON; the framework expects camelCase.
- A field that was renamed in a previous major version is reported as a typo rather than as a deprecation, because the framework does not know if you are targeting the new or the old version. Check the migration guide.
- Comments inside the config are stripped before validation. A `_comment` field is allowed and is not flagged.
