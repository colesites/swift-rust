# SR0067 — Invalid `href` passed to router

Swift-Rust's client router only handles internal navigation — paths that resolve to a route in the same `app/` tree. The router has no way to perform a full-page load, a hash jump, or a navigation to a different origin; those are the browser's job. Passing an external URL or a non-HTTP scheme to `<Link>` or `useRouter()` raises this error.

## Why this error occurred

The router assumes that every navigation it performs will result in a server-rendered HTML payload that it can stream into the current page. An external URL would skip the server entirely, a `mailto:` link would open the user's mail client, and a `tel:` link would invoke the dialer. None of those flows make sense inside a single-page app, and silently falling back to a full reload would defeat the purpose of using the client router.

Common causes:

- A team member pastes a marketing URL into a `<Link>` thinking it will work like an `<a>`.
- A `<Link href="https://…">` is used for a link to an external blog post.
- A `<Link href="#section">` is used for in-page anchors. Those should be plain `<a>` tags.
- A `useRouter().push("https://…")` is used as a substitute for `window.location.href = "…"`.

The error is raised at click time. The router refuses to navigate, so the user sees no change in the URL.

## Reproducing the error

```tsx
import Link from "swift-rust/link";

export function Footer() {
  return (
    <Link href="https://example.com/blog">Read the blog</Link>
  );
}
```

## Error output

```
[ERROR] [SR0067] invalid href passed to router
  --> app/components/Footer.tsx:4:3
   |
   = "https://example.com/blog" is an external URL. the client router
     only handles internal navigation. use a plain <a> tag for
     external links.
```

## How to fix it

1. Use a plain `<a>` tag for external URLs and `mailto:`/`tel:` links.

   ```tsx
   export function Footer() {
     return (
       <a href="https://example.com/blog" rel="noopener noreferrer">
         Read the blog
       </a>
     );
   }
   ```

2. For hash-only anchors on the same page, use a plain `<a href="#section">`.

3. If you genuinely need to navigate the current tab to an external URL from code, use the browser API directly.

   ```ts
   window.location.href = "https://example.com/blog";
   ```

## Common pitfalls

- The error fires at click time. If you have a long list of `<Link>` instances and only one is wrong, the warning will not tell you which one. Grep for `href="https://` to find candidates.
- A `<Link>` with a fully-qualified URL to the same origin is technically internal — but the framework warns because that pattern usually indicates a misconfigured base path. Pass the path-only form (`/about`) and let the framework resolve the origin.
- Protocol-relative URLs (`//cdn.example.com/foo`) are also rejected. Always include the scheme.
- This is the Swift-Rust analogue of Next.js's `invalid-href-passed`. The fix is identical.
