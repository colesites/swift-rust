# SR0069 — Invalid instant configuration

The `unstable_instant` export is an opt-in signal that a layout or page contains user interface that should appear instantly — typically a fully-rendered shell, a hero section, or above-the-fold content that must be visible before hydration completes. The framework uses it to decide whether to wait for the entire tree to be ready before sending the response.

The export accepts a boolean or a small object describing the validation level. Anything else is rejected at build time.

## Why this error occurred

The framework needs a static, machine-readable signal at build time so it can decide which routes qualify for the "instant" treatment. A non-boolean value would force it to guess, and a non-object value with extra fields could conflict with future options.

Common mistakes:

- Setting `unstable_instant = 'yes'` (a string) instead of `true`.
- Setting `unstable_instant = { level: 'verbose' }` with an unknown level.
- Setting the export in a non-route file (e.g. a utility module).
- Setting the export on a page that is a server action only (no UI).

The error fires at the next dev rebuild, or at the start of `bun swift-rust build`.

## Reproducing the error

```tsx
export const unstable_instant = "yes";
```

## Error output

```
error: [SR0069] invalid instant configuration
  --> app/[slug]/page.tsx:1:1
   |
 1 | export const unstable_instant = "yes";
   |                                  ^^^^^
   |
   = expected a boolean or `{ level: 'error' | 'warning' | 'info' }`.
   = got string "yes". if you want to mark the route as instant, use
     `export const unstable_instant = true;`.
```

## How to fix it

1. Use a boolean to mark the route as instant.

   ```tsx
   export const unstable_instant = true;

   export default function Page() {
     return <h1>Hello, world</h1>;
   }
   ```

2. Use an object to customize the validation level that the framework reports when the instant criteria are not met.

   ```tsx
   export const unstable_instant = {
     level: "warning",
   };

   export default function Page() {
     return <h1>Hello, world</h1>;
   }
   ```

3. Use `false` to opt a route out of the instant check. This is the default; the export can be omitted entirely.

   ```tsx
   export const unstable_instant = false;

   export default function Page() {
     return <h1>Hello, world</h1>;
   }
   ```

## Diagnosis

```bash
bun swift-rust doctor
RUST_LOG=swift_rust=instant=trace bun swift-rust build
```

## Common pitfalls

- `unstable_instant = true` does not disable prefetching; the framework still prefetches the route when a `<Link>` enters the viewport.
- The `level` field is reported in the build log. Setting it to `'error'` fails the build, `'warning'` prints a notice, `'info'` only appears under verbose logging.
- The export is read at build time. Changing it requires a full rebuild, not just a HMR cycle.
- The `unstable_` prefix is a signal that the option may change. Treat it as a contract that is honored in the current minor but not guaranteed in the next.
