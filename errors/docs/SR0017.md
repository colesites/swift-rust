# SR0017 — Uncached data accessed outside of <Suspense>

A component accessed request-time or async data without a parent `<Suspense>` boundary and without a `"use cache"` directive. The compiler cannot prerender the surrounding tree and is forced to wait, blocking the initial HTML response.

## Why this error occurred

When the cache-components flag is enabled, the Swift-Rust compiler tries to render as much of the tree as possible at build time. Any async read that is **not** explicitly cached is assumed to be per-request, which means the surrounding shell cannot be prerendered. To keep the streaming model fast, the compiler requires such reads to live inside a `<Suspense>` boundary so it knows what fallback to show while the data is loading.

The same condition applies to request-time APIs (`cookies()`, `headers()`, `connection()`) — those are inherently per-request and must be fenced off.

## Reproducing the error

```tsx
import { headers } from "swift-rust/headers";

export default async function Page() {
  const h = await headers();
  return <p>user-agent = {h.get("user-agent")}</p>;
}
```

## Error output

```
[WARN] [SR0017] uncached data accessed outside of <Suspense>
   --> app/page.tsx:4:10
   |
 4 |   const h = await headers();
   |              ^^^^^^^^^^^^^^^ request-time API reached from a prerenderable tree
   |
  help: wrap the component in <Suspense> with a fallback, or annotate it with "use cache".
```

## How to fix it

1. Push the data read into a child component and wrap that child in `<Suspense>` with a fallback.
2. If the data is genuinely cacheable, annotate the function with `"use cache"` and give it a `cacheLife`.
3. If the whole route only renders dynamic content, add a `loading.tsx` so the shell can stream immediately.

```tsx
import { Suspense } from "react";
import { headers } from "swift-rust/headers";

function UserAgent() {
  const h = headers();
  return <p>user-agent = {h.get("user-agent")}</p>;
}

export default function Page() {
  return (
    <Suspense fallback={<p>loading…</p>}>
      <UserAgent />
    </Suspense>
  );
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The `build` trace lists every uncached read it found, paired with the component it was reached from. Run with `--debug-prerender` to capture a stack frame at the exact await site.

## Common pitfalls

- Moving the read into a child component is not enough if the child is rendered **outside** a `<Suspense>` boundary somewhere up the tree.
- `"use cache"` and `<Suspense>` solve different problems. The cache is for data that **can** be shared across requests; Suspense is for data that **must** wait.
- `loading.tsx` only applies to the segment it lives in. Nested dynamic reads still need their own boundary.
- This warning is a soft signal. Pair it with `SR0015` to see whether the page also flipped its rendering mode at runtime.
