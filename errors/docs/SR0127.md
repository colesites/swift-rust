# SR0127 — Link prefetch={true} is deprecated

`<Link prefetch={true}>` is the default behavior in Swift-Rust and the prop is no longer needed. Passing it explicitly is a no-op and triggers a deprecation warning so you can clean up the call sites.

## Why this error occurred

Earlier versions of Swift-Rust (and Next.js before it) treated `prefetch` as an opt-in. The default was changed to "prefetch links when they enter the viewport", and the boolean was kept for opt-out only. Code that explicitly passes `true` is now redundant and is flagged for removal.

The default uses an `IntersectionObserver` to detect when a link becomes visible. The request has a low priority and yields to higher-priority network activity. If the user has data-saver enabled, the prefetch is skipped entirely.

## Reproducing the error

```tsx
import Link from "swift-rust/link";

export default function Nav() {
  return (
    <nav>
      <Link href="/about" prefetch={true}>
        About
      </Link>
      <Link href="/contact" prefetch>
        Contact
      </Link>
    </nav>
  );
}
```


## How to fix it

1. Remove the `prefetch` prop. The default behaviour is identical.

```tsx
import Link from "swift-rust/link";

export default function Nav() {
  return (
    <nav>
      <Link href="/about">About</Link>
      <Link href="/contact">Contact</Link>
    </nav>
  );
}
```

2. If a route is rarely visited and you want to avoid the prefetch cost, set `prefetch={false}`:

```tsx
<Link href="/admin/audit-log" prefetch={false}>
  Audit log
</Link>
```

## Diagnosis

```bash
bun swift-rust doctor
RUST_LOG=swift_rust=link=debug bun swift-rust dev
```

The `link=debug` log prints the prefetch decision per link, including whether the IntersectionObserver fired and whether the request was issued.

## Common pitfalls

- The deprecation is purely a lint; the build is not affected. The warning is collected by the codemod but does not fail CI by default.
- The `prefetch` prop still accepts `false` to opt out. The codemod only flags `true`.
- The default does not apply to `<a>` tags. To get prefetching on a raw anchor, use `<Link>` from `swift-rust/link`.
- Data-saver (and the `Save-Data` request header) bypass the prefetch. The behaviour is global, not per-link.
