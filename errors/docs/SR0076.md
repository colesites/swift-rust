# SR0076 — Invalid relative `href` and external `as` values

When a `<Link>` is given both an `href` and an `as` prop, the framework uses `as` to determine the URL the browser navigates to and `href` to determine which route in the route table to render. The two must agree: if `href` is internal, `as` must be internal; if `href` is external, `as` must be external. A relative `href` paired with an external `as` (or vice versa) is rejected.

## Why this error occurred

The router needs to know what page to render. If `href` is internal, the framework will try to find a matching route in the `app/` tree. If `as` is external, the URL the browser actually navigates to is on a different origin and the framework cannot render anything for it. The combination is almost always a typo:

- The author meant to pass the external URL as `href` and forgot.
- The author meant to keep `as` internal and pasted an absolute URL by accident.
- The `as` was originally a `mailto:` link left over from an older `<a>` element.

The error fires on click, not on render, mirroring the rest of the `<Link>` validation family.

## Reproducing the error

```tsx
import Link from "swift-rust/link";

export function Contact() {
  return (
    <Link href="/contact" as="mailto:hello@example.com">
      Email us
    </Link>
  );
}
```

## Error output

```
[ERROR] [SR0076] invalid relative href and external as values
  --> app/components/Contact.tsx:4:3
   |
   = href "/contact" is internal but as "mailto:hello@example.com"
     is external. they must agree. either pass the external URL as
     `href` (and drop `as`), or use a plain <a> tag for the mailto
     link.
```

## How to fix it

1. For external URLs, drop `as` and pass the URL as `href`.

   ```tsx
   import Link from "swift-rust/link";

   export function Contact() {
     return (
       <Link href="mailto:hello@example.com">
         Email us
       </Link>
     );
   }
   ```

2. For `mailto:`/`tel:` links, prefer a plain `<a>` tag — the framework does not need to do anything special for those.

   ```tsx
   export function Contact() {
     return <a href="mailto:hello@example.com">Email us</a>;
   }
   ```

3. For relative `<Link>`s, keep `as` as a path or drop it entirely.

   ```tsx
   <Link href="/about">About</Link>
   ```

## Common pitfalls

- The error fires on click, not on render. Audit the codebase with a search for `<Link` to find all candidates, not just the one that surfaces the warning.
- A `mailto:` link through `<Link>` is sometimes desirable for prefetching the contact page, but the framework already inlines the address and a `mailto:` link has nothing to prefetch. Use a plain `<a>`.
- Protocol-relative URLs (`//example.com/foo`) are also considered external and trigger this error.
- This is the Swift-Rust analogue of Next.js's `invalid-relative-url-external-as`. The fix is identical.
