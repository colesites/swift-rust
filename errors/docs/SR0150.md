# SR0150 — url property is deprecated

A page component references the legacy `url` prop. The prop was injected automatically into every page in very old versions of the framework, but it has been removed in favor of explicit router hooks. The deprecation is final; the prop is no longer set.

## Why this error occurred

The "magical" `url` prop was convenient but unpredictable. A user could not tell from the source whether a prop was framework-injected or user-supplied, and the implicit injection made it easy to shadow framework values. The framework removed the injection and exposed the equivalent data through `useRouter()` (and, for class components, the `withRouter` HOC).

The codemod handles the common patterns automatically. The deprecation message is a hint that the migration is in progress.

## Reproducing the error

```tsx
class Page extends React.Component<{ url: { pathname: string } }> {
  render() {
    return <div>{this.props.url.pathname}</div>;
  }
}

export default Page;
```

## Error output

```
[ERROR] [SR0150] `url` page prop is deprecated
  --> app/index/page.tsx:2:23
    |
  2 | class Page extends React.Component<{ url: { pathname: string } }> {
    |                                              ^^^^^^^^^^^^^^^^^^
    | the `url` prop is no longer injected into pages. use the router hooks.
    |
    help: import `useRouter` from `swift-rust/navigation`, or wrap the class in `withRouter`.
```

## How to fix it

1. For function components, use the router hook:

```tsx
"use client";

import { useRouter } from "swift-rust/navigation";

export default function Page() {
  const router = useRouter();
  return <div>{router.pathname}</div>;
}
```

2. For class components, wrap with `withRouter`:

```tsx
import { withRouter } from "swift-rust/navigation";

class Page extends React.Component<{ router: { pathname: string } }> {
  render() {
    return <div>{this.props.router.pathname}</div>;
  }
}

export default withRouter(Page);
```

3. Or run the codemod to handle the migration in bulk:

```bash
bunx @swift-rust/codemod url-to-withRouter .
```

## Diagnosis

```bash
bun swift-rust doctor
RUST_LOG=swift_rust=router=trace bun swift-rust dev
```

`router=trace` prints the resolved router object for every page render, including `pathname`, `asPath`, and `query`. The doctor output flags any remaining `url` references in your source.

## Common pitfalls

- React DevTools can synthesize a `url` prop in its tree view. Disable the extension if you see the warning without code references.
- The `url` prop was inherited from React class components. A function component without the prop type still gets a TypeScript error if you read `this.props.url`.
- The `withRouter` HOC is the supported migration path for class components. There is no "magic" replacement.
- Some codebases imported `url` from a helper module. That import is unrelated to the deprecated prop; the warning is about `props.url`, not about a named import.
