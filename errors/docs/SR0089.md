# SR0089 — Missing data-scroll-behavior attribute

Your CSS enables smooth scrolling at the document level (`html { scroll-behavior: smooth }`), but the `<html>` element in your root layout is missing the `data-scroll-behavior="smooth"` attribute. Swift-Rust uses that attribute to decide whether to temporarily reset the smooth scroll behavior during client-side navigation.

## Why this error occurred

Smooth scrolling looks great on direct page loads and on in-page anchor jumps, but it feels sluggish during client-side navigation between routes — the page "drags" to the new scroll position instead of snapping. To avoid this, the client router flips `scroll-behavior` to `auto` during the transition and restores it after, but only if the `<html>` element opts in via the `data-scroll-behavior` attribute.

When the attribute is missing, the router has no signal that the user opted into smooth scrolling, so it leaves the CSS rule alone. The result is a jerky transition. The warning is a hint, not a hard error: the page still works, but the experience is worse than the user expects.

## Reproducing the error

```css filename="app/globals.css"
html {
  scroll-behavior: smooth;
}
```

```tsx filename="app/layout.tsx"
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

The CSS sets smooth scroll, but the layout does not declare the data attribute.

## Error output

## How to fix it

1. Add `data-scroll-behavior="smooth"` to the `<html>` element in `app/layout.tsx`.
2. Or, remove the `html { scroll-behavior: smooth }` rule from your global stylesheet if you do not want smooth scrolling.

```tsx filename="app/layout.tsx"
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" data-scroll-behavior="smooth">
      <body>{children}</body>
    </html>
  )
}
```

## Diagnosis

Not applicable: this error is detected at runtime/build only, no CLI command.

## Common pitfalls

- The attribute is read on the client, so it must be present in the SSR output of `app/layout.tsx`, not added later with a `useEffect`.
- Some bundlers strip unknown data attributes in production. Swift-Rust's bundler does not — `data-*` is preserved.
- The attribute is also recognized when set to `"auto"` to opt out of the optimization, but the common case is `"smooth"`.
