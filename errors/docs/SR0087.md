# SR0087 — Too many custom routes

The total number of entries across `headers`, `redirects`, and `rewrites` exceeded 1000. The build refuses to start because request matching iterates this list on every request and a list of this size will measurably degrade throughput.

## Why this error occurred

Swift-Rust (like Next.js before it) matches a request URL against the configured `headers`, `redirects`, and `rewrites` arrays in order on every request. The cost is O(n) per request and the framework cannot optimize arbitrary user-defined patterns. To keep the worst-case request budget bounded, the framework caps the combined count at 1000 entries and raises this error at build time.

The cap is enforced even when the routes would never fire, because the framework cannot know which ones are "live" without running them.

## Reproducing the error

```ts filename="swift-rust.config.json"
{
  "redirects": async () => {
    const rows = await db.query("SELECT old_path, new_path FROM redirects")
    return rows.map((r) => ({ source: r.old_path, destination: r.new_path, permanent: true }))
  }
}
```

If the `redirects` table has more than 1000 rows, the build fails.

## Error output

```
error: [SR0087] max custom routes reached: 1247 entries (limit 1000)
  --> swift-rust.config.json:2:14
   |
 2 |   "redirects": ...
   |                ^^^^^
   |   1247 entries would be matched on every request
```

## How to fix it

1. Combine entries into regex patterns where possible.
2. Use a single dynamic route under `app/` for variable segments.
3. Move rarely-used rewrites into a runtime check inside the route handler.

```ts filename="swift-rust.config.json"
{
  "redirects": [
    {
      "source": "/(old|legacy|deprecated)-:path*",
      "destination": "/:path*",
      "permanent": true
    }
  ]
}
```

```tsx filename="app/[...slug]/page.tsx"
export default function CatchAll({ params }: { params: { slug: string[] } }) {
  const path = params.slug.join("/")
  if (LEGACY.has(path)) {
    return redirect(LEGACY.get(path)!)
  }
  return <NotFound />
}
```

## Diagnosis

```bash
bun swift-rust info --routes
RUST_LOG=swift_rust=router=trace bun swift-rust build
```

`--routes` prints the merged count and the top 20 entries by match frequency from the dev log.

## Common pitfalls

- A list of `redirects` and a list of `rewrites` count toward the same budget. Trim both.
- Dynamic routes in `app/` are not counted in this limit; only the explicit arrays in `swift-rust.config.json` are.
- "Catch-all" routes in `app/[...slug]/page.tsx` are often a better fit for legacy URL migrations than thousands of redirect rows.
