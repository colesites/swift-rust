# SR0129 — No production build found

The production server (`bun swift-rust start`) tried to load the build manifest, but the directory is missing. The server has nothing to serve, so it exits with this error.

## Why this error occurred

Swift-Rust's production mode is purely a static-asset + precompiled-bundle server. The `build/` directory (or whatever `build.outDir` points to) must contain the compiled bundles, route manifest, and static assets. If any of those are missing, the server has no way to construct a response.

Typical triggers:

- You ran `bun swift-rust start` in a project that has never been built.
- A CI step runs `bun swift-rust build` but the output is wiped (for example, by `bun run clean` or by a Docker layer that excludes it).
- The previous build failed and the output was never written.
- The server is being launched from a different working directory than the one that was built.

## Reproducing the error

```bash
# project root with swift-rust.config.json but no build output
bun swift-rust start
```

## Error output

```
[ERROR] [SR0129] no production build found
  --> .
    |
    | looked for: ./build/manifest.json
    | not found. did you forget to run `bun swift-rust build`?
    |
    help: run `bun swift-rust build` first, or use `bun swift-rust dev` for development.
```

## How to fix it

1. Run a production build:

```bash
bun swift-rust build
```

2. Confirm the build directory exists and contains a manifest:

```bash
ls build/manifest.json
```

3. Start the server:

```bash
bun swift-rust start
```

4. If you intended to run in development, use the dev command instead:

```bash
bun swift-rust dev
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
ls build/
```

The doctor output prints the build directory and whether the manifest is present. `ls build/` is the fastest sanity check.

## Common pitfalls

- A `bun run clean` script deletes `build/`. Re-run the build after cleaning.
- A Docker `COPY` that excludes `build/` produces a working build step but a broken start step. Copy the build directory in the same stage that runs the build.
- Multiple workspaces in a monorepo each have their own `build/`. Make sure you are running `start` from the right one.
- Some package managers (notably `bun` in older versions) install but do not trigger Swift-Rust's postinstall scripts. A missing build might mean the postinstall did not run.
