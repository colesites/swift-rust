# SR0063 — Invalid `assetPrefix`

`assetPrefix` controls the public URL prefix under which static assets (`_swift-rust/static/*`) are served. Swift-Rust uses it to rewrite the URLs in the HTML and CSS it emits so that assets can be hosted on a CDN or under a subpath. The framework requires the value to be a string.

## Why this error occurred

The config is parsed once at boot. A non-string `assetPrefix` is rejected immediately because the framework cannot reliably concatenate it with the asset path. Common mistakes:

- Setting `assetPrefix: process.env.CDN_URL` when the env var is unset, yielding `undefined`.
- Wrapping the URL in an object: `assetPrefix: { url: 'https://cdn.example.com' }`. The schema does not match.
- Returning a `Promise<string>` from an async function. Swift-Rust reads the value synchronously.
- Copying a Vercel-only config option (`deploymentId` or similar) into the project config and naming it `assetPrefix`.

The string can be either a path (e.g. `'/cdn'`) or a full URL (e.g. `'https://cdn.example.com'`). The framework detects which by checking for the leading `http(s)://`.

## Reproducing the error

```ts
// swift-rust.config.ts
import type { SwiftRustConfig } from "swift-rust";

const config: SwiftRustConfig = {
  assetPrefix: 3000,
};

export default config;
```

## Error output

```
error: [SR0063] invalid assetPrefix: expected string, got number
  --> swift-rust.config.ts:4:3
   |
 4 |   assetPrefix: 3000,
   |   ^^^^^^^^^^^^^^^^^^
   |
   = assetPrefix must be a string beginning with `/` or `http(s)://`.
   = set it to `/` to use the default, or a CDN URL like
     `https://cdn.example.com`.
```

## How to fix it

1. Set `assetPrefix` to a string path or a full URL.

   ```ts
   // swift-rust.config.ts
   import type { SwiftRustConfig } from "swift-rust";

   const config: SwiftRustConfig = {
     assetPrefix: "/",
   };

   export default config;
   ```

2. To host assets on a CDN, set the full URL.

   ```ts
   const config: SwiftRustConfig = {
     assetPrefix: "https://cdn.example.com",
   };

   export default config;
   ```

3. To read the prefix from an env var, fall back to a string default.

   ```ts
   const config: SwiftRustConfig = {
     assetPrefix: process.env.CDN_URL ?? "/",
   };

   export default config;
   ```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info | grep -i assetPrefix

```

## Common pitfalls

- An empty string is treated as the default. Set it to `"/"` to be explicit.
- A trailing slash is normalized. Both `/cdn` and `/cdn/` work.
- The runtime URL of the asset is `${assetPrefix}/_swift-rust/static/${path}`. If you use a CDN, the CDN must be configured to serve from the origin's `/_swift-rust/static/*` path.
- This is the Swift-Rust analogue of Next.js's `invalid-assetprefix`. The fix is identical.
