# SR0092 — Missing Suspense with CSR bailout

A client component called a runtime-only hook (such as `useSearchParams` or `usePathname`) at a level that would force the entire page to opt out of static prerendering. The build requires the call to be wrapped in a `<Suspense>` boundary so the static shell can still be served.

## Why this error occurred

Some client hooks can only resolve at request time — `useSearchParams` reads the URL query string, `usePathname` reads the URL pathname. If a component calls one of these hooks outside a `<Suspense>` boundary, React cannot stream the static shell and render the dynamic subtree in parallel; instead, the whole page has to wait for the client. The build is configured to fail in that case to avoid shipping a blank-flash experience.

The check is structural: the bundler walks the component tree and verifies that the path from the root to the hook call passes through a `<Suspense>` element. If not, this error fires.

## Reproducing the error

```tsx filename="app/search-bar.tsx"
"use client"

import { useSearchParams } from "swift-rust/navigation"

export function SearchBar() {
  const params = useSearchParams()
  return <input defaultValue={params.get("q") ?? ""} />
}
```

```tsx filename="app/page.tsx"
import { SearchBar } from "./search-bar"

export default function Page() {
  return <SearchBar />
}
```

The `SearchBar` calls `useSearchParams` without a `<Suspense>` boundary.

## Error output

```
error: [SR0092] missing Suspense boundary with useSearchParams
  --> app/search-bar.tsx:5:17
   |
 5 |   const params = useSearchParams()
   |                 ^^^^^^^^^^^^^^^^^^ must be inside a <Suspense> boundary
```

## How to fix it

1. Wrap the subtree that calls the hook in `<Suspense>`.
2. Or, push the data lookup up to a server component and pass the resolved value down.

```tsx filename="app/search-bar.tsx"
"use client"

import { useSearchParams } from "swift-rust/navigation"

export function SearchBar() {
  const params = useSearchParams()
  return <input defaultValue={params.get("q") ?? ""} />
}
```

```tsx filename="app/page.tsx"
import { Suspense } from "react"
import { SearchBar } from "./search-bar"

export default function Page() {
  return (
    <Suspense fallback={<input placeholder="Search..." disabled />}>
      <SearchBar />
    </Suspense>
  )
}
```

## Diagnosis

```bash
bun swift-rust doctor --check suspense
RUST_LOG=swift_rust=ssr=trace bun swift-rust build
```

`--check suspense` runs the static analysis pass that flags un-suspended client hooks.

## Common pitfalls

- Wrapping the call site in a custom component does not satisfy the check — the boundary has to be a literal `<Suspense>` element.
- Layouts in `app/` cannot be wrapped in `<Suspense>` (only pages can). Move the dynamic subtree to a child page or to a route segment.
- A `useSearchParams` call inside a server component is a different error (this one is about client-side usage). Server components read `searchParams` from the `Page` props.
