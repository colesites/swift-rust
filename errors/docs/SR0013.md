# SR0013 — <Container> component is deprecated

A root layout in `app/layout.tsx` still wraps its tree in `<Container>`. That component was carried over from very early Swift-Rust templates where it managed hash-based scrolling. The framework now does that itself, so the wrapper is dead weight and emits a deprecation warning on every render.

## Why this error occurred

In the pre-0.1 Swift-Rust alpha, the root layout had to opt into scroll restoration by wrapping its children in `<Container>`. The behavior was moved into the framework's router so it applies to every layout automatically. The component still exists for backwards compatibility, but importing it is a strong signal that a template is stale.

The deprecation is non-fatal. Builds succeed and pages render, but every render logs the warning so it is impossible to miss in CI.

## Reproducing the error

```tsx
import { Container } from "swift-rust/app";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <Container>{children}</Container>
      </body>
    </html>
  );
}
```

## Error output

```
[WARN] [SR0013] <Container> is deprecated: the wrapper does nothing in current Swift-Rust
   --> app/layout.tsx:6:8
   |
 6 |       <Container>{children}</Container>
   |       ^^^^^^^^^^ remove this wrapper; scroll restoration is automatic
   |
  help: render {children} directly. See https://swift-rust.dev/docs/app/layout.
```

## How to fix it

1. Remove the `<Container>` import and wrapper.
2. If you depended on the scroll behavior it provided, set `scrollRestoration: "auto"` (or `"manual"`) under the `routing` key in `swift-rust.config.json`.
3. Re-run the dev server and confirm the warning is gone.

```tsx
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust dev
```

The trace log records every render path that entered the deprecated component, which makes it easy to find any third-party layout preset that is still bundling the wrapper.

## Common pitfalls

- The component is exported from `swift-rust/app`, not `swift-rust/server` — make sure the import path matches what your preset actually has.
- Some starter templates (older versions of `create-swift-rust`) emitted `<Container>` automatically. Re-running the scaffolder with the current CLI drops the wrapper.
- The deprecation will become a hard error in Swift-Rust 0.4.0. Fixing it now is cheaper than carrying it forward.
