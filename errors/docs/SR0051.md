# SR0051 — Google font display strategy is not recommended

The CSS `font-display` descriptor tells the browser how to render text while a custom web font is still loading. Some strategies — `auto`, `block`, and `fallback` — can leave users staring at a blank page for several seconds on a slow connection. Swift-Rust prints this warning when it detects that one of those strategies has been chosen for a Google Font loaded through `swift-rust/font/google`.

## Why this error occurred

`swift-rust/font/google` is a wrapper around the Google Fonts CSS API. It takes your font choice, subset, weight, and `display` option and produces a self-hosted CSS file at build time. The wrapper accepts every legal value of `font-display`, but the linter flags anything other than `swap` or `optional` because the other values degrade perceived performance:

- `auto` — lets the browser decide, which on most engines is `block` (invisible text for up to 3s).
- `block` — invisible text for up to 3s. FOIT.
- `fallback` — invisible text for ~100ms, then falls back to the system font forever, even if the custom font loads. Hidden shift.
- `swap` — fallback is shown immediately, then swapped in once the custom font loads. Visible shift.
- `optional` — fallback is shown for ~100ms; if the custom font has not loaded by then, the fallback is kept for the lifetime of the page. No shift, no FOIT.

`optional` is almost always the right choice for body text; `swap` is appropriate for display headlines where the custom face is part of the brand.

## Reproducing the error

```tsx
import { Inter } from "swift-rust/font/google";

const inter = Inter({
  subsets: ["latin"],
  display: "block",
});

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" className={inter.className}>
      <body>{children}</body>
    </html>
  );
}
```

## How to fix it

1. Switch to `optional` for body fonts.

   ```tsx
   import { Inter } from "swift-rust/font/google";

   const inter = Inter({
     subsets: ["latin"],
     display: "optional",
   });
   ```

2. Use `swap` only for brand-critical display fonts where the custom face must appear even at the cost of a brief reflow.

   ```tsx
   import { Krona_One } from "swift-rust/font/google";

   const krona = Krona_One({
     weight: "400",
     display: "swap",
   });
   ```

## Common pitfalls

- The warning is also raised when you embed a raw `<link href="https://fonts.googleapis.com/…display=auto" />` directly in your layout. Convert those to `swift-rust/font/google` so the display value is controlled in code.
- `display: 'fallback'` was once a sensible default for icon fonts (Material Icons, etc.). For those, use `display: 'block'` and silence the warning by adding an inline comment in the project config — or, better, host the icon font yourself.
- This is a `warning`, not an `error`. The build will succeed. Treat the message as a perf-budget nudge.
- `font-display` only affects the first paint; on the second visit the font is cached, so the strategy is only relevant for cold loads.
