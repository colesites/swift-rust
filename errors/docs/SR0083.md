# SR0083 — Large page data

A page's serialized props payload exceeded the 128 kB warning threshold. Large payloads inflate the initial HTML response, delay hydration, and bloat the in-memory representation of the page on the client.

## Why this error occurred

Swift-Rust streams data to the client through the RSC payload. The size of that payload is measured at build time and at the start of a request. When the measured size exceeds 128 kB, this warning fires.

Large payloads hurt in three measurable ways:

- **Heavier HTML response.** The serialized data is inlined, increasing Time to First Byte and slowing the document load.
- **Slower hydration.** React cannot make the page interactive until the full payload has been parsed.
- **Higher memory footprint.** The client holds the entire payload in memory even if only a fraction is used.

The warning is not fatal — the build still succeeds — but it is a strong signal that the page is doing too much work upfront.

## Reproducing the error

```tsx filename="app/users/page.tsx"
import { db } from "@/lib/db"

export default async function Page() {
  const users = await db.user.findMany()
  return (
    <ul>
      {users.map((u) => (
        <li key={u.id}>
          {u.name} — {u.email} — {u.bio} — {u.avatarUrl} — {u.createdAt.toISOString()}
        </li>
      ))}
    </ul>
  )
}
```

If `users` resolves to thousands of records with long bios, the serialized payload will exceed 128 kB.

## Error output

```
warn: [SR0083] large page data: 412.3 kB for /users, threshold 128 kB
  --> app/users/page.tsx:1:1
   |
   | payload top contributors:
   |   users[*].bio        187.2 kB
   |   users[*].avatarUrl  104.6 kB
   |   users[*].email       62.0 kB
```

## How to fix it

1. Return only the fields the page actually renders.
2. Paginate large lists and fetch additional pages on the client.
3. Move non-critical fields to a follow-up fetch that runs after hydration.

```tsx filename="app/users/page.tsx"
import { db } from "@/lib/db"

export default async function Page() {
  const users = await db.user.findMany({
    select: { id: true, name: true, email: true },
    take: 25,
  })
  return (
    <ul>
      {users.map((u) => (
        <li key={u.id}>{u.name} — {u.email}</li>
      ))}
    </ul>
  )
}
```

### Adjusting the threshold

You can raise the threshold from `swift-rust.config.json` if you have a measured reason:

```json filename="swift-rust.config.json"
{
  "bundling": {
    "largePageDataBytes": 200000
  }
}
```

## Diagnosis

```bash
bun swift-rust info --bundle-size /users
RUST_LOG=swift_rust=bundling=trace bun swift-rust build
```

`--bundle-size` prints the top contributors to the payload.

## Common pitfalls

- Selecting `*` from a SQL table when the model has long text columns.
- Returning full audit log entries when only a count is needed.
- Embedding base64-encoded images in props instead of using `<Image>` from `swift-rust/image`.
- This warning applies to the RSC payload. The HTML markup itself is not counted.
