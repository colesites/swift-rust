# SR0030 — Global CSS imported outside the root layout

A file other than the root layout (`app/layout.tsx`) imported a global stylesheet. Swift-Rust only guarantees the load order of CSS that comes from the root layout; importing global CSS anywhere else produces undefined cascade behavior, so the build warns and continues.

## Why this error occurred

Global CSS rules apply to every element on the page. Their load order matters: a later rule overrides an earlier one. The root layout is the only place Swift-Rust can guarantee is loaded exactly once, in a stable position, for every page. Importing a global stylesheet from a component means the order depends on whether the component is rendered, when it is rendered, and whether its parent has already loaded a conflicting rule. To avoid surprises, the build warns whenever it sees a non-module CSS import outside the root layout.

The warning is non-fatal because the cascade happens to work in many simple cases. But "happens to work" is a poor basis for a design system.

## Reproducing the error

```css
body {
  font-family: system-ui;
  background: #fafafa;
}
```

```tsx
import "./styles.css";

export default function Card() {
  return <div className="card">hello</div>;
}
```

## Error output

```
[WARN] [SR0030] global CSS imported outside the root layout
   --> app/components/card.tsx:1:1
   |
 1 | import "./styles.css";
   | ^^^^^^^^^^^^^^^^^^^^^^ move global CSS to app/layout.tsx, or rename the file to *.module.css
   |
  help: global stylesheets are only guaranteed to load in the correct order from the root layout.
```

## How to fix it

1. If the styles are genuinely global, move the import to `app/layout.tsx`.
2. If the styles are component-scoped, rename the file to `card.module.css` and import it as a CSS Module.
3. Split the file so tokens (`:root`, `body`, font face) live in a global stylesheet imported once, and component styles live as modules.

```tsx
import "./card.module.css";

export default function Card() {
  return <div className={styles.card}>hello</div>;
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log lists every global CSS import it found, paired with the importing file. The list makes it easy to refactor a project to the root layout in one pass.

## Common pitfalls

- A `*.css` import in a `layout.tsx` that is **not** the root (e.g. `app/dashboard/layout.tsx`) still triggers the warning. Only the root layout is trusted.
- A `*.module.css` import is fine anywhere, because CSS Modules scope the class names and the load order does not matter.
- A Sass file with nested rules that escape the local scope (e.g. `@import "global";`) is treated as global. Move those imports to the root layout.
- This warning does not block builds, but `SR0019` does if the CSS pipeline is also disabled by a legacy plugin.
