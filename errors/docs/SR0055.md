# SR0055 — `href` interpolation failed

Swift-Rust's router builds a final URL from the route pattern and the values you pass for each dynamic segment. When the `href` is an object of the form `{ pathname: '/blog/[post]/[comment]', query: { post: 'post-1' } }`, the router looks for a value for `comment` and aborts with this error if it cannot find one.

## Why this error occurred

Dynamic routes are defined with bracket syntax in the file system: `app/blog/[post]/[comment]/page.tsx`. The route table knows that this pattern needs a `post` and a `comment`. When you build a link to that route, you must supply both — even if the value is empty.

The error fires when:

- The `query` object in the `href` is missing a key.
- A helper function builds the `href` object and one of the upstream callers forgot to spread the params.
- The link is constructed from a `useSearchParams` call that does not include the catch-all param.
- An `as` string is used as a fallback for an `href` object, and the two are out of sync.

This is a runtime error: it only fires when the link is clicked, not when it is rendered. The reason is that Swift-Rust avoids paying the cost of static route analysis on every render.

## Reproducing the error

```tsx
import Link from "swift-rust/link";

export default function BlogIndex() {
  return (
    <Link
      href={{
        pathname: "/blog/[post]/[comment]",
        query: { post: "post-1" },
      }}
    >
      Open comment
    </Link>
  );
}
```

## Error output

```
[ERROR] [SR0055] href interpolation failed
  --> app/blog/page.tsx:5:3
   |
   = route `/blog/[post]/[comment]` requires `comment`.
     received query: { post: "post-1" }
   = the link will not navigate. fix the query object to include every
     dynamic segment of the route.
```

## How to fix it

1. Provide a value for every dynamic segment.

   ```tsx
   import Link from "swift-rust/link";

   export default function BlogIndex() {
     return (
       <Link
         href={{
           pathname: "/blog/[post]/[comment]",
           query: { post: "post-1", comment: "comment-1" },
         }}
       >
         Open comment
       </Link>
     );
   }
   ```

2. For catch-all routes, provide an array (or an empty array to match the parent).

   ```tsx
   <Link
     href={{
       pathname: "/docs/[...parts]",
       query: { parts: ["getting-started", "install"] },
     }}
   >
     Read the install guide
   </Link>
   ```

3. When the link lives inside the dynamic route itself, derive the params from the current URL.

   ```tsx
   import Link from "swift-rust/link";
   import { useParams } from "swift-rust/navigation";

   export default function CommentLink() {
     const { post, comment } = useParams<{ post: string; comment: string }>();

     return (
       <Link
         href={{
           pathname: "/blog/[post]/[comment]",
           query: { post, comment },
         }}
       >
         Permalink
       </Link>
     );
   }
   ```

## Common pitfalls

- The error fires on click, not on render. If you have many `Link` instances and only one is broken, the warning does not tell you which one — look at the route table printed by `bun swift-rust route:list` to find the route, then grep your codebase for it.
- The `[...catchAll]` syntax requires an array. A bare string is rejected even if it has the right shape.
- Optional catch-all (`[[...catchAll]]`) accepts an empty array as "no value". Regular catch-all does not.
- This error is intentionally a runtime error to avoid scanning every render for missing params. A stricter linter is on the roadmap.
