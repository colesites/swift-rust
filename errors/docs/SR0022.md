# SR0022 — Cannot output to /public

The configured build output directory is `public/`, but Swift-Rust uses that folder to serve static assets. Writing the build output there would clobber user assets and create a cycle, so the build refuses to run.

## Why this error occurred

`public/` is the only directory Swift-Rust serves verbatim at runtime. Anything in it is exposed at the URL path matching its filename. If the build output lived in the same folder, every cached chunk, sourcemap, and intermediate file would become publicly addressable — and worse, the framework's own static-asset pipeline would serve stale or partial output as if it were an asset.

The check is the same whether the path was set via `distDir` or chosen as a static-export target.

## Reproducing the error

```json
{
  "name": "site",
  "distDir": "public"
}
```

## Error output

```
[ERROR] [SR0022] cannot output to /public: that directory is reserved for static assets
   --> swift-rust.config.json:2:12
   |
 2 |   "distDir": "public"
   |              ^^^^^^^^ pick a different output directory
   |
  help: remove the field to use the default (.swift-rust), or set distDir to e.g. "out".
```

## How to fix it

1. Remove the `distDir` field and let the default (`.swift-rust/`) take over.
2. If you want a self-contained output, point `distDir` at any folder that is not `public/`, `static/`, or a source folder.
3. Re-run the build and confirm the error is gone.

```json
{
  "name": "site"
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log records the resolved absolute path of `distDir`, so you can confirm what the framework actually saw (relative paths are resolved against the project root).

## Common pitfalls

- The check is on the **resolved** path. A symlink that points into `public/` will also be rejected.
- A static export that targets a subdirectory of `public/` (e.g. `public/build`) is allowed, but the contents will be served as static assets and exposed at build URLs.
- The same check covers `static/` — see `SR0023` for the static-asset equivalent.
