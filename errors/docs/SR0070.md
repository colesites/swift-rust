# SR0070 — Invalid multi-match

A multi-match parameter in path-to-regexp (the syntax Swift-Rust uses for custom routes and rewrites) is a placeholder followed by `*`, e.g. `:path*` or `[...slug]`. It matches zero or more path segments. When the parameter is referenced in the `destination`, the `*` must be repeated so that the resolver knows to expand every captured segment.

## Why this error occurred

The resolver does not know whether `:path` in the destination means "the first captured segment" or "every captured segment". The convention is unambiguous:

- `:path` in the destination refers to a single segment. If the multi-match captured multiple segments, only the first is forwarded.
- `:path*` in the destination refers to all captured segments, joined by `/`.

Without the `*`, the build cannot decide which behaviour is intended, and the route is rejected.

The most common sources of the mistake:

- A rewrite forwards a catch-all path to a single-segment target, dropping everything after the first slash.
- A destination template is copy-pasted from a different rewrite that used a non-multi-match placeholder.
- A team member forgets that path-to-regexp distinguishes the two cases by the trailing `*`, not by context.

## Reproducing the error

```json
{
  "rewrites": [
    {
      "source": "/:path*",
      "destination": "/another/:path"
    }
  ]
}
```

## Error output

```
error: [SR0070] invalid multi-match
  --> swift-rust.config.json:3:5
   |
 3 |     {
 4 |       "source": "/:path*",
 5 |       "destination": "/another/:path",
 6 |     }
   |                  ^^^^^^^^^
   |
   = the multi-match placeholder `:path*` is referenced in the
     destination without the trailing `*`. add `*` to repeat every
     captured segment, e.g. `/another/:path*`.
```

## How to fix it

1. Add `*` after the placeholder in the destination.

   ```json
   {
     "rewrites": [
       {
         "source": "/:path*",
         "destination": "/another/:path*"
       }
     ]
   }
   ```

2. If you want only the first segment, change the source to a single-match placeholder.

   ```json
   {
     "rewrites": [
       {
         "source": "/:path",
         "destination": "/another/:path"
       }
     ]
   }
   ```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust route:list --rewrites
RUST_LOG=swift_rust=routing=trace bun swift-rust dev
```

## Common pitfalls

- The convention is the same in rewrites, redirects, headers, and proxy `matcher` configs.
- The trailing `*` is required even when the source is an optional catch-all (`[[...slug]]`). The destination is treated as a multi-match unless explicitly scoped.
- This is the Swift-Rust analogue of Next.js's `invalid-multi-match`. The fix is identical.
