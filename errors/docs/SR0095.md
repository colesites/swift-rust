# SR0095 — Nested reserved page

A reserved route file (such as `error.tsx`, `not-found.tsx`, `loading.tsx`, or `layout.tsx`) was placed inside a directory that is not a valid route segment. The file is ignored by the router, but the build flags it so the mistake is caught in CI rather than at runtime.

## Why this error occurred

In Swift-Rust's `app/` directory, every folder is a route segment, and a few filenames are reserved to provide a piece of behavior for the segment they live in: `layout.tsx` defines the segment's chrome, `error.tsx` defines the segment's error boundary, `loading.tsx` defines the segment's loading state, `not-found.tsx` defines the segment's 404 page, and `default.tsx` is the parallel-route fallback.

These files only have meaning when they live in a folder that the router already considers a segment. If you put `error.tsx` inside a subdirectory that is not itself a segment (for example, a `components/` folder, or a folder that is excluded by route groups), the file is silently ignored — there is no warning at runtime, the segment just doesn't have an error boundary.

The build performs a static check and refuses to leave the file in place, because an ignored reserved file almost always indicates a mistake.

## Reproducing the error

```text
app/
├── layout.tsx
├── page.tsx
└── components/
    └── error.tsx
```

`components/` is not a route segment (it is a private folder — folders prefixed with `_` are private, and `components/` is treated as a private folder by convention in many codebases). The `error.tsx` inside it has no effect, and the build complains.

## Error output

```
error: [SR0095] nested reserved page: error.tsx inside a non-segment folder
  --> app/components/error.tsx:1:1
   |
 1 | export default function Error() { ... }
   |                  ^^^^^^ file is ignored; move it to a route segment
```

## How to fix it

1. Move the file to the route segment where you want the behavior to apply.
2. If you want a route-wide error boundary, put it directly in `app/`: `app/error.tsx`.
3. If you want a per-segment boundary, put it in the segment folder: `app/dashboard/error.tsx`.
4. If the file is not actually needed, delete it.

```text
app/
├── layout.tsx
├── page.tsx
├── dashboard/
│   ├── page.tsx
│   └── error.tsx        ← segment-level error boundary
└── error.tsx            ← app-wide fallback
```

## Diagnosis

```bash
bun swift-rust doctor --check routes
bun swift-rust info --routes
```

`--routes` prints the full route table including which reserved files apply to each segment.

## Common pitfalls

- A folder that starts with `_` (e.g. `_components/`) is private and will not become a route. Reserved files inside it are ignored.
- Route groups (folders wrapped in `()` like `(marketing)/`) **are** route segments for the purposes of `layout.tsx` and `error.tsx`. The reserved file is valid inside a group.
- If you have a `not-found.tsx` at `app/not-found.tsx`, it is the global 404. Nesting it in a segment makes it segment-local.
