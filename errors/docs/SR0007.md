# SR0007 — Font error

The font service fetches the requested family from Google Fonts (or a local file), subsets it to the requested glyphs, and serves the WOFF2 bundle. Failures at any step surface here.

## Why this error occurred

The font service is invoked at build time for any font declared in the root layout. The build downloads the family, generates the WOFF2 subsets, and emits a CSS file that the browser uses to load the right subset. A failure at any of those steps means the build cannot complete, because every page in the app expects the font to be available.

The most common cause is a family name that does not exist on Google Fonts (a typo, a deprecated family, or a name with the wrong case). Local fonts fail for the same reason — a missing file or an unsupported format.

## Reproducing the error

```tsx filename="app/layout.tsx"
import { MyCustomFont } from "swift-rust/font/google";

const custom = MyCustomFont({ subsets: ["latin"] });

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" className={custom.className}>
      <body>{children}</body>
    </html>
  );
}
```

`MyCustomFont` is not a real Google Fonts family. The build fails on the first request.

```tsx filename="app/layout.tsx"
import localFont from "swift-rust/font/local";

const inter = localFont({ src: "./fonts/Missing.woff2" });
```

The file `./fonts/Missing.woff2` does not exist.

## How to fix it

1. Verify the family name against the [Google Fonts catalog](https://fonts.google.com). The name is case-sensitive.
2. For local fonts, confirm the file exists at the path you passed to `src` and is a valid TTF, OTF, or WOFF2 file.
3. Re-run the build to confirm the font is reachable.

```tsx filename="app/layout.tsx"
import { Inter } from "swift-rust/font/google";

const inter = Inter({ subsets: ["latin"], display: "swap" });
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info --fonts
RUST_LOG=swift_rust::font=trace bun swift-rust build
```

`info --fonts` prints every font the layout declared, with the resolved URL and the cache state. The trace log includes the HTTP status of the Google Fonts request and the path that was attempted for local files.

## Common pitfalls

- A family that is `display=swap` and `display=optional` are different things. `display=optional` makes the font invisible until it loads, which is correct for performance but trips some up.
- Subsets are not auto-detected. If you load a CJK family but only specify `subsets: ["latin"]`, the CJK glyphs will be missing.
- Google Fonts has a rate limit. CI builds that pull dozens of families in parallel may hit it; the framework retries with backoff, but a hard limit surfaces as a 429.
- The font cache lives at `~/.cache/swift-rust/fonts/`. Clearing it forces a re-fetch.
