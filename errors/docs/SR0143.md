# SR0143 ‚Äî Parallel route slot missing default.tsx

A parallel route slot directory under `app/` (for example `app/@team/`) does not have a `default.tsx` file. The build cannot infer a fallback for the slot when navigating to a route segment where the slot has no matching page, so the build fails with this error.

## Why this error occurred

Parallel route slots are directories prefixed with `@` that the router merges into the layout as a prop (`children`, `modal`, `team`, ...). For each route segment, the router looks for a `page.tsx` in the slot. If none is found, the slot needs a `default.tsx` to fall back to. Without it, the router cannot produce a complete tree for the route, and the build aborts.

The error fires at build time, naming the slot directory that is missing the fallback. The same error fires for every slot that has the problem.

## Reproducing the error

```
app/
‚îú‚îÄ‚îÄ layout.tsx
‚îú‚îÄ‚îÄ page.tsx
‚îú‚îÄ‚îÄ @team/
‚îÇ   ‚îî‚îÄ‚îÄ settings/
‚îÇ       ‚îî‚îÄ‚îÄ page.tsx
‚îî‚îÄ‚îÄ @analytics/
    ‚îî‚îÄ‚îÄ page.tsx
```

Visiting `/settings` matches `app/@team/settings/page.tsx` for the `team` slot, but `app/@analytics/page.tsx` does not match. The router needs `app/@analytics/default.tsx` to know what to render for the `analytics` slot on `/settings`.

## Error output

```
[ERROR] [SR0143] parallel route slot missing default.tsx
  --> app/@analytics
    |
    | slot `analytics` has no `default.tsx`.
    | add a fallback file at `app/@analytics/default.tsx` so the router can render
    | this slot on route segments that do not have a matching `page.tsx`.
    |
    help: add `default.tsx` (returning `null` is fine) or move all routes into a single layout.
```

## How to fix it

1. Add a `default.tsx` to the slot directory. The simplest implementation returns `null`:

```tsx
// app/@analytics/default.tsx
export default function Default() {
  return null;
}
```

2. If you want the legacy 404 behaviour, call `notFound()`:

```tsx
// app/@analytics/default.tsx
import { notFound } from "swift-rust/navigation";

export default function Default() {
  notFound();
}
```

3. Repeat for every slot that the build flagged. Each missing slot is a separate error.

The full corrected structure:

```
app/
‚îú‚îÄ‚îÄ layout.tsx
‚îú‚îÄ‚îÄ page.tsx
‚îú‚îÄ‚îÄ @team/
‚îÇ   ‚îú‚îÄ‚îÄ default.tsx
‚îÇ   ‚îî‚îÄ‚îÄ settings/
‚îÇ       ‚îî‚îÄ‚îÄ page.tsx
‚îî‚îÄ‚îÄ @analytics/
    ‚îú‚îÄ‚îÄ default.tsx
    ‚îî‚îÄ‚îÄ page.tsx
```

## Diagnosis

```bash
bun swift-rust doctor
RUST_LOG=swift_rust=router=trace bun swift-rust build
```

`router=trace` prints every slot the router discovered and whether it has a `default.tsx` and a `page.tsx`. The build also fails fast, so the slot name is in the error message.

## Common pitfalls

- A `default.tsx` is only required for *named* slots (those starting with `@`). The `children` slot is implicit and does not need one ‚Äî though adding one is allowed and helps the user experience.
- A `default.tsx` can render anything: `null`, a loading state, or a fallback page. The framework only requires that the file exists.
- An empty `default.tsx` that exports a non-component default (for example, a string) is a separate error (SR0128 ‚Äî prerender error).
- The error fires for every segment that needs a fallback. Fix them all in one pass to avoid multiple rebuilds.
