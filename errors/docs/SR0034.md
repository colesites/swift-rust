# SR0034 â€” Entire page deopted into client-side rendering

A component on a prerenderable page called `useSearchParams()` without a parent `<Suspense>` boundary. Because search params are per-request, the entire tree up to the closest boundary (or, in this case, the whole page) is forced to render on the client. The build warns so the regression is visible in CI.

## Why this error occurred

Prerendered pages are generated once at build time and served from a CDN. Reading search params at the top of the page means the rendered HTML depends on a per-request value, so the page cannot be prerendered. Swift-Rust handles this by "deopting" the affected subtree to client-side rendering: the shell ships as a static HTML placeholder and the actual content is rendered in the browser once the params are known.

That fallback is slow and uses more JavaScript than necessary. The build warns so the regression is visible, and the recommended fix is to fence off the search-params read with a `<Suspense>` boundary so the rest of the page can still be prerendered.

## Reproducing the error

```tsx
"use client";
import { useSearchParams } from "swift-rust/navigation";

export default function SearchBar() {
  const params = useSearchParams();
  return <p>search = {params.get("q") ?? "all"}</p>;
}
```

```tsx
import SearchBar from "./search-bar";

export default function Page() {
  return (
    <>
      <nav>
        <SearchBar />
      </nav>
      <h1>Dashboard</h1>
    </>
  );
}
```

## How to fix it

1. Move the `useSearchParams` call into a small leaf component (e.g. `search-bar.tsx`).
2. Wrap that leaf in `<Suspense>` from `app/dashboard/page.tsx` and provide a fallback UI.
3. Confirm the page is now prerenderable (the trace log will report it as static again).

```tsx
import { Suspense } from "react";
import SearchBar from "./search-bar";

function SearchBarFallback() {
  return <p>searchingâ€¦</p>;
}

export default function Page() {
  return (
    <>
      <nav>
        <Suspense fallback={<SearchBarFallback />}>
          <SearchBar />
        </Suspense>
      </nav>
      <h1>Dashboard</h1>
    </>
  );
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log shows which page was deopted and the offending component. Run with `--debug-prerender` to capture a stack frame at the `useSearchParams` call site.

## Common pitfalls

- A `<Suspense>` boundary that lives **below** the offending component does not help; the boundary must be above the consumer.
- `useSearchParams` is a client hook. It can only live in a `"use client"` file.
- Search params in a server component are read from the `searchParams` prop â€” that is a different code path and does not trigger this warning.
- The fallback UI is rendered in the initial HTML and replaced once the params resolve. Make sure it conveys the same intent as the final content.
