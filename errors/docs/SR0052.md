# SR0052 — Google font preconnect hint missing

`preconnect` is a resource hint that tells the browser to open a TCP/TLS connection to a third-party origin before it actually needs to fetch anything from it. For Google Fonts the relevant origin is `https://fonts.gstatic.com` — that is where the actual `.woff2` files live, even though the CSS comes from `https://fonts.googleapis.com`. Adding the preconnect shaves 100–300 ms off the first font request on a cold load.

## Why this error occurred

Swift-Rust's font wrapper (`swift-rust/font/google`) adds the preconnect for you as part of the `<link rel="stylesheet">` it injects into `<head>`. The wrapper also adds a `crossOrigin="anonymous"` attribute, which is required for the browser to actually open the connection (preconnect to a CORS-restricted origin is silently ignored).

This `info` message appears when:

- The font is loaded by hand from a third-party script or a raw `<link>` tag in `app/layout.tsx` that omits the preconnect.
- A pre-`app/` style import is still in the codebase and was never migrated.
- The Google Font is loaded inside a third-party widget (e.g. an embedded form) that does its own CSS.

The warning is purely advisory — the font still loads, just a little slower.

## Reproducing the error

```tsx
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <head>
        <link
          rel="stylesheet"
          href="https://fonts.googleapis.com/css2?family=Inter&display=swap"
        />
      </head>
      <body>{children}</body>
    </html>
  );
}
```

## How to fix it

1. Prefer the framework font wrapper. It adds the preconnect and inlines the resolved font CSS.

   ```tsx
   import { Inter } from "swift-rust/font/google";

   const inter = Inter({ subsets: ["latin"] });

   export default function Layout({ children }: { children: React.ReactNode }) {
     return (
       <html lang="en" className={inter.className}>
         <body>{children}</body>
       </html>
     );
   }
   ```

2. If you must use a raw link, add the preconnect yourself. Note the `crossOrigin` attribute — without it the preconnect is dropped on most browsers.

   ```tsx
   <head>
     <link rel="preconnect" href="https://fonts.gstatic.com" crossOrigin="anonymous" />
     <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter&display=swap" />
   </head>
   ```

3. As a fallback for browsers that do not support `preconnect`, a `dns-prefetch` can be added — Swift-Rust emits this automatically when you use the font wrapper.

## Diagnosis

```bash
bun swift-rust doctor
RUST_LOG=swift_rust=fonts=debug bun swift-rust build
```

## Common pitfalls

- `preconnect` to `https://fonts.googleapis.com` is wrong — that origin serves only the CSS. The heavy transfer is the `.woff2` files at `https://fonts.gstatic.com`.
- `crossOrigin="anonymous"` is required. Without it the preconnect is effectively a no-op because the font response is fetched with CORS mode.
- Swift-Rust emits a `<link rel="preconnect">` only when the font is used on a route that is actually rendered. Fonts imported but never referenced are dead code and do not get the hint.
- This used to be a warning in older versions of Next.js. In Swift-Rust it is `info`-level to keep the build output clean for teams that explicitly opt out of the framework font wrapper.
