# SR0112 — Page-only custom font

A custom font was added in a way that scopes it to a single page or that bypasses the framework's font optimization. The build warns because the cost of a font (a network round-trip and a layout shift while it loads) is paid every time the user lands on a page that does not have the font preloaded, even if the rest of the site uses it.

## Why this error occurred

Web fonts are notoriously expensive: the bytes are large, the FOUT is visible, and a wrong preload strategy costs the user a real second of perceived performance. The `swift-rust/font` API solves this by:

- Inlining the font's `@font-face` declaration in the document head with a hashed filename so the browser can match it deterministically.
- Preloading the most important weight used above the fold.
- Subsetting the font to the glyphs your app actually uses.
- Deduplicating across pages so the same weight is loaded once.

None of this works if the font is declared in a page component, because the framework only knows about fonts that are declared in the root layout (or in a layout that is reachable from the root). A `<link>` tag in a page's head bypasses the API entirely and produces a `<link rel="stylesheet">` that the framework's [SR0107](SR0107.md) check also flags.

## Reproducing the error

```tsx filename="app/about/page.tsx"
import { Inter } from "swift-rust/font/google"

const inter = Inter({ subsets: ["latin"] })

export default function About() {
  return (
    <main className={inter.className}>
      <h1>About</h1>
    </main>
  )
}
```

The font is declared inside a page, not the root layout.

## Error output

```
warn: [SR0112] page-only custom font: `inter` declared in app/about/page.tsx
  --> app/about/page.tsx:3:14
   |
 3 | const inter = Inter({ subsets: ["latin"] })
   |              ^^^^ declare fonts in the root layout so the framework can preload and deduplicate them
```

## How to fix it

1. **Recommended.** Move the font declaration to the root layout and import it from there.

```tsx filename="app/layout.tsx"
import type { ReactNode } from "react"
import { Inter } from "swift-rust/font"

const inter = Inter({ subsets: ["latin"] })

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="en" className={inter.className}>
      <body>{children}</body>
    </html>
  )
}
```

2. **Self-hosted font.** Use `localFont` from the same API.

```tsx filename="app/layout.tsx"
import type { ReactNode } from "react"
import localFont from "swift-rust/font/local"

const inter = localFont({
  src: "./fonts/Inter-Variable.woff2",
  display: "swap",
})

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="en" className={inter.className}>
      <body>{children}</body>
    </html>
  )
}
```

3. **Per-segment override.** If a route really does need a different font (e.g. a marketing page with a display face), declare the font in that segment's layout. It is still hoisted to the head, but the framework knows it is segment-specific and will preload it only when the user lands on that segment.

```tsx filename="app/marketing/layout.tsx"
import { Playfair_Display } from "swift-rust/font"

const display = Playfair_Display({ subsets: ["latin"], display: "swap" })

export default function MarketingLayout({ children }: { children: React.ReactNode }) {
  return <div className={display.className}>{children}</div>
}
```

## Diagnosis

```bash
bun swift-rust doctor --check fonts
bun swift-rust info --fonts
```

`--fonts` lists every font declared in the project, the layout that declared it, and whether it is preloaded.

## Common pitfalls

- The `swift-rust/font` API works at build time, so the font must be reachable at build time. Network restrictions in CI can break the build if a Google Font is requested from a region that is blocked.
- Local fonts are bundled into the output; a 2 MB woff2 is a 2 MB download for every user. Subset and convert to woff2 before committing.
- The `display: "swap"` value is the right default for body text. Use `display: "optional"` for LCP-critical text where the user will not see the swap.
- The warning is suppressed if the page is the only route in the app. A single-page project has no need for a global font.
