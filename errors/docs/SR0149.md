# SR0149 — Expected segment not rendered during instant UI validation

A segment of the route tree is supposed to be available without a server roundtrip (an "instant" segment), but the prerendered HTML does not contain it. The framework's instant-UI validator catches the divergence and fails the build.

## Why this error occurred

Swift-Rust can precompute certain segments of the route tree at build time and ship them with the initial document. When the user navigates, the next route's HTML is already on the page (or in a cache close enough to feel instant), and no server roundtrip is needed.

For this to work, every segment that participates in the instant guarantee must appear in the SSR HTML. The validator walks the tree and confirms each segment is present. If a segment is missing — usually because a parent layout omitted it, or a client component returned a different tree on the server vs. the client — the build fails.

The most common cause is a parent layout that conditionally renders a parallel slot. If the condition is `false` during SSR but `true` after hydration, the slot disappears from the instant tree.

## Reproducing the error

```tsx
// app/dashboard/layout.tsx
export default function Layout({
  children,
  modal,
}: {
  children: React.ReactNode;
  modal: React.ReactNode;
}) {
  return (
    <main>
      {children}
      {showModal ? modal : null}
    </main>
  );
}
```

If `showModal` is `false` during SSR, the `modal` slot's content is missing from the HTML, and the instant validator flags it.

## Error output

```
[ERROR] [SR0149] expected segment not rendered during instant UI validation
  --> app/dashboard/layout.tsx:9:7
    |
  9 |       {showModal ? modal : null}
    |       ^^^^^^^^^^^^^^^^^^^^^^^^^^
    | slot `modal` is part of the instant guarantee but is omitted during SSR.
    |
    help: render the slot unconditionally, or opt the segment out with `export const unstable_instant = false`.
```

## How to fix it

1. Render the slot unconditionally:

```tsx
export default function Layout({
  children,
  modal,
}: {
  children: React.ReactNode;
  modal: React.ReactNode;
}) {
  return (
    <main>
      {children}
      {modal}
    </main>
  );
}
```

2. If the slot is intentionally not always rendered (e.g. a modal that only appears on certain routes), opt it out of the instant guarantee:

```tsx
// app/dashboard/@modal/page.tsx
export const unstable_instant = false;

export default function ModalPage() {
  return null;
}
```

3. If a client component wraps a server segment, make sure it renders its children during SSR (no `ssr: false`, no `typeof window` guard).

## Diagnosis

```bash
bun swift-rust doctor
RUST_LOG=swift_rust=router=trace bun swift-rust build
```

`router=trace` prints the instant-UI decision for every segment and lists the ones that failed validation.

## Common pitfalls

- The opt-out (`unstable_instant = false`) is per-page, not per-layout. Set it on the page that owns the segment.
- A `default.tsx` that returns `null` on the server and a real tree on the client also fails the check. Make `default.tsx` deterministic.
- A `dynamic(() => import(...), { ssr: false })` boundary removes every segment below it from the instant tree. Use it sparingly.
- The instant-UI guarantee is best-effort. Some routes legitimately cannot participate (e.g. they depend on a per-request secret). The opt-out is the right tool for those.
