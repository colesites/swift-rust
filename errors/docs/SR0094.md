# SR0094 — Multiple dev tabs warning

You opened the Swift-Rust dev server in several tabs of the same browser. Each tab opens two long-lived connections (one for HMR, one for the page keep-alive), and the browser caps the total number of concurrent connections to a single host. This informational message suggests you may be near the cap.

## Why this error occurred

Modern browsers enforce a per-host connection limit (6 in Chrome, 6 in Firefox, 6 in Safari) per [RFC 6202](https://tools.ietf.org/html/rfc6202#section-5.1). Swift-Rust's dev server holds two persistent connections per open page to deliver hot updates without polling, which is fast for the user but expensive in connection count.

When you have more than a handful of tabs open to the same dev port, you start hitting the limit. Symptoms include:

- Hot updates not arriving until you click the tab.
- "Loading…" that never resolves.
- Static asset requests stalled behind the HMR connection.

This is purely a dev-time concern. Production builds do not use persistent connections and the limit does not apply.

## Reproducing the error

Open the same Swift-Rust dev URL in eight or more tabs of the same browser. The warning is printed in the dev server's terminal output the first time the connection cap is approached.

## Error output

```
info: [SR0094] many dev tabs open: 8 connections held, browser limit typically 6
       close unused tabs or raise the per-host limit if your network allows
```

## How to fix it

1. Close tabs you are not actively using.
2. If you are on Firefox, raise the limit by visiting `about:config` and increasing `network.http.max-persistent-connections-per-server`.
3. If you are testing several apps at once, run them on different ports so the limit is per-app, not global.

```bash
bun swift-rust dev --port 3001
```

## Diagnosis

Not applicable: this error is detected at runtime/build only, no CLI command.

## Common pitfalls

- The warning is informational only and does not block the build. The dev server continues to work, just with degraded HMR.
- The browser limit is per host, not per tab. Eight tabs against `localhost:3000` and two tabs against `localhost:3001` are independent.
- Some browser extensions open "background" tabs that hold connections silently. Disable extensions if the warning persists with a small number of visible tabs.
