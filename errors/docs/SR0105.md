# SR0105 — beforeInteractive script outside document

A `<Script>` component with the `beforeInteractive` loading strategy was rendered somewhere other than the root layout (`app/layout.tsx`). The build refuses to compile this because `beforeInteractive` is a document-level strategy that only makes sense in the place where the document is defined.

## Why this error occurred

`beforeInteractive` is a loading strategy that runs a script before any of the page's React code, before hydration, and as part of the initial HTML payload. It is intended for things that the rest of the app depends on — analytics that need to be initialized before the first render, polyfills that must run before user code, feature flags that flip a class on `<html>`.

To deliver on that promise, the framework inlines the script tag into the document head, which is only available in the root layout. If a `beforeInteractive` script is placed in a deeper layout, the framework would have to retroactively insert a `<script>` into the document head, which is brittle and breaks streaming.

The error is a guardrail: instead of silently doing the wrong thing, the build asks you to move the script or pick a different strategy.

## Reproducing the error

```tsx filename="app/about/page.tsx"
import Script from "swift-rust/script"

export default function About() {
  return (
    <main>
      <Script src="https://example.com/analytics.js" strategy="beforeInteractive" />
      <h1>About</h1>
    </main>
  )
}
```

`/about` is not the root layout; the `beforeInteractive` strategy is invalid here.

## Error output

```
error: [SR0105] beforeInteractive script outside document
  --> app/about/page.tsx:5:6
   |
 5 |       <Script src="https://example.com/analytics.js" strategy="beforeInteractive" />
   |       ^^^^^^ beforeInteractive is only valid in app/layout.tsx
```

## How to fix it

1. Move the script into the root layout (`app/layout.tsx`).
2. Or pick a different strategy that is valid at any level of the tree.

```tsx filename="app/layout.tsx"
import type { ReactNode } from "react"
import Script from "swift-rust/script"

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="en">
      <body>
        {children}
        <Script src="https://example.com/analytics.js" strategy="beforeInteractive" />
      </body>
    </html>
  )
}
```

```tsx filename="app/about/page.tsx"
import Script from "swift-rust/script"

export default function About() {
  return (
    <main>
      <Script src="https://example.com/analytics.js" strategy="afterInteractive" />
      <h1>About</h1>
    </main>
  )
}
```

## Diagnosis

Not applicable: this error is detected at runtime/build only, no CLI command.

## Common pitfalls

- `beforeInteractive` is one of three strategies. `afterInteractive` and `lazyOnload` are valid at any level of the tree and are usually what you want for page-specific code.
- The check is on the file, not on the URL. A `<Script>` rendered from a component imported into a page is still flagged.
- In Swift-Rust, `<Script>` is a thin wrapper around a standard `<script>` tag, not a custom element. The `strategy` prop is compile-time advice to the bundler; it does not exist at runtime.
