# SR0108 — Duplicate head

The rendered document contained more than one `<head>` element. The framework injects exactly one `<head>` per response, and any additional one you render is rejected at build time.

## Why this error occurred

In Swift-Rust, the `<head>` element is owned by the framework. The framework collects the `metadata` and `viewport` exports from your root layout, merges them with per-page metadata, and produces a single `<head>` tag with the correct ordering, deduplicated link tags, and the right charset. Rendering your own `<head>` would either be ignored (and the framework's `<head>` used instead) or, if both somehow ended up in the output, would produce an invalid HTML document with two `<head>` elements.

The build is strict: a literal `<head>` in your JSX is flagged immediately, because the only legitimate use cases (raw HTML injection) are handled by the `metadata` API.

## Reproducing the error

```tsx filename="app/layout.tsx"
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <head>
        <title>My App</title>
      </head>
      <body>{children}</body>
    </html>
  )
}
```

## Error output

```
error: [SR0108] duplicate <head>: root layout must not render a <head> element
  --> app/layout.tsx:4:5
   |
 4 |     <head>
   |     ^^^^^ use the `metadata` export instead
```

## How to fix it

1. Remove the literal `<head>` from the JSX.
2. Declare the page title and other metadata in the `metadata` export.

```tsx filename="app/layout.tsx"
import type { ReactNode } from "react"
import type { Metadata } from "swift-rust"

export const metadata: Metadata = {
  title: {
    default: "My App",
    template: "%s — My App",
  },
  description: "The best app",
}

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

For per-page metadata, export a `metadata` constant from the page or any layout in the segment.

```tsx filename="app/about/page.tsx"
import type { Metadata } from "swift-rust"

export const metadata: Metadata = {
  title: "About",
}

export default function About() {
  return <h1>About</h1>
}
```

## Diagnosis

Not applicable: this error is detected at runtime/build only, no CLI command.

## Common pitfalls

- A `<head>` in a deeper layout is also rejected — metadata is hoisted, not duplicated.
- The framework does not support a "raw HTML in head" escape hatch directly. If you need a third-party snippet (a meta verification tag, for example), put it in `metadata.verification` or `metadata.other`, both of which the framework will render correctly.
- A `<head>` in a `not-found.tsx` boundary is also rejected; the global 404 page shares the root layout's metadata.
