# SR0088 — Minification disabled in production

The production build produced unminified JavaScript bundles. The output still works, but every user downloads larger files, the server parses more code, and SSR memory usage goes up. This is reported as an informational message because the build succeeds — it is a performance warning, not a correctness error.

## Why this error occurred

Swift-Rust's bundler (esbuild, wrapped by a thin Rust shim) minifies by default in production. Minification is controlled by `optimization.minimize` and `optimization.minimizer` in `swift-rust.config.json`. If either is set to disable the default minimizer, the build emits bundles with full variable names, comments, and whitespace. The build does not crash because the output is still valid JavaScript, but you almost certainly did not intend this.

Disabling minification in production is occasionally legitimate (e.g. when bundling for a target that requires readable source), so the framework warns rather than errors.

## Reproducing the error

```json filename="swift-rust.config.json"
{
  "optimization": {
    "minimize": false
  }
}
```

## Error output

```
info: [SR0088] minification disabled in production
       bundles will not be minified; users will download unminified JavaScript
  --> swift-rust.config.json:2:16
   |
 2 |   "optimization": {
 3 |     "minimize": false
   |              ^^^^^^
```

## How to fix it

1. Remove the `optimization.minimize: false` override.
2. If you replaced the minimizer, restore the default by removing the `minimizer` key.
3. If you need a custom minimizer (Terser, esbuild with custom config), keep the array non-empty and point it at your function.

```json filename="swift-rust.config.json"
{
  "optimization": {
    "minimize": true
  }
}
```

## Diagnosis

```bash
bun swift-rust info --field bundler.minify
RUST_LOG=swift_rust=bundling=trace bun swift-rust build
```

`--field bundler.minify` reports the resolved minifier configuration.

## Common pitfalls

- A minifier replacement that returns the input unchanged (e.g. `TerserPlugin` with `parallel: false` and no actual minify call) silently disables minification.
- Production builds via `bun run build` respect this flag; dev mode does not minify at all, so the warning only appears in production.
- If you are intentionally producing unminified output for source-map-only debugging, you can disable the warning in `swift-rust.config.json` under `diagnostics.suppress`.
