# SR0037 — Duplicate Sass dependencies

Both `sass` and `node-sass` are installed in the project. The two packages are different implementations of the same Sass spec, and Swift-Rust only compiles with one of them. The build fails so you are forced to pick.

## Why this error occurred

`node-sass` was the historical implementation, written as a native binding around LibSass. It is deprecated upstream and no longer receives fixes. `sass` (also called `dart-sass`) is the modern, pure-JS implementation and is the canonical package today. Having both installed at once means the resolver has to pick one, and the other silently does nothing — which is a source of subtle bugs (e.g. a `.sass` syntax that only `sass` understands will fail to compile under the chosen one).

The build refuses to start because the choice should be deliberate, not silent.

## Reproducing the error

```json
{
  "name": "site",
  "dependencies": {
    "sass": "^1.70.0",
    "node-sass": "^7.0.0"
  }
}
```

## Error output

```
[ERROR] [SR0037] duplicate Sass dependencies: both `sass` and `node-sass` are installed
   --> package.json:4:5
   |
 4 |     "node-sass": "^7.0.0"
   |     ^^^^^^^^^^^^^^^^^^^^^^ remove this package; `sass` is the supported implementation
   |
  help: `sass` covers the full Sass spec and is faster to install. Run `bun remove node-sass`.
```

## How to fix it

1. Remove `node-sass` from `package.json`.
2. Verify no other dependency in the tree still requires it (`bun why node-sass` will print the dep chain).
3. Reinstall to refresh the lockfile, and run the build to confirm the error is gone.

```bash
bun remove node-sass
bun install
```

```json
{
  "name": "site",
  "dependencies": {
    "sass": "^1.70.0"
  }
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log names the package version that would be selected and the one that would be ignored, so you can confirm the build is now deterministic.

## Common pitfalls

- Some legacy starters ship `node-sass` by default. Re-running `create-swift-rust` with the current CLI drops it.
- A peer dependency that still requires `node-sass` will fail to install once `node-sass` is removed. Pin the offending package or migrate to its modern alternative.
- `sass` has a different CLI than `node-sass` (`sass` is `dart-sass`). Custom build scripts that call the old binary need to be updated.
- The build error fires for any pair of competing implementations, not just these two. If you maintain a fork, register it under a different name.
