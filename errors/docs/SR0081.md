# SR0081 — Invalid Swift-Rust server options

The framework's `createServer` factory inspects its options object at boot. When the object is `null`, `undefined`, or missing required keys, this error is raised before the listener can bind.

## Why this error occurred

The Swift-Rust server factory is shaped around an options object with strongly-typed fields: `dev`, `port`, `hostname`, `dir`, and `conf`. If you import the factory from a custom entrypoint (for example a Bun script, an Express wrapper, or a `server.ts` file) and forget to forward the resolved options, the factory receives `undefined`. The same happens if the options object is created inside a conditional branch that never executes, or if the import is wrong and returns a different symbol.

Custom servers in Swift-Rust are uncommon because the framework ships its own axum-based server runtime. When you do need one — for example to share a port with an existing service, or to add a custom health endpoint — you must pass a complete options object.

## Reproducing the error

```ts filename="server.ts"
import { createServer } from "swift-rust/server"

const app = createServer()
```

This compiles, but at startup the server factory reads `options.dir` and receives `undefined`, which fails the contract check.

## Error output

```
error: [SR0081] invalid server options: options is undefined, expected { dev, port, hostname, dir, conf }
  --> server.ts:3:19
   |
 3 | const app = createServer()
   |                   ^^
```

## How to fix it

1. Pass a literal options object with the keys your app needs.
2. Read the values from environment variables or your Swift-Rust config.
3. Make sure the import resolves to the public factory from `swift-rust/server`.

```ts filename="server.ts"
import { createServer } from "swift-rust/server"

const app = createServer({
  dev: process.env.NODE_ENV !== "production",
  port: Number(process.env.PORT ?? 3000),
  hostname: "0.0.0.0",
  dir: process.cwd(),
  conf: "./swift-rust.config.json",
})
```

## Diagnosis

```bash
bun swift-rust info --field server
RUST_LOG=swift_rust=trace bun swift-rust dev
```

## Common pitfalls

- Passing `null` from a JSON config that omitted the `server` block.
- Using `const { dev } = options; createServer({ dev })` and silently dropping the other keys.
- Importing the symbol from a path that does not export the factory (a typed module that re-exports a stub).
