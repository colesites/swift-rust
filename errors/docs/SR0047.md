# SR0047 — Failed to fetch devPagesManifest

When you run `bun swift-rust dev`, the framework exposes a small JSON manifest at `/_swift-rust/dev-pages-manifest.json` that lists every route known to the dev server. The client router fetches this manifest once on hydration to know which routes are eligible for client-side navigation and prefetching. If the fetch fails, navigation falls back to full page reloads and prefetching silently breaks.

## Why this error occurred

The dev manifest is an optimization that is only present in `bun swift-rust dev`. It is served from the same Bun process that runs the dev server, so any layer that intercepts the request can break it.

The most common culprits are:

- A browser extension (ad blocker, privacy tool) that rewrites or cancels requests to `/_swift-rust/*`.
- A reverse proxy in front of the dev server (nginx, Caddy, Cloudflare Tunnel) that has not been told to forward `_swift-rust/*` paths.
- The dev server crashed or restarted between the page render and the manifest fetch.
- Mixed-content blocking when the page is served over `https://` but the manifest URL resolves to `http://`.

In Swift-Rust, the manifest is named `dev-pages-manifest.json` (Next.js uses `_devPagesManifest.json`). The semantics are otherwise identical: it is the source of truth for the router's in-memory page table.

## Reproducing the error

```tsx
import Link from "swift-rust/link";

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div>
      <nav>
        <Link href="/about">About</Link>
      </nav>
      {children}
    </div>
  );
}
```

```bash
bun swift-rust dev
```

## Error output

```
[ERROR] [SR0047] failed to fetch devPagesManifest: net::ERR_BLOCKED_BY_CLIENT
  --> GET /_swift-rust/dev-pages-manifest.json
  |
  = hint: a browser extension, ad blocker, or proxy is blocking the request.
          the router will fall back to full page loads and skip prefetching.
```

## How to fix it

1. Disable any ad blocker or privacy extension in the affected browser and reload.
2. Confirm the dev server is reachable at the origin you are loading.

   ```bash
   curl -i http://localhost:3000/_swift-rust/dev-pages-manifest.json
   ```

3. If you proxy the dev server, forward the manifest path.

   ```nginx
   location /_swift-rust/ {
     proxy_pass http://127.0.0.1:3000;
   }
   ```

4. In production this manifest does not exist, so the warning cannot appear there. If you see it in a built app, you are loading a dev build by mistake.

## Diagnosis

```bash
bun swift-rust doctor
curl -sI http://localhost:3000/_swift-rust/dev-pages-manifest.json | head -1
RUST_LOG=swift_rust=trace bun swift-rust dev
```

## Common pitfalls

- This error is dev-only. In a production build (`bun swift-rust build` + `bun swift-rust start`) the manifest is not fetched.
- Cloudflare Access and most zero-trust proxies will block the request unless you add `/_swift-rust/*` to the bypass list.
- Some Brave shields treat any path containing `_` as a tracking endpoint; the `_swift-rust` prefix is intentional so it is easy to allowlist.
