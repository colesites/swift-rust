# SR0123 — Image placeholder=blur without blurDataURL

The `<Image>` component from `swift-rust/image` was asked to render a low-resolution blur placeholder, but the component has nothing to display. Swift-Rust cannot synthesize a preview from a remote URL on the fly, and the static import is either missing or in an unsupported format.

## Why this error occurred

The blur placeholder needs an explicit data URL. When you write `placeholder="blur"`, the `<Image>` component renders a tiny base64-encoded JPEG (the `blurDataURL`) over the real image until it loads, hiding the pop-in. The data URL must come from one of three sources:

- You supplied it directly via the `blurDataURL` prop.
- Swift-Rust auto-generated it from a static import of a supported format.
- You used a built-in preset (`empty`, `shimmer`, or `color`) which Swift-Rust ships in the framework.

If none of those match, the placeholder is missing and the build emits this warning. The image still renders, but without the blur effect.

## Reproducing the error

```tsx
import Image from "swift-rust/image";

export default function Avatar({ url }: { url: string }) {
  return <Image src={url} alt="avatar" width={64} height={64} placeholder="blur" />;
}
```

```tsx
import Image from "swift-rust/image";
import hero from "./hero.gif";

export default function Hero() {
  return <Image src={hero} alt="hero" placeholder="blur" />;
}
```

## How to fix it

1. Provide an explicit `blurDataURL`:

```tsx
import Image from "swift-rust/image";

const blur =
  "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAQAAAC1HAwCAAAAC0lEQVR42mNkYAAAAAYAAjCB0C8AAAAASUVORK5CYII=";

export default function Avatar({ url }: { url: string }) {
  return (
    <Image
      src={url}
      alt="avatar"
      width={64}
      height={64}
      placeholder="blur"
      blurDataURL={blur}
    />
  );
}
```

2. Or import the image statically and use a supported format (jpg, png, webp, avif):

```tsx
import Image from "swift-rust/image";
import hero from "./hero.webp";

export default function Hero() {
  return <Image src={hero} alt="hero" placeholder="blur" width={1280} height={720} />;
}
```

3. Or drop the `placeholder` prop entirely if you do not need the blur.

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=image=debug bun swift-rust dev
```

The `image=debug` log shows which `blurDataURL` resolution path was attempted (static import, prop, preset) and which one was missing.

## Common pitfalls

- Static imports of animated webp/gif still fail this check; Swift-Rust does not generate blur previews for animated sources.
- `blurDataURL` should be tiny — 10 to 30 pixels on the longest side. A 1024-pixel JPEG defeats the point of the placeholder.
- The `<Image>` component from `swift-rust/image` does not auto-fall-back to fetching the source to generate a blur preview. The data URL is required.
- For remote sources, you can compute the blur on a CI step with `sharp` and inject the result as a prop.
