# SR0023 — Cannot output to /static

The configured build output directory is `static/`, but Swift-Rust uses that folder to serve static assets. Writing the build output there would clobber user assets and create a cycle, so the build refuses to run.

## Why this error occurred

`static/` (like `public/`) is exposed verbatim at runtime. Any file placed there becomes a publicly addressable URL. If the build output lived in the same directory, the framework's own static-asset pipeline would serve intermediate output as if it were an asset, and the user's own files would conflict with the build's chunk filenames.

The check is the same whether the path was set via `distDir` or chosen as a static-export target.

## Reproducing the error

```json
{
  "name": "site",
  "distDir": "static"
}
```

## Error output

```
[ERROR] [SR0023] cannot output to /static: that directory is reserved for static assets
   --> swift-rust.config.json:2:12
   |
 2 |   "distDir": "static"
   |              ^^^^^^^^ pick a different output directory
   |
  help: remove the field to use the default (.swift-rust), or set distDir to e.g. "out".
```

## How to fix it

1. Remove the `distDir` field and let the default (`.swift-rust/`) take over.
2. If you want a self-contained output, point `distDir` at any folder that is not `public/`, `static/`, or a source folder.
3. Re-run the build and confirm the error is gone.

```json
{
  "name": "site"
}
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log records the resolved absolute path of `distDir`, so you can confirm what the framework actually saw (relative paths are resolved against the project root).

## Common pitfalls

- The check is on the **resolved** path. A symlink that points into `static/` will also be rejected.
- A static export that targets a subdirectory of `static/` (e.g. `static/build`) is allowed, but the contents will be served as static assets and exposed at build URLs.
- The same check covers `public/` — see `SR0022` for the public-folder equivalent.
