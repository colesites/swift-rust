# SR0043 — experimental.jest transformer is deprecated

The `experimental.useJestTransformer` flag and the companion `@swift-rust/jest` package are deprecated. The framework no longer ships a first-party Jest transformer; use the upstream `@swc/jest` plugin instead.

## Why this error occurred

`@swift-rust/jest` was a thin wrapper around SWC's Jest plugin, kept in the monorepo to ensure version compatibility with the compiler. The wrapper duplicated work that `@swc/jest` already does, and tracking two releases per SWC bump was not worth the benefit. The deprecation unifies the story: projects use `@swc/jest` directly and the framework no longer carries a fork.

The deprecation is non-fatal. The flag is parsed and the legacy package still resolves, but the build prints a warning so the migration is unmissable.

## Reproducing the error

```json
{
  "name": "site",
  "experimental": {
    "useJestTransformer": true
  }
}
```

```json
{
  "devDependencies": {
    "@swift-rust/jest": "^0.1.0"
  }
}
```

## Error output

```
[WARN] [SR0043] experimental.useJestTransformer is deprecated: use @swc/jest directly
   --> swift-rust.config.json:3:5
   |
 3 |     "useJestTransformer": true
   |     ^^^^^^^^^^^^^^^^^^^^^^^^^^^ remove this field; install @swc/jest and configure jest.config.ts to use it
   |
  help: see https://swift-rust.dev/docs/testing for the recommended Jest setup.
```

## How to fix it

1. Remove `experimental.useJestTransformer` from `swift-rust.config.json`.
2. Uninstall `@swift-rust/jest` and install `@swc/jest` instead.
3. Update `jest.config.ts` to use `@swc/jest` as the transformer.
4. Run the test suite to confirm everything still works.

```bash
bun remove @swift-rust/jest
bun add -D @swc/jest
```

```ts
import type { Config } from "jest";

const config: Config = {
  transform: {
    "^.+\\.(t|j)sx?$": ["@swc/jest", { /* options */ }],
  },
};

export default config;
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust test
```

The trace log lists every read of the deprecated field, so you can confirm no third-party tooling is silently depending on it.

## Common pitfalls

- A nested `experimental` block can have other valid keys. Remove only `useJestTransformer`.
- `@swift-rust/jest` will be removed in a future major. Migrate as soon as practical.
- The new transformer is configured via `jest.config.ts`, not `swift-rust.config.json`. The two configs are independent.
- Other test runners (Vitest, Bun's test runner) do not need a special transformer at all — they handle TypeScript natively.
- This deprecation will become a hard error in Swift-Rust 0.4.0.
