# SR0028 — Conflicting public file and page

A file in `public/` and a route under `app/` resolve to the same URL. Swift-Rust refuses to ship a project where two different resources are addressable at the same path because the resolution order (page vs. static asset) is undefined.

## Why this error occurred

`public/` files are served verbatim by the Swift-Rust server. App routes are also served at their segment path. If a `public/hello` file and an `app/hello/page.tsx` both exist, the URL `/hello` is ambiguous — depending on which middleware is consulted first, the user could see either response. The framework treats the ambiguity as a configuration error so you must pick one.

The same rule covers `static/` (the legacy alias) and any file extension a route segment does not have, so a `public/hello.txt` is fine but a `public/hello` (no extension) is not.

## Reproducing the error

```bash
public/
  hello
app/
  hello/
    page.tsx
```

## Error output

```
[ERROR] [SR0028] conflicting public file and page: /hello
   --> public/hello:1:1
   |
 1 | (binary or text file)
   | ^^^^^^^^^^^^^^^^^^^^^^^ same URL as app/hello/page.tsx
   |
  help: rename the public file (e.g. public/hello.txt) or move the page to a different segment.
```

## How to fix it

1. Pick which resource should win and rename the other so the URLs differ.
2. If you want to keep the static file but the file has no extension, give it one (e.g. `public/hello.txt`).
3. If you actually want the static file to be served at the root, delete the page.

```bash
public/
  hello.txt
app/
  hello/
    page.tsx
```

## Diagnosis

```bash
bun swift-rust doctor
bun swift-rust info
RUST_LOG=swift_rust=trace bun swift-rust build
```

The trace log lists every public-file/page pair that collided, so you can fix all of them in one pass.

## Common pitfalls

- Extensions matter. `public/hello` collides with `app/hello/page.tsx`; `public/hello.txt` does not.
- `public/_next/...` files are owned by the build and never count as a conflict, even if you have a `/_next` route (which is reserved).
- A trailing slash does not help: `public/hello/` and `app/hello/page.tsx` are still the same URL.
- This error is detected at build time. It will not appear for a file added to `public/` after the build.
