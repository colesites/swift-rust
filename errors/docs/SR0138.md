# SR0138 — revalidateTag called with one argument

`revalidateTag` now requires a second argument that specifies the cache life profile (`'max'` or any custom profile defined in `swift-rust.config.json`). A call with only the tag is rejected.

## Why this error occurred

Cache invalidation has two distinct semantics:

- **Stale-while-revalidate** — mark the entry as stale, serve the cached value to the next request, and refresh in the background. This is what `revalidateTag(tag, 'max')` does.
- **Immediate expiration** — drop the entry, force the next request to wait for fresh data. This is what `updateTag(tag)` does, and it is only available inside Server Actions.

The single-argument form was ambiguous about which behavior the caller wanted, so the framework removed it. The error fires at compile time (or at request time, if the call site is inside a dynamic function) so the migration is easy to spot.

## Reproducing the error

```ts
import { revalidateTag } from "swift-rust/cache";

export async function action() {
  await doSomething();
  revalidateTag("posts");
}
```

## Error output

```
[ERROR] [SR0138] revalidateTag requires a second argument
  --> app/actions.ts:5:3
    |
  5 |   revalidateTag("posts");
    |   ^^^^^^^^^^^^^
    | `revalidateTag` now requires a cache-life profile as the second argument.
    |
    help: pass `'max'` (or a custom profile) as the second argument, or use `updateTag` for read-your-own-writes.
```

## How to fix it

1. Add a second argument specifying the profile:

```ts
import { revalidateTag } from "swift-rust/cache";

export async function action() {
  await doSomething();
  revalidateTag("posts", "max");
}
```

2. If you are inside a Server Action and need immediate expiration (read-your-own-writes), use `updateTag` instead:

```ts
"use server";

import { updateTag } from "swift-rust/cache";

export async function createPost() {
  await db.posts.insert({ ... });
  updateTag("posts");
}
```

3. If you have a custom profile defined in your config, pass its name:

```json
{
  "cache": {
    "profiles": {
      "fast": { "stale": 30, "revalidate": 60, "expire": 300 }
    }
  }
}
```

```ts
revalidateTag("posts", "fast");
```

## Diagnosis

```bash
bun swift-rust doctor
RUST_LOG=swift_rust=cache=trace bun swift-rust dev
```

`cache=trace` logs every `revalidateTag` and `updateTag` call with the resolved profile, which makes it easy to confirm the migration landed.

## Common pitfalls

- A Server Action that does `revalidateTag('posts')` and then immediately reads from a query that depends on `posts` will return stale data. Use `updateTag` for that pattern.
- The two-argument form is required at compile time, but the framework also enforces it at runtime in case a dynamic call site slipped through.
- A custom profile name that does not exist in the config throws a different error (SR0021) at runtime.
- `revalidatePath` and `revalidateTag` are separate APIs; this change does not affect `revalidatePath`.
