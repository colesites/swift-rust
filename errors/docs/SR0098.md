# SR0098 — Unconfigured image host

The `<Image>` component received a `src` whose host (or protocol, port, or path) is not covered by the `image.remotePatterns` allow-list in `swift-rust.config.json`. The build refuses to optimize the image, which would leak a request to an arbitrary origin.

## Why this error occurred

Image optimization is a server-side function: the framework fetches the image, transcodes it, caches it, and returns the optimized URL. Allowing it to fetch from any URL would be an SSRF vector and would also produce untrackable egress traffic. For both reasons, Swift-Rust requires every remote origin to be declared in `image.remotePatterns`.

The match is strict and case-sensitive on every part of the URL:

- **Protocol.** `https` and `http` do not match each other.
- **Hostname.** `cdn.example.com` does not match `www.example.com` or `assets.cdn.example.com`.
- **Port.** If the pattern omits the port, URLs with an explicit port are rejected.
- **Pathname.** A pattern of `/images/foo.jpg` matches only that exact path. Use `/images/**` to cover nested paths.
- **Search.** Patterns can pin a specific query string; if `search` is omitted, any query string is allowed; if `search: ''` is set, only URLs without a query string match.

A failure on any one of these produces this error. The framework reports the URL it tried to match so the misconfiguration is obvious.

## Reproducing the error

```tsx filename="app/page.tsx"
import Image from "swift-rust/image"

export default function Page() {
  return (
    <Image
      src="https://assets.cdn.example.com/account123/hero.jpg"
      alt="Hero"
      width={1200}
      height={600}
    />
  )
}
```

`swift-rust.config.json`:

```json
{
  "image": {
    "remotePatterns": [
      { "protocol": "https", "hostname": "cdn.example.com", "pathname": "/**" }
    ]
  }
}
```

The pattern covers `cdn.example.com` but the URL uses `assets.cdn.example.com` — a subdomain mismatch.

## How to fix it

1. Add a pattern that matches the actual host and path.
2. Use `**` to cover nested paths.
3. If you need multiple hosts, list each one.

```json filename="swift-rust.config.json"
{
  "image": {
    "remotePatterns": [
      {
        "protocol": "https",
        "hostname": "assets.cdn.example.com",
        "pathname": "/account123/**"
      }
    ]
  }
}
```

```tsx filename="app/page.tsx"
import Image from "swift-rust/image"

export default function Page() {
  return (
    <Image
      src="https://assets.cdn.example.com/account123/hero.jpg"
      alt="Hero"
      width={1200}
      height={600}
    />
  )
}
```

## Diagnosis

```bash
bun swift-rust doctor --check image
bun swift-rust info --image
```

`--image` lists every configured pattern and the hosts it covers.

## Common pitfalls

- The older `image.domains` array (a plain list of hostnames) is still accepted for backward compatibility, but the new `remotePatterns` form is more flexible. Prefer the new form.
- A pattern without a `pathname` matches every path. If you want a narrower scope, set an explicit glob.
- The `URL` constructor strips the search string when none is provided. If your images include `?v=...`, the search needs to be allowed in the pattern.
