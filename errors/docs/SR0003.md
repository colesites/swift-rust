# SR0003 — Compilation error

The SWC plugin transforms JSX into an intermediate form, and the Rust code generator emits a `.rs` file per route. A failure in either stage surfaces here, with the source position pointed at the offending construct.

## Why this error occurred

Swift-Rust is not a JavaScript runtime. JSX/TSX is a *source language* that gets compiled down to native Rust view code, and that pipeline is the framework's hottest path. Any construct that the SWC plugin cannot lower to the Rust AST is a hard error.

The plugin runs first, then the type-checker (via `tsc`), then the Rust codegen. The reported position is the first stage at which the file failed, so the underlined line is usually the real bug, not a downstream symptom.

## Reproducing the error

```tsx filename="app/page.tsx"
import { Foo } from "@/components/missing";

export default function Home() {
  return <Foo />;
}
```

The `Foo` component does not exist. The plugin fails at the import, then the Rust generator never runs.

```tsx filename="app/page.tsx"
export default function Home() {
  return <div>{unknownVar}</div>;
}
```

A `noUnusedLocals`-style error or a missing import that the type-checker catches.

## Error output

```

  --> app/page.tsx:3:7
   |
 3 |   <Foo />
   |   ^^^ no export `Foo` from `@/components/missing`
   |
   help: check the export name in the target file
```

```
error: [SR0003] compilation error in app/page.tsx: cannot find name `unknownVar`
  --> app/page.tsx:2:14
   |
 2 |   return <div>{unknownVar}</div>;
   |              ^^^^^^^^^^ not declared in this scope
```

## How to fix it

1. Open the file at the underlined position and read the line in context.
2. Check the import path: is the file at the expected location? Is the export name spelled correctly?
3. For type errors, run `bunx tsc --noEmit` to get the full list — the plugin reports the first one it hits and stops.

```tsx filename="app/page.tsx"
import { Foo } from "@/components/foo";

export default function Home() {
  return <Foo />;
}
```

```tsx filename="components/foo.tsx"
export function Foo() {
  return <div>hello</div>;
}
```

## Diagnosis

```bash
bun swift-rust build --verbose
RUST_LOG=swift_rust::compiler=trace bun swift-rust build
RUST_LOG=swift_rust::swc=trace bun swift-rust build
```

`--verbose` shows every file the compiler processed, plus the order in which they were processed. The two `RUST_LOG` channels split the trace between the Rust codegen stage and the SWC plugin stage, so you can tell which one failed.

## Common pitfalls

- The SWC plugin reports position from the *post-transform* AST, which can be off by a line for files that mix TSX with type-only imports.
- A `// @ts-expect-error` directive silences the TypeScript checker but not the SWC plugin; if the plugin cannot see the symbol, the directive does not help.
- Decorators, enums, and `as const` assertions require explicit flags in `swift-rust.config.json` under `compiler.experimental`.
- Compilation errors abort the build, but a syntax-only error in a single `.tsx` file does not stop the dev server. The dev server logs the error and continues with the previous successful compile of that file.
